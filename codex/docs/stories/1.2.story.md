# Story 1.2 - Espelho SGD com Filtro por SRE e Objeto

## Status
In Progress

## Objetivo
Construir o primeiro fluxo funcional do produto: coletar cotações do SGD em painel proprio e permitir filtro por SRE, municipio e objeto.

## Criterios de Aceite

- [x] Login assistido no SGD funcional para coleta.
- [x] Tabela de cotações espelhada com colunas: ID, Ano, Escola, Prazo, Status.
- [x] Escola mapeada para SRE e municipio.
- [x] Busca por objeto com classificacao inicial.
- [x] Filtros combinados funcionando (SRE + municipio + objeto + status + prazo).
- [x] Exportacao CSV da lista filtrada.
- [x] Log de execucao da coleta salvo.

## Tarefas

- [x] Criar modelo de dados `cotacoes`, `escolas`, `sres`, `objetos`.
- [x] Implementar coletor base da rota `/compras/orcamentos`.
- [x] Construir mapeamento escola -> SRE (seed inicial).
- [x] Implementar classificador simples de objeto (keywords).
- [x] Criar tela de listagem com filtros e busca.
- [x] Criar exportacao CSV.
- [x] Registrar evidencias de teste com capturas.

## Riscos

- Mudancas de layout no SGD podem quebrar coleta.
- Texto do objeto pode vir inconsistente/insuficiente.

## Mitigacoes

- Versionar seletores de scraping por pagina.
- Logar falhas por etapa e ter retentativa.
- Permitir correcao manual de classificacao do objeto.

## Definition of Done

- [x] Criterios de aceite validados.
- [x] Story atualizada com evidencias.
- [x] Lista de arquivos alterados atualizada.

## Evidencias

- Coleta real do SGD executada com sucesso (`10` cotacoes extraidas).
- Exportacao CSV adicionada na tabela priorizada (`Exportar CSV`) com base na visao filtrada.
- Score de oportunidade (0-100) implementado no dashboard e no CSV (`score_oportunidade`).
- Alertas criticos de prazo adicionados no dashboard (top prazos <= 5 dias por filtro).
- Coletor atualizado com classificacao de objeto por keywords e `confiancaObjeto` (`alta/media/baixa`).
- Mapeamento escola -> municipio/SRE movido para arquivo curavel (`school-territory-map.json`) com `confiancaTerritorio`.
- Score de oportunidade refinado com peso estrategico por tipo de objeto.
- Auditoria automatica de cobertura territorial criada (`dashboard:audit:territory`) com relatorio em JSON e Markdown.
- Acoes rapidas no dashboard adicionadas: `Prioridade Alta`, `Prazo 48h`, `Baixa confianca` e `Exportar Urgentes`.
