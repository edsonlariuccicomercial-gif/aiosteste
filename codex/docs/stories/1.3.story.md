# Story 1.3 - Integracao de Pedido Pos-Licitacao (LicitIA) para Olist

## Status
Done (Fase 2 retomada em 03/03/2026)

## Objetivo
Quando um pedido for gerado e aprovado no modulo de pos-licitacao do LicitIA, replicar automaticamente esse pedido na aba de pedidos do Olist para operacao e emissao de nota posterior.

> Nota de priorizacao: bloco retomado em 03/03/2026 para concluir observabilidade e status operacional de pedidos.

## Criterios de Aceite

- [x] Evento de pedido aprovado para faturamento no LicitIA detectado.
- [x] Pedido correspondente criado no Olist automaticamente.
- [x] Campos minimos sincronizados: id pedido interno, escola, itens, quantidades, valores, prazo.
- [x] Rastreabilidade cruzada persistida (`internal_order_id` e `olist_order_id`).
- [x] Idempotencia ativa (nao criar pedido duplicado no Olist).
- [x] Retentativa automatica em falhas transitorias.
- [x] Status de sincronizacao visivel no dashboard (fila/enviado/aceito/erro).
- [x] Log tecnico detalhado por tentativa para suporte operacional.

## Tarefas

- [x] Definir contrato de dados Pos-Licitacao LicitIA -> Olist.
- [x] Implementar adaptador de criacao de pedido no Olist.
- [x] Implementar fila de sincronizacao e retentativa.
- [x] Implementar estrategia de idempotencia por pedido interno do LicitIA.
- [x] Persistir tabela de correlacao de IDs.
- [x] Exibir status no dashboard (fila/enviado/aceito/erro).
- [x] Criar teste de fluxo feliz e fluxo com falha/reprocessamento.

## Riscos

- Mudanca no fluxo interno de geracao/aprovacao de pedido no LicitIA.
- Limites de API/autenticacao do Olist.
- Duplicidade de eventos de pedido.

## Mitigacoes

- Camada de adaptacao de eventos internos do modulo pos-licitacao.
- Controle de rate limit e retentativa exponencial.
- Chave idempotente e validacao antes de criar pedido.
