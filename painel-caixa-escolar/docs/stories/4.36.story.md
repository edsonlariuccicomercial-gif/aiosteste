## Story

**As a** operador que gera demandas a partir de pedidos,
**I want** que o sistema converta automaticamente as quantidades do pedido (unidade do contrato) para as quantidades reais de compra (unidade do mercado),
**so that** eu saiba exatamente o que comprar e em qual quantidade.

---

## Contexto / Motivação

O fluxo pós-licitação da Caixa Escolar é:

```
Contrato → Pedido (da escola) → Nota Fiscal → Demanda → Estoque / Compras
```

Quando uma escola pede 50 unidades de "PAPEL TOALHA INTERFOLHADO 100% CELULOSE VIRGEM 23X21CM" (a descrição do contrato, onde UN = 1 folha), o operador precisa converter isso para a unidade de mercado real. Se o produto vinculado é "Papel Toalha Mili cx c/ 1000fls" (1 CX = 1000 folhas) e cada folha equivale a 20 usos, então 50 UN contrato → 1 CX.

Esta conversão manual é fonte de erro e lentidão. Após o vínculo estabelecido na Story 4.35, o sistema tem todos os dados para fazer a conversão automaticamente ao gerar uma demanda.

### Estrutura existente relevante

- `gdp.contratos.v1` — contratos com itens no formato do contrato
- `gdp.pedidos.v1` — pedidos das escolas com itens e quantidades
- `gdp.equivalencias.v1` — vínculo descricaoContrato → SKU do produto real (Story 4.35)
- `bancoPrecos.itens[]` — produtos com campos `unidadeCompra`, `custo`, `sku`
- `UNIT_CONVERSIONS` — conversão de embalagens existente no app.js

## Acceptance Criteria

### AC-1: Regras de conversão por equivalência

- [ ] Cada vínculo em `gdp.equivalencias.v1` pode ter uma regra de conversão associada em `gdp.conversoes.v1`:
  ```
  { "descricaoNormalizada": { fatorConversao: 1000, unidadeOrigem: "UN", unidadeDestino: "CX", nota: "1 CX = 1000 folhas" } }
  ```
- [ ] Ao vincular um produto pela primeira vez (Story 4.35), o sistema tenta inferir o fator a partir de `UNIT_CONVERSIONS` e do nome do produto
- [ ] Se não for possível inferir automaticamente, o fator fica como `null` e é marcado para configuração manual
- [ ] O operador pode editar o fator de conversão a qualquer momento pelo modal de vínculo

### AC-2: Engine de conversão — `converterDemanda(pedidoItens)`

- [ ] Função `converterDemanda(pedidoItens)` recebe array de itens do pedido e retorna array de itens convertidos
- [ ] Para cada item do pedido:
  1. Busca equivalência: `getEquivalencia(item.descricao)` → SKU
  2. Busca produto: `getProdutoBySku(sku)` → produto com `unidadeCompra`, `custo`
  3. Busca regra de conversão: `getConversao(item.descricao)` → `{ fatorConversao, unidadeOrigem, unidadeDestino }`
  4. Calcula `qtdCompra = Math.ceil(item.quantidade / fatorConversao)`
  5. Calcula `custoEstimado = qtdCompra * produto.custo`
- [ ] Item sem equivalência: retorna com status `"sem_vinculo"` — não bloqueia o processamento dos demais
- [ ] Item sem fator de conversão: retorna com status `"sem_conversao"` — exige ação manual
- [ ] Item com tudo resolvido: status `"convertido"`

### AC-3: View de demanda com colunas original × convertido

- [ ] Existe uma view de "Demanda" (nova seção ou modal) acessível a partir de um pedido confirmado
- [ ] A view exibe tabela com as colunas:
  - Item (descrição do contrato)
  - Qtd Pedida (unidade do contrato, ex: 50 UN)
  - Produto Vinculado (nomeComercial ou badge "Sem vínculo")
  - Qtd Compra (unidade de mercado, ex: 1 CX)
  - Custo Estimado (ex: R$ 45,90)
  - Status (badge: "Convertido" / "Sem vínculo" / "Sem conversão")
- [ ] Linha de totais no rodapé: custo estimado total da demanda
- [ ] Itens com status "Sem vínculo" ou "Sem conversão" ficam destacados em vermelho/amarelo

### AC-4: Itens sem vínculo — ação inline

- [ ] Itens com status "Sem vínculo" exibem botão "Vincular agora" que abre o modal da Story 4.35
- [ ] Após vincular, a linha é atualizada automaticamente sem recarregar a página
- [ ] Itens com status "Sem conversão" exibem botão "Definir fator" que abre mini-modal de configuração

### AC-5: Mini-modal de configuração do fator de conversão

- [ ] Modal com campos:
  - Descrição do item (somente leitura)
  - Produto vinculado (somente leitura)
  - Fator de conversão (number — ex: 1000)
  - Unidade de origem (ex: UN)
  - Unidade de destino (ex: CX)
  - Nota explicativa (ex: "1 CX = 1000 folhas de 20x21cm")
- [ ] Ao salvar, a linha é recalculada e o status muda para "Convertido"
- [ ] A regra é salva em `gdp.conversoes.v1` e reutilizada automaticamente em demandas futuras com o mesmo item

### AC-6: Override manual da quantidade convertida

- [ ] O operador pode editar manualmente o campo "Qtd Compra" para qualquer item
- [ ] Edição manual cria flag `overrideManual: true` no item da demanda
- [ ] Item com override exibe ícone de lápis e tooltip "Quantidade ajustada manualmente"
- [ ] Override não altera a regra de conversão salva — é por-demanda

### AC-7: Salvar demanda convertida

- [ ] Botão "Salvar Demanda" persiste a demanda convertida em `gdp.demandas.v1`
- [ ] Estrutura da demanda salva inclui: id, pedidoId, contratoId, dataCriacao, itens convertidos, custoTotalEstimado, status (`"rascunho"` ou `"confirmada"`)
- [ ] Status inicial: `"rascunho"` — só se torna `"confirmada"` quando o operador confirmar explicitamente

### AC-8: Sem regressão

- [ ] Pedidos existentes em `gdp.pedidos.v1` sem demanda continuam acessíveis
- [ ] A geração de demanda é opcional — pedido pode existir sem demanda associada
- [ ] `UNIT_CONVERSIONS` existente não é alterado

## Dev Notes

### Engine de conversão

```javascript
const GDP_CONV_KEY = 'gdp.conversoes.v1';

function getConversao(descricao) {
  const conv = JSON.parse(localStorage.getItem(GDP_CONV_KEY) || '{}');
  return conv[normalizedText(descricao)] || null;
}

function converterItemDemanda(item) {
  const sku = getEquivalencia(item.descricao);
  if (!sku) return { ...item, status: 'sem_vinculo', qtdCompra: null, custoEstimado: null };

  const produto = getProdutoBySku(sku);
  const regra = getConversao(item.descricao);

  if (!regra || !regra.fatorConversao) {
    return { ...item, produtoVinculado: produto, status: 'sem_conversao', qtdCompra: null, custoEstimado: null };
  }

  const qtdCompra = Math.ceil(item.quantidade / regra.fatorConversao);
  const custoEstimado = qtdCompra * (produto.custo || produto.preco || 0);

  return {
    ...item,
    produtoVinculado: produto,
    sku,
    qtdCompra,
    unidadeCompra: regra.unidadeDestino,
    custoEstimado,
    status: 'convertido',
    overrideManual: false
  };
}

function converterDemanda(pedidoItens) {
  return pedidoItens.map(converterItemDemanda);
}
```

### Inferência automática de fator de conversão

```javascript
function inferirConversao(descricao, produto) {
  // Tenta extrair quantidade da embalagem do nome do produto
  // Ex: "cx c/ 1000fls" → fator 1000, UN→CX
  const matchFls = produto.nomeComercial?.match(/c[\/\s]+(\d+)\s*fls?/i);
  if (matchFls) {
    return { fatorConversao: parseInt(matchFls[1]), unidadeOrigem: 'UN', unidadeDestino: 'CX' };
  }

  // Consultar UNIT_CONVERSIONS existente
  const conv = UNIT_CONVERSIONS[produto.unidadeCompra];
  if (conv) return { fatorConversao: conv.factor, unidadeOrigem: conv.from, unidadeDestino: produto.unidadeCompra };

  return null; // não foi possível inferir
}
```
