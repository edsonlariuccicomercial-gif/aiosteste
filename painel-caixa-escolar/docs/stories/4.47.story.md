## Story

**As a** operador que cadastra produtos na Inteligência do GDP,
**I want** poder selecionar qualquer tipo de unidade (UN, CX, KG, LT, DZ, PCT, MÇ, RS, BD, etc.) como unidade base do produto,
**so that** eu consiga cadastrar produtos vendidos por unidade (abacaxi), por maço (alface), por caixa (leite), por resma (papel), por dúzia (ovos), além dos já existentes por peso (g/kg) e volume (ml/lt).

---

## Contexto / Motivação

O sistema atual só oferece `g` (gramas) e `ml` (mililitros) no select de unidade base do produto (`ei-produto-unidade`). Isso impossibilita o cadastro correto de:

- **Frutas/verduras inteiras** → unidade (UN), maço (MÇ), bandeja (BD)
- **Produtos em caixa** → caixa (CX), fardo (FD)
- **Produtos por contagem** → dúzia (DZ), resma (RS)
- **Produtos por peso maior** → quilograma (KG)
- **Produtos por volume maior** → litro (LT), galão (GL)

O arquivo `server-lib/product-utils.js` já possui um `UNIT_MAP` com 24 unidades normalizadas — o formulário precisa usar esse mapa.

**Impacto na embalagem:** O campo `quantidade_base` da embalagem já é genérico (número). Exemplo:
- Produto: Abacaxi (unidade_base: "UN") → Embalagem: "Caixa c/ 6 abacaxis" (quantidade_base: 6)
- Produto: Arroz (unidade_base: "KG") → Embalagem: "Pacote 5kg" (quantidade_base: 5)
- Produto: Leite (unidade_base: "LT") → Embalagem: "Caixa 1L" (quantidade_base: 1)

**Arquivo principal:** `painel-caixa-escolar/squads/caixa-escolar/dashboard/gdp-contratos.html`

## Acceptance Criteria

### AC1 — Expandir select de unidade base no formulário de produto
- [ ] Substituir o select `ei-produto-unidade` (que só tem g/ml) por um select com as unidades agrupadas por categoria:
  - **Peso:** g (Grama), KG (Quilograma)
  - **Volume:** ml (Mililitro), LT (Litro), GL (Galão)
  - **Contagem:** UN (Unidade), DZ (Dúzia)
  - **Embalagem:** CX (Caixa), PCT (Pacote), FD (Fardo), SC (Saco/Sachê), BD (Bandeja), PT (Pote)
  - **Medida:** MÇ (Maço), RS (Resma), RL (Rolo), FR (Frasco), TB (Tubo), GF (Garrafa), LA (Lata)
- [ ] Cada option deve exibir: `SIGLA — Nome completo` (ex: "UN — Unidade", "CX — Caixa")
- [ ] Default: "UN" (unidade) em vez de "g"

### AC2 — Exibir unidade por extenso na tabela de produtos
- [ ] Na tabela de produtos (`ei-produtos-tbody`), exibir a unidade com label legível
- [ ] Criar ou reusar função `formatUnidadeLabel(sigla)` que retorna "Grama", "Unidade", "Caixa" etc.

### AC3 — Compatibilidade com produtos existentes
- [ ] Produtos já cadastrados com `unidade_base: "g"` ou `"ml"` devem continuar funcionando
- [ ] Nenhuma migração de dados necessária — os valores antigos são válidos

### AC4 — Label dinâmico na embalagem
- [ ] No formulário de embalagem, o label "Qtd base (g ou ml)" deve atualizar para refletir a unidade do produto selecionado
- [ ] Exemplo: se produto é "Abacaxi (UN)", label deve ser "Qtd base (UN)"
- [ ] Exemplo: se produto é "Arroz (KG)", label deve ser "Qtd base (KG)"

### AC5 — Cálculos de estoque/demanda compatíveis
- [ ] A lógica de cálculo em `calcularNecessidadeItem()` e similares deve funcionar com qualquer unidade
- [ ] O campo `quantidade_base` da embalagem já é genérico — garantir que não há hardcode de "g" ou "ml" nos cálculos
