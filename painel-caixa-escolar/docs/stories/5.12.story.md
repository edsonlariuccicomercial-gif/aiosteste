# Story 5.12 — Consolidar sync layers

**Epic:** 5 | **Sprint:** 4 (Consolidação) | **Status:** Draft
**Agente:** @dev | **Débitos:** TD-A10 | **Estimativa:** 8-16h

## Acceptance Criteria

- [ ] AC1: Migrar toda lógica útil de `app-sync.js` para `gdp-api.js`
- [ ] AC2: Deprecar `app-sync.js` (marcar como legacy, remover após validação)
- [ ] AC3: `cloudSave`/`cloudLoad` passam a usar gdp-api internamente
- [ ] AC4: Eliminar sync_data table como intermediário (usar tabelas diretas)
- [ ] AC5: Sync funciona corretamente em todos os cenários (online, offline, multi-browser)

**Nota:** app-sync.js tem 457 linhas com lógica complexa:
- cloudSave/cloudLoad via sync_data table (legacy)
- Dirty tracking com quickHash
- Deleted IDs filtering (10 tabelas)
- Polling cross-machine (6 tabelas REST diretas)
- Visibility change listeners (save on hide, poll on visible)
- Config sync (nexedu.*, logomarca merge)
- AbortController debouncing
Nenhuma dessas features tem equivalente em gdp-api.js.
**Status: Backlog longo prazo** — requer testes e2e antes de consolidar.
