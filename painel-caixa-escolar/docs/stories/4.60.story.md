## Story

**As a** gestor do sistema GDP,
**I want** checkboxes na aba Caixa para excluir lançamentos, remover checkbox dos contratos, nome do cliente clicável nos pedidos, e nome do produto editável na edição em massa,
**so that** a interface seja mais prática e consistente com as demais seções.

---

## Acceptance Criteria

### AC-1 — Checkbox + rodapé de exclusão na aba Caixa

**Arquivo:** `gdp-pedidos.js` — `renderCaixa()` (~linha 1850-1870)
**Arquivo:** `gdp-contratos.html` — seção fin-content-caixa

- [ ] Adicionar checkbox no lado esquerdo de cada lançamento na tabela do extrato conciliado (`caixa-extrato-tbody`)
- [ ] Criar Set `_selectedCaixaIds` para rastrear selecionados
- [ ] Ao selecionar 1+ lançamentos, exibir rodapé (mesmo padrão de CP/CR) com: quantidade selecionada, valor total, botão "Excluir"
- [ ] Botão excluir deve remover os lançamentos selecionados de `loadConciliacao()` e `saveConciliacao()`
- [ ] Atualizar stats do extrato após exclusão com `atualizarExtratoStats()`
- [ ] Rodapé segue padrão visual das abas Contas a Pagar / Contas a Receber

---

### AC-2 — Remover checkbox dos cards de contratos

**Arquivo:** `gdp-contratos-module.js` — `renderCard()` (~linha 2151-2158)

- [ ] Remover `<input type="checkbox" class="contrato-check">` do template do card
- [ ] Remover variável `_selectedContratoIds` (linha ~2089)
- [ ] Remover função `toggleContratoCheck()` (linhas ~2091-2106)
- [ ] Remover variável `isChecked` e estilo condicional de borda/background do card
- [ ] Remover atributo `data-contrato-id` do card (não mais necessário)
- [ ] Badge de status (ativo/encerrado) continua clicável para toggle

---

### AC-3 — Pedidos: nome do cliente em azul clicável (em vez do protocolo)

**Arquivo:** `gdp-pedidos.js` — `renderPedidos()` (~linha 96)

- [ ] Na coluna onde aparece o protocolo em azul (`PED-20260519-09642`), substituir pelo nome do cliente em azul
- [ ] Ao clicar no nome do cliente, abrir o detalhe do pedido (mesma função `verPedidoDetalhe(p.id)`)
- [ ] Manter o protocolo visível em coluna separada com fonte menor e cor discreta (monospace, `var(--dim)`)
- [ ] Protocolo exibido no formato curto `#XXXXX` (últimos 5 dígitos do ID, ex: `PED-20260519-09642` → `#09642`)
- [ ] Tooltip no protocolo curto deve mostrar o ID completo
- [ ] Tooltip no nome do cliente deve mostrar o ID completo do pedido
- [ ] Novos pedidos criados via `genId('PED')` mantêm o formato interno completo — só a EXIBIÇÃO é curta

---

### AC-4 — Central de Produtos: nome do produto editável na edição em massa

**Arquivo:** `gdp-estoque-intel.js` — `editarMassaProdutos()` (~linha 2474)

- [ ] Trocar o `<td>` de texto plain do nome do produto por `<input class="bk-nome" type="text">` editável
- [ ] Estilo consistente com os demais campos editáveis (SKU, NCM, etc.)
- [ ] Na função `aplicarEditarMassaProdutos()` (~linha 2513-2539), extrair valor de `.bk-nome` e atualizar `p.nome` se alterado
- [ ] Salvar produto alterado com a função de save existente
