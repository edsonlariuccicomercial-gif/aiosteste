## Story

**As a** gestor do sistema GDP,
**I want** que os rodapés das páginas reflitam as quantidades e valores dos dados filtrados,
**so that** eu saiba exatamente quantos registros e qual valor total estou visualizando após aplicar filtros.

---

## Contexto / Motivação

Todas as telas com footer (rodapé fixo mostrando quantidade e valor total) calculam os totais com base no **array completo**, ignorando filtros de busca, período, categoria, etc. O usuário filtra por "este mês" e o footer continua mostrando o total geral — confuso e enganoso.

**Causa raiz:** As funções `_updateXxxFooterTotals()` em `gdp-init.js` leem o array completo (ex: `notasFiscais`, `pedidos`) com filtro apenas pelo status tab ativo. Não aplicam os mesmos filtros que o `renderXxx()` aplica (busca, período, categoria).

**Solução:** Cada `renderXxx()` já calcula `filtered` — basta passar esse array filtrado para a função de footer correspondente, ou chamar a footer update com os dados filtrados após o render.

---

## Acceptance Criteria

### AC-1 — NFs: footer reflete filtros de busca e período
- [x] Ao filtrar notas fiscais por busca de texto, o footer (quantidade e valor) deve mostrar apenas os dados filtrados
- [x] Ao filtrar por período (hoje, semana, mês, intervalo), o footer deve refletir o período
- [x] Ao filtrar por status tab + busca + período combinados, o footer deve refletir todos os filtros
- [x] Arquivo: `gdp-pedidos.js` → `renderNotasFiscais()` chama `_updateNfFooterTotals(filtered)`
- [x] Arquivo: `gdp-init.js` → `_updateNfFooterTotals(items)` aceita parâmetro opcional

### AC-2 — Pedidos: footer reflete filtros de busca
- [x] Ao filtrar pedidos por busca de texto, o footer deve mostrar apenas os dados filtrados
- [x] Ao filtrar por status tab + busca combinados, o footer deve refletir ambos
- [x] Arquivo: `gdp-pedidos.js` → `renderPedidos()` chama `_updatePedidosFooterTotals(filtered)`
- [x] Arquivo: `gdp-init.js` → `_updatePedidosFooterTotals(items)` aceita parâmetro opcional

### AC-3 — Contas a Pagar: footer reflete filtros de busca e categoria
- [x] Ao filtrar contas a pagar por busca ou categoria, o footer deve refletir
- [x] Arquivo: `gdp-pedidos.js` → `renderContasPagar()` chama `_updateCpFooterTotals(filtered)`

### AC-4 — Contas a Receber: footer reflete filtros de busca e categoria

### AC-5 — Central de Produtos: footer reflete filtros de busca, categoria e tipo

## Notas Técnicas

### Abordagem recomendada (menor risco)

1. Alterar cada `_updateXxxFooterTotals()` para aceitar parâmetro opcional `items`:
```javascript
// ANTES:
function _updateNfFooterTotals() {
  const filtered = notasFiscais.filter(...); // só status tab
  // ...
}

// DEPOIS:
function _updateNfFooterTotals(items) {
  const filtered = items || notasFiscais; // usa items se passado, senão array completo
  // ...
}
```

2. Em cada `renderXxx()`, após calcular `filtered`, chamar a footer update passando `filtered`:
```javascript
// No final de renderNotasFiscais():
_updateNfFooterTotals(filtered);
```
