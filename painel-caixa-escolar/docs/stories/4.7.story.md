## Story

**As a** fornecedor que utiliza o sistema em múltiplos computadores,
**I want** que os dados sincronizem corretamente entre dispositivos, o status/saldo dos contratos persista, e funcionalidades de catálogo e cadastro ERP sejam aprimoradas,
**so that** não perco alterações ao trocar de computador e consigo gerenciar produtos e pedidos com mais agilidade.

### Root Cause Analysis — Sync Bugs

**Bug #1 (Radar divergente):**
- `caixaescolar.orcamentos` está AUSENTE do `SYNC_KEYS` (app.js:28-32)
- Resultado: dados do radar nunca vão para o cloud, cada PC tem seu próprio snapshot

**Bug #2 (Contrato revertendo + saldo zerado):**
- `syncFromCloud()` compara `row.data.updatedAt` vs `localData.updatedAt`
- `gdp.contratos.v1` salva como array puro `[...]` — array não tem `.updatedAt`
- Ambos timestamps = 0, comparação `0 > 0 = false` → cloud **nunca** sobrescreve local
- Contrato encerrado no PC-A nunca chega ao PC-B
- PC-B mantém dados antigos com `qtdEntregue: 0` → saldo mostra como se nada foi entregue

**Fix definitivo:** Wrapper com `updatedAt` (Task 2) + fallback para `row.updated_at` do Supabase (Task 1)

### Funções existentes reutilizáveis
- `cadastrarTinyItem(contratoId, itemIdx)` — já faz POST ao `/api/tiny-produtos`
- `recalcularSaldoContrato(contratoId)` — fallback manual se sync falhar
- `adicionarItemContrato(contratoId)` — modal de +Item existente
- `saveBancoProdutos()` — salva `gdp.produtos.v1` no localStorage
