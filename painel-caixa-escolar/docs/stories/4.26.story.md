## Story

**As a** operador comercial que analisa preços de múltiplos fornecedores,
**I want** que o sistema agrupe automaticamente produtos iguais com descrições diferentes sob um "Item Mestre" único,
**so that** eu consiga comparar preços reais entre fornecedores sem confusão entre nomes variados do mesmo produto.

---

## Contexto / Motivação

O `bancoPrecos` atual trata cada importação como item independente. O mesmo produto aparece duplicado com nomes diferentes:
- "Papel A4 Chamex 500 folhas"
- "Papel sulfite A4 75g 500fls"
- "RESMA PAPEL A4 CHAMEX"

Isso impede comparação real de preços e infla o banco. O conceito de **Item Mestre** é a fundação para toda a inteligência de preços.

### Estrutura existente que será evoluída

- `bancoPrecos.itens[]` — array flat de itens
- `findBancoItem(nome)` — matching simples por texto normalizado
- `UNIT_CONVERSIONS` — conversão básica de embalagens
- `normalizedText()` — remove acentos e lowercase

## Acceptance Criteria

### AC-1: Estrutura de Item Mestre

- [ ] Cada item do banco pode ter um campo `mesterId` que aponta para o item mestre
- [ ] O item mestre agrega: nome canônico, categoria, unidade base, atributos extraídos
- [ ] Os itens mestres são salvos em `localStorage` com chave `caixaescolar.itens-mestres`
- [ ] Estrutura do item mestre:
  ```
  { id, nomeCanonico, categoria, unidadeBase, atributos: { marca, volume, gramatura, peso },
    aliases: ["nome1", "nome2"], criadoEm, atualizadoEm }
  ```

### AC-2: Normalização automática na importação

- [ ] Ao importar itens (Excel/PDF/CSV), o sistema deve tentar encontrar o item mestre correspondente
- [ ] Matching por: texto normalizado, aliases conhecidos, similaridade (Levenshtein ou token-based)
- [ ] Se match >= 80% de confiança → vincular automaticamente ao item mestre
- [ ] Se match entre 50-80% → sugerir ao usuário (modal de confirmação)
- [ ] Se match < 50% → criar novo item mestre candidato

### AC-3: Extração de atributos

- [ ] Ao criar/vincular item mestre, extrair automaticamente:
  - Marca (ex: "Chamex", "Report", "Neve")
  - Volume/Gramatura (ex: "500ml", "75g", "500fls")
  - Unidade de embalagem (ex: "caixa c/12", "pacote c/10", "fardo c/6")
  - Peso (ex: "1kg", "500g")
- [ ] Usar regex patterns para extrair esses atributos da descrição
- [ ] Calcular **preço unitário equivalente** quando unidade difere

### AC-4: UI de gestão de itens mestres

- [ ] Nova sub-seção ou modal no Banco de Preços para ver/editar itens mestres
- [ ] Mostrar para cada mestre: nome canônico, aliases vinculados, qtd de fornecedores, range de preço
- [ ] Permitir merge manual de dois itens mestres (arrastar/confirmar)
- [ ] Permitir desvincular um alias de um mestre

### AC-5: Preço unitário equivalente

- [ ] Expandir `UNIT_CONVERSIONS` para cobrir mais embalagens comuns em licitação
- [ ] Ao vincular item a um mestre, calcular o preço unitário base
- [ ] Exibir preço unitário na tabela do banco ao lado do preço original
- [ ] Usar preço unitário para comparações entre fornecedores

## Dev Notes

### Regex para extração de atributos

```javascript
function extractAttributes(desc) {
  const norm = desc.toUpperCase();
  return {
    marca: norm.match(/\b(CHAMEX|REPORT|NEVE|DONA BENTA|SADIA|PERDIGAO|TILIBRA|FABER.CASTELL)\b/)?.[0] || "",
    volume: norm.match(/(\d+)\s*(ML|L|LITRO)/)?.[0] || "",
    gramatura: norm.match(/(\d+)\s*(G|GR|KG|GRAMA)/)?.[0] || "",
    folhas: norm.match(/(\d+)\s*(FLS|FOLHAS|FL)/)?.[0] || "",
    peso: norm.match(/(\d+[\.,]?\d*)\s*(KG|G)\b/)?.[0] || "",
  };
}
```

### Algoritmo de similaridade (token-based Jaccard)

```javascript
function calcSimilarity(a, b) {
  const tokensA = new Set(normalizedText(a).split(/\s+/).filter(t => t.length > 2));
  const tokensB = new Set(normalizedText(b).split(/\s+/).filter(t => t.length > 2));
  const intersection = [...tokensA].filter(t => tokensB.has(t)).length;
  const union = new Set([...tokensA, ...tokensB]).size;
  return union === 0 ? 0 : intersection / union;
}
```
