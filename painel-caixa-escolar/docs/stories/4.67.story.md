## Bug Report

| Campo | Detalhe |
|-------|---------|
| **Erro** | `TypeError: extratos.find is not a function` |
| **Local** | `renderConciliacao()` em `gdp-core.js:2164` |
| **Causa raiz** | `loadExtratos()` retornava objeto wrapped `{_v, updatedAt, items}` do cloud sync ao inves de array |
| **Afetados** | Todos os usuarios com sync cloud ativo |
| **Severidade** | Critica ‚Äî aba inteira nao funcionava |

### Analise tecnica

O cloud sync (`syncFromCloud` em `gdp-core.js:333-334`) converte arrays de `gdp.extratos.v1` para formato wrapped `{ _v: 1, updatedAt: "...", items: [...] }` antes de gravar no localStorage.

- `loadConciliacao()` (linha 2031-2038) **tratava** corretamente ambos os formatos (wrapped e array puro)
- `loadExtratos()` (linha 2052-2053) **NAO tratava** o formato wrapped ‚Äî retornava o objeto diretamente
- `saveExtratos()` (linha 2056-2058) salvava como array puro, inconsistente com `saveConciliacao()`

Resultado: todas as operacoes de array (`.find()`, `.length`, `.map()`, `.filter()`) falhavam.

---

## Correcoes Aplicadas

### Fix 1 ‚Äî loadExtratos() (gdp-core.js:2052)
Adicionado unwrap de formato wrapped igual a `loadConciliacao()`:
```javascript
function loadExtratos() {
  try {
    var raw = JSON.parse(localStorage.getItem(EXTRATOS_KEY) || "[]");
    if (raw && raw.items && Array.isArray(raw.items)) return raw.items;
    if (Array.isArray(raw)) return raw;
    return [];
  } catch(_) { return []; }
}
```

### Fix 2 ‚Äî saveExtratos() (gdp-core.js:2056)
Salvar em formato wrapped consistente:
```javascript
function saveExtratos(list) {
  var arr = list || [];
  arr.forEach(function(it) { if (!it.id) it.id = 'ext-' + Date.now() + '-' + Math.random().toString(36).slice(2, 8); });
  var wrapped = { _v: 1, updatedAt: new Date().toISOString(), items: arr };
  localStorage.setItem(EXTRATOS_KEY, JSON.stringify(wrapped));
  if (typeof cloudSave === 'function') cloudSave(EXTRATOS_KEY, wrapped);
}
```
