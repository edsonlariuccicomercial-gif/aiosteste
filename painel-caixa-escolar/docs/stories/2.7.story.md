# Story 2.7 — Modularizacao Frontend (app.js + Layout Compartilhado)

**Epic:** 2 — Resolucao de Debito Tecnico
**Sprint:** 3 (Semana 6-7)
**Prioridade:** P2 — ALTO
**Estimativa:** 52 horas
**Debitos resolvidos:** SYS-1, SYS-3, UX-03, UX-05, UX-06, UX-23

---

## Descricao

Separar o monolito `app.js` (2546 linhas) em modulos ES independentes e criar um layout compartilhado (shell) para todas as paginas, eliminando duplicacao de HTML/CSS/JS. Tambem corrigir a sanitizacao inconsistente de innerHTML para prevenir XSS.

**IMPORTANTE:** Esta story so deve iniciar APOS Story 2.5 (Testes) estar completa — testes e2e sao a rede de seguranca para refatoracao.

---

## Criterios de Aceitacao

- [ ] `app.js` dividido em >= 6 modulos ES, nenhum com mais de 500 linhas
- [ ] Modulos: `state.js`, `render.js`, `sgd.js`, `banco.js`, `preorcamento.js`, `utils.js`
- [ ] Layout compartilhado (`shell.js` ou `<ce-shell>` Web Component) usado por todas as paginas
- [ ] Topbar unificada com navegacao consistente entre paginas
- [ ] Breadcrumbs implementados para navegacao GDP
- [ ] Todos os usos de innerHTML auditados e sanitizados com `escapeHtml()`
- [ ] Nenhum dado externo (SGD, usuario) injetado sem sanitizacao
- [ ] Testes e2e passam APOS modularizacao (zero regressoes)
- [ ] Funcionalidade 100% preservada — nenhuma feature removida ou quebrada

---

## Tarefas

### Modularizacao app.js

- [ ] Criar `dashboard/js/state.js` — gerenciamento de estado (orcamentos, bancoPrecos, preOrcamentos, selectedOrcIds)
- [ ] Criar `dashboard/js/render.js` — funcoes de renderizacao (renderOrcamentos, renderBanco, renderKPIs)
- [ ] Criar `dashboard/js/sgd.js` — integracao SGD (scan, submit, proxy)
- [ ] Criar `dashboard/js/banco.js` — CRUD banco de precos (add, edit, delete, import)
- [ ] Criar `dashboard/js/preorcamento.js` — logica de pre-orcamentos (gerar, aprovar, recusar)
- [ ] Criar `dashboard/js/utils.js` — utilidades (escapeHtml, formatCurrency, debounce, normalize)
- [ ] Criar `dashboard/js/filters.js` — logica de filtros e busca
- [ ] Criar `dashboard/js/app.js` — orquestrador (imports, init, event binding)
- [ ] Atualizar `index.html` para usar `<script type="module" src="js/app.js">`

### Layout Compartilhado

- [ ] Criar `dashboard/js/components/shell.js` — Web Component `<ce-shell>`
- [ ] Shell inclui: topbar, sidebar (mobile), navegacao, area de conteudo
- [ ] Navegacao: Painel, GDP Dashboard, Contratos, Gestao, Portal, Entregador
- [ ] Implementar breadcrumbs na shell
- [ ] Migrar `index.html` para usar `<ce-shell>`
- [ ] Migrar `gdp-dashboard.html` para usar `<ce-shell>`
- [ ] Migrar `gdp-contratos.html` para usar `<ce-shell>`
- [ ] Migrar `gdp-gestao.html` para usar `<ce-shell>`
- [ ] Migrar `gdp-portal.html` para usar `<ce-shell>`
- [ ] Migrar `gdp-entregador.html` para usar `<ce-shell>`
- [ ] Remover topbar duplicado de cada pagina GDP

### Sanitizacao

- [ ] Auditar todos os usos de `innerHTML` em app.js e paginas GDP
- [ ] Garantir que `escapeHtml()` e aplicado em todos os dados externos
- [ ] Criar funcao `safeHTML()` wrapper que sanitiza automaticamente
- [ ] Migrar pontos criticos para DOM API (createElement + textContent)

### Verificacao

- [ ] Executar todos os testes e2e e verificar zero regressoes
- [ ] Testar navegacao entre todas as paginas
- [ ] Testar funcionalidades de cada aba (Orcamentos, Pre-Orc, Banco, SGD)
- [ ] Testar em mobile (responsive)

## Dependencias

- **Story 2.5** — Testes e2e DEVEM estar prontos antes de iniciar modularizacao
- **Story 2.6** — Design system unificado facilita a criacao do shell

## Riscos

- Funcionalidades nao documentadas podem quebrar — testes e2e minimizam esse risco
- `<script type="module">` pode ter issues com browsers antigos — definir browser matrix
- Web Components sem polyfill nao funcionam no IE11 — irrelevante para o publico-alvo

- `squads/caixa-escolar/dashboard/app.js` (refatorar em modulos)
- `squads/caixa-escolar/dashboard/js/` (novo diretorio, 8+ modulos)
- `squads/caixa-escolar/dashboard/js/components/` (novo, Web Components)
- Todas as 7 paginas HTML (usar shell, script modules)
