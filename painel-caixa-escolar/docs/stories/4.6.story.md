## Story

**As a** fornecedor que utiliza o módulo GDP Contratos,
**I want** a navegação renomeada e reordenada com acesso rápido ao catálogo escolar e importação de clientes,
**so that** consigo gerenciar clientes, acessar catálogos para pedidos internos e importar dados em massa com mais eficiência.

---

## Acceptance Criteria

### AC-1: Renomear abas
- [ ] Aba "Usuarios" → **"Clientes"**
- [ ] Aba "Banco de Produtos" → **"Banco de Produtos ERP"**
- [ ] Todas as referências internas (labels, mensagens, toasts) devem refletir os novos nomes

### AC-2: Reordenar abas
- [ ] Nova ordem das tabs: **Clientes → Banco de Produtos ERP → Contratos → Itens do Contrato → Pedidos → Entregas**
- [ ] Tab "Contratos" permanece como aba ativa por padrão (manter comportamento atual)

### AC-3: Remover botão "Enviar Todos ao Tiny" dos detalhes do contrato
- [ ] Remover o botão `Enviar Todos ao Tiny` da barra de ações dos itens do contrato (função `abrirContrato`, ~linha 3152)
- [ ] Manter os demais botões (Editar Selecionados, Sync Banco, + Item)

### AC-4: Botão "Catálogo Escolar" nos detalhes do contrato
- [ ] Adicionar botão **"📋 Catálogo Escolar"** na barra de ações dos itens do contrato (posição onde estava o Tiny), estilo `btn btn-sm` com fundo `rgba(139,92,246,.15)` e cor `var(--purple)`
- [ ] O botão deve abrir `gdp-portal.html` em nova aba, passando parâmetro para ir direto ao **catálogo de produtos** (tab catalogo)
- [ ] Mecanismo de acesso: gravar dados da escola em `sessionStorage` com chave `gdp-portal-autoLogin` (JSON com id, nome, login, senha da escola). O `gdp-portal.html` lê esse valor no `DOMContentLoaded`, faz auto-login e redireciona para a tab catálogo. Não usar query params com credenciais na URL
- [ ] **Múltiplas escolas vinculadas:** se o contrato tiver mais de uma escola, exibir dropdown/select para o usuário escolher qual escola abrir o catálogo. Se houver apenas uma, abrir diretamente
- [ ] Se nenhuma escola estiver vinculada ao contrato, mostrar toast: "Vincule uma escola ao contrato primeiro (aba Clientes)"
- [ ] O portal deve abrir **apenas no catálogo** — finalidade: realizar pedidos pela escola internamente

### AC-5: Botão de importação de clientes na aba Clientes
- [ ] Adicionar botão **"📥 Importar Clientes"** ao lado do botão "+ Cadastrar Escola" na aba Clientes (ex-Usuarios)
- [ ] Aceitar arquivos **PDF** e **Excel** (.xlsx, .xls, .csv)
- [ ] Parsear dados do arquivo (nome, CNPJ, município, SRE, responsável, email, telefone)
- [ ] Exibir modal de preview com tabela dos dados parseados antes de confirmar importação
- [ ] Inserir clientes importados na lista sem duplicar (verificar por CNPJ)
- [ ] **Dados incompletos:** linhas com campos obrigatórios faltando (nome ou CNPJ) são destacadas em amarelo no preview com aviso — o usuário decide se importa ou descarta. Linhas sem CNPJ NÃO são importadas automaticamente (checkbox para forçar)
- [ ] **Validação de arquivo:** aceitar apenas extensões permitidas, tamanho máximo 10MB, mostrar toast de erro para formatos inválidos
