# Story 2.8 — Autenticacao Segura com Supabase

**Epic:** 2 — Resolucao de Debito Tecnico
**Sprint:** 4 (Semana 8-9)
**Prioridade:** P2 — ALTO
**Estimativa:** 24 horas
**Debitos resolvidos:** SEC-1, UX-01, UX-12, UX-13, SEC-4 (parcial)

---

## Descricao

Substituir a autenticacao client-side (SHA-256 hardcoded) por autenticacao real usando Supabase Auth. Implementar 4 roles (fornecedor, admin, escola, entregador) com login seguro, sessions server-side, e IndexedDB como fallback para localStorage. O login atual sera desativado e substituido pelo fluxo Supabase.

---

## Criterios de Aceitacao

- [ ] Conta Supabase criada e configurada para o projeto
- [ ] Login de fornecedor via Supabase Auth (email + senha)
- [ ] Login de admin via Supabase Auth (email + senha, role admin)
- [ ] Login de escola via Supabase Auth (email + senha, role escola)
- [ ] Login de entregador via Supabase Auth (codigo de acesso ou email)
- [ ] Sessions gerenciadas pelo Supabase (tokens JWT, refresh automatico)
- [ ] Credenciais SGD armazenadas no Supabase (encrypted, nao em localStorage)
- [ ] `auth.js` antigo (SHA-256) removido ou desativado
- [ ] IndexedDB implementado como fallback para dados de sessao
- [ ] Dados de pre-orcamentos e banco de precos persistidos em IndexedDB (alem de localStorage)
- [ ] Row Level Security (RLS) configurada no Supabase para isolar dados por role
- [ ] Testes de autenticacao para cada role (login, logout, session expired, unauthorized)
- [ ] Fluxo de "esqueci senha" funcional

## Tarefas

### Setup Supabase

- [ ] Criar projeto no Supabase (free tier)
- [ ] Configurar autenticacao (email provider)
- [ ] Criar tabela `profiles` (id, role, nome, escola_id)
- [ ] Criar tabela `credentials` (id, user_id, sgd_cnpj, sgd_password_encrypted)
- [ ] Configurar RLS policies para cada tabela
- [ ] Adicionar SUPABASE_URL e SUPABASE_ANON_KEY no .env

### Frontend Auth

- [ ] Instalar Supabase JS client: CDN `@supabase/supabase-js`
- [ ] Criar `dashboard/js/auth-supabase.js` — wrapper de autenticacao
- [ ] Implementar login fornecedor (email + senha)
- [ ] Implementar login escola (email + senha, redireciona para GDP Portal)
- [ ] Implementar login entregador (codigo de acesso)
- [ ] Implementar login admin
- [ ] Implementar logout com limpeza de sessao
- [ ] Implementar guard de rota (redirecionar para login se nao autenticado)
- [ ] Atualizar `login.html` para usar formulario Supabase
- [ ] Remover ou desativar `auth.js` (SHA-256)
- [ ] Implementar "Esqueci minha senha" com Supabase

### IndexedDB Fallback

- [ ] Criar `dashboard/js/storage.js` — camada de persistencia
- [ ] Implementar `storage.get(key)` — tenta IndexedDB, fallback localStorage
- [ ] Implementar `storage.set(key, value)` — salva em ambos
- [ ] Migrar pre-orcamentos para usar `storage.js`
- [ ] Migrar banco de precos para usar `storage.js`
- [ ] Migrar dados de sessao para usar `storage.js`

### Migracao de Usuarios

- [ ] Criar script de migracao para criar usuarios Supabase
- [ ] Migrar fornecedor (1 usuario)
- [ ] Migrar escolas (5 usuarios)
- [ ] Migrar admin (1 usuario)
- [ ] Documentar novas credenciais e comunicar via canal seguro
