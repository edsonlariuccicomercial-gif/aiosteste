# Story 3.1 — Sidebar de Navegacao e Layout Principal NexEdu

**Epic:** 3 — NexEdu Layout e Navegacao
**Prioridade:** Alta
**Estimativa:** 24 horas
**Status:** InProgress

---

## Descricao

Reestruturar o layout de `index.html` de uma navegacao plana (4 tabs horizontais) para um layout com **sidebar fixa a esquerda** contendo 3 modulos hierarquicos e uma **area principal a direita** com dashboard + conteudo do modulo ativo. O sistema passa a se chamar **NexEdu**.

### Estado Atual

- Topbar com identidade do fornecedor (pills SRE, empresa)
- 5 KPIs horizontais
- Painel de inteligencia (collapsible)
- 4 tabs horizontais: Orcamentos | Pre-Orcamento | Banco de Precos | SGD
- GDP em paginas HTML separadas (gdp-dashboard.html, gdp-portal.html, etc.)
- Sem sidebar, sem hierarquia de modulos

### Hierarquia de Modulos

**Modulo Radar:**
- Orcamentos (tab unica — conteudo atual de `#tab-orcamentos`)

**Modulo Inteligencia de Precos:**
- Pre-Orcamento (conteudo atual de `#tab-pre-orcamento`)
- Banco de Precos (conteudo atual de `#tab-banco-precos`)
- SGD (conteudo atual de `#tab-sgd`)

**Modulo GDP:**
- Manter estrutura atual de paginas separadas
- Ao clicar em GDP no sidebar, navegar para `gdp-dashboard.html`

**Configuracoes (engrenagem):**
- Dados da Empresa (nome, CNPJ, SRE, contato)
- Cadastro de Usuarios (lista de usuarios com acesso ao sistema)

## Criterios de Aceitacao

- [ ] Sidebar fixa a esquerda com largura ~220px
- [ ] Logo "NexEdu" no topo da sidebar (fixa, estilo logomarca)
- [ ] 3 modulos na sidebar: Radar, Inteligencia de Precos, GDP
- [ ] Modulo ativo destacado visualmente (accent color)
- [ ] Icone de engrenagem abaixo dos modulos para Configuracoes
- [ ] Ao clicar em "Radar": exibe aba Orcamentos na area principal
- [ ] Ao clicar em "Inteligencia de Precos": exibe sub-abas Pre-Orcamento, Banco de Precos, SGD
- [ ] Ao clicar em "GDP": navega para `gdp-dashboard.html`
- [ ] Dashboard panel (KPIs + inteligencia) visivel acima do conteudo do modulo ativo
- [ ] Configuracoes abre painel com: Dados da Empresa e Cadastro de Usuarios
- [ ] Topbar permanece no topo (info do fornecedor, botoes de acao)
- [ ] Layout responsivo: sidebar collapsa em telas < 768px (hamburger menu)
- [ ] Tema escuro mantido (variaveis CSS atuais)
- [ ] Transicao suave ao trocar modulo/aba
- [ ] Tab navigation horizontal aparece apenas dentro do modulo Inteligencia de Precos
- [ ] Estado do modulo ativo persiste em `localStorage`

### Decisoes Tecnicas

- **Sidebar position: fixed** — fica visivel durante scroll
- **Modulo GDP como link externo** — manter paginas separadas (nao integrar em SPA)
- **Configuracoes em localStorage** — sem backend por enquanto (pre-Supabase)
- **Logo NexEdu** — texto estilizado por enquanto (sem arquivo de imagem)

### Restricoes

- NAO alterar logica de negocio (renderOrcamentos, renderPreOrcamentos, etc.)
- NAO alterar integracao SGD
- NAO alterar sistema de autenticacao (sera feito na Story 2.8)
