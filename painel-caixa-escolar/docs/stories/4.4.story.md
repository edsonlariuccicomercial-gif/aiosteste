## Descricao

Adicionar selecao multipla de protocolos na aba Pedidos do GDP Contratos e botao para gerar uma Lista de Compras consolidada. Quando multiplos protocolos sao selecionados, produtos iguais (mesmo SKU ou descricao) devem ter suas quantidades somadas. A lista gerada deve ser exportavel em CSV e PDF.

### Algoritmo de Consolidacao

```
1. Para cada protocolo selecionado:
   a. Coletar todos os itens (descricao, SKU, unidade, quantidade)
2. Criar mapa de consolidacao: chave = SKU (preferencia) ou descricao.toLowerCase().trim()
3. Para cada item:
   a. Se chave existe no mapa → somar quantidade, adicionar protocolo a lista de origem
   b. Se chave nao existe → criar entrada nova
4. Ordenar resultado por descricao alfabetica
5. Retornar array consolidado
```

### Restricoes

- NAO alterar os pedidos originais (lista e somente leitura/export)
- NAO persistir a lista de compras (gerada sob demanda)
- Manter compatibilidade com todas as funcionalidades existentes da aba Pedidos

---

## Dependencias

- Nenhuma (usa dados existentes de `gdp.pedidos.v1`)

## Riscos

- Produtos com descricoes ligeiramente diferentes (ex: "ARROZ 5KG" vs "Arroz 5kg") podem nao consolidar — mitigado com case-insensitive + trim
- jsPDF ou html2pdf pode precisar ser adicionado via CDN se nao disponivel
