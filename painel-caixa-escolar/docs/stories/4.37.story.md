## Story

**As a** operador que gerencia estoque e compras,
**I want** que as demandas convertidas alimentem automaticamente o estoque (baixa) e gerem lista de compras (reposição),
**so that** eu tenha controle do que tenho, do que precisa comprar e de quanto vai custar.

---

## Contexto / Motivação

O fluxo pós-licitação da Caixa Escolar é:

```
Contrato → Pedido (da escola) → Nota Fiscal → Demanda → Estoque / Compras
```

Esta story é a etapa final do fluxo. Quando uma demanda é confirmada (Story 4.36), o sistema precisa:

1. **Baixar do estoque** os itens que já estão disponíveis
2. **Gerar lista de compras** para o que falta (itens sem estoque ou estoque insuficiente)
3. **Agrupar por fornecedor** para facilitar a emissão de pedidos de compra
4. **Exportar** para CSV/impressão para enviar ao fornecedor

O estoque é por produto real (SKU), não por descrição de contrato. A conversão feita na Story 4.36 já resolveu a tradução de descrição → SKU → quantidade real de compra.

### Estrutura existente relevante

- `gdp.demandas.v1` — demandas com itens convertidos (Story 4.36)
- `gdp.equivalencias.v1` — vínculo descrição → SKU (Story 4.35)
- `bancoPrecos.itens[]` — produtos com `sku`, `fornecedorPadrao`, `custo`, `unidadeCompra`
- Função `confirmarDemanda(demandaId)` — muda status para `"confirmada"` (Story 4.36)

## Acceptance Criteria

### AC-1: Estrutura de estoque — `gdp.estoque.v1`

- [ ] Estoque é salvo em `localStorage` com chave `gdp.estoque.v1` como array de entradas por produto:
  ```
  [{ sku, nomeComercial, unidadeCompra, qtdDisponivel, qtdComprometida, ultimaAtualizacao }]
  ```
- [ ] `qtdDisponivel` = quantidade física em estoque
- [ ] `qtdComprometida` = quantidade reservada por demandas confirmadas mas ainda não atendidas
- [ ] `qtdLivre = qtdDisponivel - qtdComprometida` — quantidade efetivamente disponível para novas demandas
- [ ] Entradas de estoque podem ser criadas manualmente (lançamento de entrada) ou automaticamente ao receber compra

### AC-2: Baixa de estoque ao confirmar demanda

- [ ] Ao chamar `confirmarDemanda(demandaId)`, o sistema processa todos os itens convertidos da demanda:
  - Para cada item com status `"convertido"`:
    - Busca entrada de estoque pelo SKU
    - Se `qtdLivre >= item.qtdCompra`: debita de `qtdDisponivel` e registra movimentação de saída
    - Se `qtdLivre < item.qtdCompra`: debita o disponível e o restante vai para lista de compras
    - Se sem estoque: item vai inteiramente para lista de compras
  - Itens com status diferente de `"convertido"` são ignorados pelo estoque (não bloqueiam a confirmação)
- [ ] Após processamento, a demanda recebe campo `estoqueProcessado: true`

### AC-3: Lista de compras gerada automaticamente

- [ ] Ao confirmar uma demanda, os itens com quantidade insuficiente em estoque são adicionados à lista de compras em `gdp.lista-compras.v1`
- [ ] Cada entrada na lista de compras:
  ```
  { id, sku, nomeComercial, qtdNecessaria, unidadeCompra, fornecedor, custoUnitario, custoTotal,
    demandaOrigemId, contratoId, status: "pendente" | "pedido_emitido" | "recebido" }
  ```
- [ ] Itens do mesmo SKU de demandas diferentes são consolidados: a lista mostra a quantidade total acumulada por SKU
- [ ] Lista de compras persiste entre sessões

### AC-4: View de estoque

- [ ] Nova seção "Estoque" acessível pelo menu principal (tab ou link)
- [ ] Tabela com colunas: Produto (nomeComercial), SKU, Unidade, Disponível, Comprometido, Livre, Última Atualização
- [ ] Linha com `qtdLivre <= 0` destacada em vermelho
- [ ] Linha com `qtdLivre > 0 && qtdLivre <= 5` destacada em amarelo (estoque baixo)
- [ ] Botão "Lançar Entrada" por produto: abre mini-modal para adicionar quantidade ao estoque
- [ ] Botão "Ver Movimentações" por produto: exibe histórico de entradas/saídas

### AC-5: View da lista de compras

- [ ] Nova seção "Lista de Compras" acessível pelo menu principal
- [ ] Tabela agrupada por fornecedor: cada grupo mostra os itens daquele fornecedor com subtotal
- [ ] Colunas: Produto, Qtd Necessária, Unidade, Custo Unit., Custo Total, Origem (demanda/contrato), Status
- [ ] Total geral de custo ao final da tabela
- [ ] Botão "Marcar como Pedido Emitido" por item ou por fornecedor (em grupo)
- [ ] Botão "Exportar CSV" e "Imprimir" na seção

### AC-6: Exportação da lista de compras

- [ ] Botão "Exportar CSV" gera arquivo `lista-compras-{data}.csv` com todas as colunas da tabela
- [ ] CSV inclui cabeçalho: Fornecedor, Produto, SKU, Qtd, Unidade, Custo Unit, Custo Total, Contrato
- [ ] Itens agrupados por fornecedor no CSV (mesma ordem da view)
- [ ] Botão "Imprimir" abre janela de impressão com layout limpo (sem menus/nav)

### AC-7: Histórico de movimentações de estoque

- [ ] Cada entrada e saída de estoque gera um registro em `gdp.movimentacoes-estoque.v1`:
  ```
  { id, sku, tipo: "entrada"|"saida", quantidade, motivo, demandaId, dataCriacao }
  ```
- [ ] `motivo` pode ser: `"confirmacao_demanda"`, `"entrada_manual"`, `"ajuste"`, `"recebimento_compra"`
- [ ] View de movimentações mostra histórico cronológico por produto

### AC-8: Sem regressão

- [ ] Demandas com status `"rascunho"` não afetam o estoque — só demandas `"confirmadas"` são processadas
- [ ] Produtos sem entrada de estoque (`qtdDisponivel = 0`) são tratados como estoque zero — não geram erro
- [ ] `gdp.pedidos.v1` e `gdp.contratos.v1` existentes não são modificados por esta story

### Engine de baixa de estoque

```javascript
function processarBaixaEstoque(demanda) {
  const itensBaixados = [];
  const itensParaCompra = [];

  const itenConvertidos = demanda.itens.filter(i => i.status === 'convertido');

  for (const item of itenConvertidos) {
    const estoqueItem = getEstoqueItem(item.sku);
    const disponivel = calcQtdLivre(estoqueItem);
    const necessario = item.qtdCompra;

    if (disponivel >= necessario) {
      // estoque suficiente: debitar tudo
      upsertEstoqueItem(item.sku, {
        qtdDisponivel: estoqueItem.qtdDisponivel - necessario
      });
      addMovimentacao(item.sku, 'saida', necessario, 'confirmacao_demanda', demanda.id);
      itensBaixados.push({ ...item, qtdDebitada: necessario, qtdCompra: 0 });
    } else {
      // estoque parcial ou zero
      const debitado = disponivel;
      const aComprar = necessario - debitado;

      if (debitado > 0) {
        upsertEstoqueItem(item.sku, { qtdDisponivel: 0 });
        addMovimentacao(item.sku, 'saida', debitado, 'confirmacao_demanda', demanda.id);
      }
      itensParaCompra.push({ ...item, qtdDebitada: debitado, qtdCompra: aComprar });
    }
  }

  return { itensBaixados, itensParaCompra };
}
```

### Exportação CSV com UTF-8 BOM

```javascript
function exportarListaComprasCSV() {
  const lista = loadListaCompras();
  const grupos = agruparListaPorFornecedor(lista);

  const linhas = [
    ['Fornecedor', 'Produto', 'SKU', 'Qtd', 'Unidade', 'Custo Unit.', 'Custo Total', 'Contrato', 'Status']
  ];

  for (const [fornecedor, itens] of Object.entries(grupos)) {
    for (const item of itens) {
      linhas.push([
        fornecedor,
        item.nomeComercial,
        item.sku,
        item.qtdNecessaria,
        item.unidadeCompra,
        item.custoUnitario.toFixed(2).replace('.', ','),
        item.custoTotal.toFixed(2).replace('.', ','),
        item.contratoId || '',
        item.status
      ]);
    }
  }

  const csv = '\uFEFF' + linhas.map(l => l.map(c => `"${c}"`).join(';')).join('\n');
  const blob = new Blob([csv], { type: 'text/csv;charset=utf-8;' });
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = `lista-compras-${new Date().toISOString().split('T')[0]}.csv`;
  a.click();
  URL.revokeObjectURL(url);
}
```
