## Story

**As a** operador que sincroniza itens de contratos com o banco de produtos,
**I want** que as alterações feitas no sync banco persistam após recarregar a página,
**so that** eu não perca o trabalho de vincular SKU/NCM/unidade toda vez que atualizo o browser.

---

## Contexto / Motivação

O operador reporta que ao fazer sync banco (vincular produtos), as alterações aparecem corretamente na tela, mas ao recarregar a página (F5) os dados voltam ao estado anterior. O problema é uma race condition entre `syncFromCloud()` (que puxa dados do Supabase) e `localStorage` (que tem os dados corretos).

### Causa raiz identificada

1. `syncFromCloud()` na inicialização puxa dados do Supabase que podem ser mais antigos
2. A lógica de comparação de timestamps tem edge cases onde o cloud vence indevidamente
3. `syncToCloud()` pode não completar antes do usuário recarregar a página
4. Há um `ReferenceError: _selectedPedidoIds` que interrompe parte da inicialização
5. Os 6 warnings "Cloud save failed" indicam que o `cloudSave()` está falhando silenciosamente

### Tentativas anteriores (parcialmente eficazes)

- Gravar direto no localStorage antes do cloud sync ✅
- Cloud sync bloqueante (await) ✅
- Verificação pós-sync ✅
- Cloud só vence se timestamps > 0 ✅
- `var` ao invés de `let` para bancoProdutos ✅
- Render antes do cloud sync ✅

Mesmo assim, o operador reporta que "não funciona" no browser real.

## Acceptance Criteria

### AC-1: Persistência garantida

- [ ] Após aplicar sync banco e recarregar a página (F5), os SKUs/NCMs/unidades salvos devem permanecer
- [ ] Testar em Chrome, Edge e Firefox
- [ ] Testar com e sem conexão ao Supabase

### AC-2: Cloud sync robusto

- [ ] Eliminar os 6 warnings "Cloud save failed" — investigar e corrigir a causa (TypeError na linha 837)
- [ ] Se Supabase estiver indisponível, dados locais devem ser preservados sem interferência
- [ ] Ao reconectar, dados locais devem subir para o cloud (não o contrário)

### AC-3: Inicialização sem erros

- [ ] Eliminar `ReferenceError: _selectedPedidoIds` na inicialização
- [ ] Eliminar todas as variáveis com `let` que sofrem temporal dead zone no initGDP
- [ ] Zero erros no console ao carregar a página

### AC-4: Fluxo de sync banco funcional end-to-end

- [ ] Abrir contrato → Sync Banco → Trocar match via dropdown ou busca → Aplicar → F5 → Dados persistem
- [ ] Testar com contrato CTR-20260313-3740 (16 itens) como caso de referência
