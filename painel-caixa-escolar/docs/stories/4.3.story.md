# Story 4.3 — Sincronizacao Itens do Contrato com Banco de Produtos

**Epic:** 4 — Evolucao Licit-IAX: Dashboard Unificado, Banco de Produtos e Lista de Compras
**Prioridade:** Alta
**Estimativa:** 6 horas
**Status:** Done

---

## Descricao

Adicionar botao de sincronizacao nos detalhes do contrato (GDP Contratos) que permite vincular os itens do contrato aos produtos do Banco de Produtos ERP (Story 4.2). A sincronizacao atualiza apenas descricao, SKU e unidade dos itens — preco e quantidade permanecem os valores originais do contrato.

---

## Criterios de Aceitacao

### Botao de Sincronizacao

- [ ] Botao "Sincronizar com Banco de Produtos" visivel na area de itens dentro dos detalhes do contrato
- [ ] Botao com icone de sync (🔄) e tooltip explicativo
- [ ] Botao desabilitado se Banco de Produtos estiver vazio (com mensagem informativa)

### Processo de Sincronizacao

- [ ] Ao clicar, exibir modal com preview da sincronizacao
- [ ] Modal mostra tabela comparativa: Item Atual → Item do Banco (matched)
- [ ] Matching por similaridade de descricao (fuzzy match) OU por SKU exato
- [ ] Itens sem match destacados em amarelo com opcao de vincular manualmente (select/dropdown)
- [ ] Itens ja sincronizados destacados em verde

### Regras de Sincronizacao

- [ ] Campos sincronizados: descricao, SKU (codigo), unidade
- [ ] Campos PRESERVADOS (nao alterar): preco unitario, quantidade, valor total
- [ ] Se item do contrato tem SKU e existe no banco com mesmo SKU → match automatico
- [ ] Se item nao tem SKU → match por similaridade de descricao (score >= 70%)
- [ ] Usuario pode aceitar/rejeitar cada match individualmente antes de confirmar

### Confirmacao e Persistencia

- [ ] Botao "Confirmar Sincronizacao" aplica as mudancas selecionadas
- [ ] Itens do contrato atualizados no localStorage (`gdp.contratos.v1`)
- [ ] Registro de sincronizacao salvo (data, qtd itens sincronizados)
- [ ] Sync automatico com Supabase apos alteracao

### Feedback Visual

- [ ] Toast/notificacao com resultado: "X de Y itens sincronizados"
- [ ] Itens sincronizados marcados com badge/icone na tabela de itens do contrato
- [ ] Se nenhum match encontrado, mensagem clara ao usuario

### Algoritmo de Matching

```
1. Para cada item do contrato:
   a. Se tem SKU → buscar match exato por SKU no banco
   b. Se nao tem SKU ou nao achou → buscar por similaridade de descricao
   c. Se similaridade >= 70% → sugerir match (amarelo)
   d. Se similaridade < 70% → sem match (vermelho), permitir vincular manual
2. Itens ja sincronizados anteriormente → manter match (verde)
```

### Restricoes

- NUNCA alterar preco ou quantidade dos itens do contrato
- Sincronizacao e sempre opt-in (usuario confirma cada match)
- Banco de Produtos deve existir (Story 4.2) antes desta feature funcionar

---

## Dependencias

- **Story 4.2** — Banco de Produtos ERP deve estar implementado (fonte de dados)

## Riscos

- Fuzzy matching pode gerar falsos positivos em descricoes similares
- Performance se contrato tiver muitos itens (>500) — considerar paginacao no modal
