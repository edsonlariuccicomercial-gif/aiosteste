## Story

**As a** fornecedor que envia propostas ao SGD,
**I want** baixar uma versão em PDF da proposta diretamente na aba "Envio SGD",
**so that** eu tenha um documento formal e imprimível da proposta para arquivo, envio ao cliente ou conferência antes do envio ao sistema.

---

## Contexto / Motivação

Hoje a aba "Envio SGD" permite apenas:
- **Enviar** a proposta via API ao SGD
- **Baixar Payload** (JSON técnico, não legível para o cliente)
- **Imprimir** (abre popup do browser, sem controle de layout)

O fornecedor precisa de um PDF formatado da proposta contendo: dados da escola, itens cotados, preços, totais, observações e dados do fornecedor — pronto para arquivo ou envio ao comprador.

**Solução:** Usar a biblioteca **jsPDF + jsPDF-AutoTable** (via CDN, client-side) para gerar o PDF diretamente no browser, sem backend.

**Arquivo principal:** `painel-caixa-escolar/squads/caixa-escolar/dashboard/app-sgd-integration.js`

## Technical Notes

### Função principal de geração

```javascript
function gerarPdfProposta(orcId) {
  const pre = preOrcamentos[orcId];
  if (!pre) return;

  const { jsPDF } = window.jspdf;
  const doc = new jsPDF();

  // Cabeçalho
  doc.setFontSize(16);
  doc.text("Proposta Comercial", 105, 20, { align: "center" });

  // Dados escola + itens via autoTable
  doc.autoTable({
    head: [["#", "Descrição", "Marca", "Qtd", "Preço Unit.", "Preço Total"]],
    body: pre.itens.map((item, i) => [
      i + 1,
      item.nome,
      item.marca || "—",
      item.quantidade,
      brl.format(item.precoUnitario),
      brl.format(item.precoTotal),
    ]),
    startY: 60,
  });

  doc.save(`proposta-${orcId}.pdf`);
}
```

### Integração no renderSgd()

Adicionar na seção `actions` de cada status:
```javascript
`<button class="btn btn-inline" onclick="gerarPdfProposta('${p.orcamentoId}')">PDF</button>`
```
