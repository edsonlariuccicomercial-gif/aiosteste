## Story

**As a** gestor do sistema GDP,
**I want** que o saldo inicial das contas bancarias aceite valores no formato brasileiro (virgula como separador decimal),
**so that** o valor digitado seja armazenado e exibido corretamente sem perda de dados financeiros.

---

## Acceptance Criteria

- [ ] **AC1:** Ao digitar `8822,88` no campo saldo inicial, o valor armazenado deve ser `8822.88` (numero correto)
- [ ] **AC2:** Ao editar uma conta existente, o campo saldo inicial deve exibir o valor formatado com virgula (ex: `8822,88`)
- [ ] **AC3:** Valores com ponto (formato EN) tambem devem funcionar normalmente (ex: `8822.88`)
- [ ] **AC4:** Campo vazio ou invalido deve resultar em `0`

---

## Root Cause (Analise @analyst)

`parseFloat()` nativo do JavaScript nao entende formato brasileiro. Ao receber `"8822,88"`, para na virgula e retorna apenas `8822` ou valor incorreto dependendo do locale do browser.

---

## Technical Tasks

- [x] **T1:** Corrigir parsing do saldo inicial em `app-config.js` linha 689 — `.replace(",", ".")` antes do `parseFloat()`
- [x] **T2:** Corrigir exibicao do saldo ao editar conta em `app-config.js` linha 786 — formatar com virgula
- [x] **T2b:** Corrigir exibicao do saldo na listagem em `app-config.js` linha 756 — `.replace(".", ",")` no toFixed
- [ ] **T3:** Testar com valores: `8822,88` / `8822.88` / `0` / vazio / `1.234,56`
