## Root Cause (Analise @analyst)

### Bug 1 — Saldo corrupto
Valor 8.82 foi salvo no Supabase ANTES do fix da Story 4.63. syncFromCloud() sobrescreve local com dado corrupto. Fix do codigo ja aplicado (Story 4.63), dado no Supabase verificado e confirmado correto (8822.88).

### Bug 2 — Exclusoes reaparecem (PRINCIPAL)
**4 problemas combinados causavam um ciclo infinito:**

1. `gdp.conciliacao.v1` em `SHARED_SYNC_KEYS` (app-state.js:643) — forca cloud overwrite via `isSharedKey`
2. Protecao de merge so ativa quando local > cloud (app-sync.js:144-147) — delete faz local < cloud, protecao nao ativa
3. `DELETED_KEY_MAP` (gdp-api.js:138-153) nao tem entrada para conciliacao — sem tracking de deletes
4. **CICLO SYNC:** Browser com dados antigos (array puro) fazia `syncToCloud` → repolui Supabase → outro browser puxa dados antigos → exclusoes revertidas infinitamente

### Diagrama do ciclo (ANTES do fix)
```
Angela exclui 10 itens → saveConciliacao([]) → localStorage limpo
→ 30s: syncToCloud envia [] para Supabase ✓
→ Browser Edson abre → localStorage tem 10 itens (array puro antigo)
→ syncToCloud envia 10 itens para Supabase ✗ (repolui!)
→ Angela faz sync → Supabase tem 10 itens → restaura no localStorage ✗
→ Ciclo infinito
```

### Diagrama do ciclo (DEPOIS do fix)
```
Angela exclui itens → saveConciliacao({_v:1, updatedAt:now, items:[]})
→ syncToCloud envia formato wrapped ✓
→ Browser Edson abre → localStorage tem array puro antigo
→ syncToCloud BLOQUEIA array puro (nao envia) ✓
→ syncFromCloud puxa wrapped vazio do cloud → converte para wrapped ✓
→ Exclusao permanente ✓
```

### Invariantes criticas (NUNCA remover/alterar sem nova story)

1. **`syncToCloud` DEVE bloquear arrays puros para conciliacao/extratos**
   - Arquivo: `app-sync.js` linhas 222-224
   - Arquivo: `js/gdp-core.js` linhas 393-395
   - Razao: arrays puros causam ciclo de sync que reverte exclusoes

2. **`syncFromCloud` DEVE converter arrays puros para formato wrapped**
   - Arquivo: `app-sync.js` linhas 186-188
   - Razao: garante que localStorage sempre tem updatedAt para comparacao

3. **Conciliacao DEVE usar timestamp (nao isSharedKey) no sync**
   - Arquivo: `app-sync.js` linhas 150-154
   - Razao: isSharedKey ignora timestamps e sempre sobrescreve local com cloud

4. **`saveConciliacao()` DEVE gerar IDs nos itens e usar formato wrapped**
   - Arquivo: `js/gdp-core.js` linhas 1991-1997
   - Razao: IDs necessarios para delete tracking; wrapped necessario para updatedAt

5. **`excluirCaixaLancamentos()` DEVE rastrear IDs deletados**
   - Arquivo: `js/gdp-pedidos.js` linhas 1934-1958
   - Razao: IDs rastreados em `gdp.conciliacao.deleted.v1` para filtrar no sync

6. **`DELETED_KEY_MAP` DEVE incluir conciliacao**
   - Arquivo: `gdp-api.js` linha 143
   - Arquivo: `app-sync.js` linha 168
   - Razao: sem mapeamento, sync restaura itens deletados

7. **Forcar Sync DEVE limpar `gdp.conciliacao.deleted.v1`**
   - Arquivo: `js/gdp-core.js` linha 430
   - Razao: forcar sync e reset total, deleted IDs devem ser limpos
