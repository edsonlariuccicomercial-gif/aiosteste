## Story

**As a** operador que precisa encontrar rapidamente quem está cotando um produto específico (café, polpa de fruta, etc),
**I want** poder buscar por nome de item em todas as abas e ver o resumo dos itens como informação principal no lugar do objeto,
**so that** eu identifique oportunidades por produto sem precisar abrir cada orçamento.

---

## Contexto / Motivação

O campo "Objeto" do SGD é genérico e longo (ex: "Aquisição de gêneros alimentícios para merenda escolar..."). O que o operador realmente precisa ver são os ITENS (café, açúcar, polpa de fruta, papel toalha).

Situação atual:
- Coluna Objeto no Radar exibe o texto longo do objeto como informação principal
- O resumo dos itens (implementado anteriormente) aparece como informação secundária, abaixo do objeto
- A busca por texto já pesquisa parcialmente em itens, mas precisa ser verificada e garantida
- A aba Envio SGD já possui coluna Itens com resumo — manter como está
- As abas Pré-Orçamento, Aprovados e Histórico não têm resumo de itens visível

Mudanças necessárias:
1. Na coluna Objeto do Radar, substituir o texto longo do objeto pelo resumo dos itens como conteúdo principal
2. O objeto original passa a ser acessível via tooltip (hover) e botão de edição (✏️)
3. Garantir que busca por texto pesquisa nos nomes dos itens
4. Aplicar o mesmo padrão de resumo de itens nas abas Pré-Orçamento, Aprovados e Histórico
5. Se orçamento não tem itens, usar o texto do objeto como fallback

## Dev Notes

### Helper getItemsSummary

```javascript
function getItemsSummary(orcamento, maxItems = 5) {
  const itens = orcamento.itens || orcamento.items || [];
  if (!itens.length) return null;
  const nomes = itens
    .slice(0, maxItems)
    .map(i => i.nome || i.descricao || i.item || '')
    .filter(Boolean);
  if (!nomes.length) return null;
  const resumo = nomes.join(', ');
  return itens.length > maxItems ? `${resumo}... (+${itens.length - maxItems})` : resumo;
}
```

### Padrão de célula no Radar (renderOrcamentos)

```javascript
// Antes: exibir objeto como principal
`<td title="${o.objeto || ''}">${o.objeto || ''}</td>`

// Depois: exibir itens como principal, objeto no tooltip
const itemsSummary = getItemsSummary(o);
const display = itemsSummary || o.objeto || '';
const tooltip = o.objeto || '';
`<td title="${tooltip}">
  <span class="items-summary">${display}</span>
  <button class="btn-edit-objeto" data-id="${o.id}" title="Editar objeto">✏️</button>
</td>`
```

### Busca por itens em filteredOrcamentos

```javascript
// Garantir que a busca inclui nomes de itens
const termo = searchText.toLowerCase();
return orcamentos.filter(o => {
  const itens = o.itens || o.items || [];
  const contemItem = itens.some(i =>
    (i.nome || i.descricao || '').toLowerCase().includes(termo)
  );
  return (
    (o.objeto || '').toLowerCase().includes(termo) ||
    (o.municipio || '').toLowerCase().includes(termo) ||
    (o.fornecedor || '').toLowerCase().includes(termo) ||
    contemItem
  );
});
```
