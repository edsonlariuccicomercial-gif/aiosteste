## Story

**As a** gestor financeiro,  
**I want** controlar contas a pagar e contas a receber dentro do GDP,  
**so that** o pos-licitacao tenha visao financeira operacional e cobranca automatica.

---

## Acceptance Criteria

### AC-1: Contas a pagar
- [ ] Deve existir aba `Contas a Pagar`
- [ ] Cada conta deve ter descricao, categoria, forma, valor, vencimento e status

### AC-2: Contas a receber
- [ ] Deve existir aba `Contas a Receber`
- [ ] Contas devem poder nascer automaticamente de notas fiscais e tambem por inclusao manual
- [ ] Cada conta deve ter categoria

### AC-3: Formas de pagamento/recebimento
- [ ] O sistema deve suportar `boleto`, `pix`, `ted` e `incluir`

### AC-4: Cobranca automatica
- [ ] Contas a receber devem suportar cobranca automatica por WhatsApp e e-mail
- [ ] O sistema deve registrar data do ultimo disparo

### AC-5: Base para conciliacao
- [ ] O modelo deve prever conciliacao bancaria por API
- [ ] Cada cobranca deve possuir identificador conciliavel
