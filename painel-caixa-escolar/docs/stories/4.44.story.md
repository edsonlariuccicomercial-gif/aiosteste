## Story

**As a** operador financeiro que gerencia contas a pagar no GDP,
**I want** checkboxes para ações em lote, menu contextual "..." por conta, campo de data de emissão no formulário e tela de detalhes com edição,
**so that** eu consiga gerenciar contas a pagar de forma mais eficiente, com menos cliques e mais controle.

---

## Contexto / Motivação

Atualmente a seção Contas a Pagar tem botões de ação (Baixar, Clonar, Excluir) inline numa coluna "Ação" que ocupa espaço e não permite operações em lote. O formulário de nova conta não possui campo de data de emissão. Não existe tela de detalhes da conta — o operador não consegue ver/editar todos os campos sem abrir o formulário de clone.

**Padrão de referência:** A seção de Pedidos já usa checkboxes com bulk actions (`_selectedPedidoIds`, `toggleSelectAllPedidos`, `atualizarSelecaoPedidos`) — seguir o mesmo padrão.

**Arquivo principal:** `painel-caixa-escolar/squads/caixa-escolar/dashboard/gdp-contratos.html`

## Acceptance Criteria

### AC1 — Campo "Data de Emissão" no formulário
- [ ] Adicionar campo `cp-data-emissao` (type=date) no formulário de nova conta a pagar (modal `cp-form-card`)
- [ ] Campo posicionado no grid do formulário, antes do campo Vencimento
- [ ] Ao salvar (`registrarContaPagar()`), gravar `dataEmissao` no objeto da conta
- [ ] Campo obrigatório — validar junto com os demais campos
- [ ] Default: data de hoje (`new Date().toISOString().split('T')[0]`)

### AC2 — Redirect para aba "Emitida" após salvar
- [ ] Após `registrarContaPagar()` salvar com sucesso, mudar a tab ativa para "emitida" (`CONTAS_PAGAR_STATUS_TABS[0]`)
- [ ] Re-renderizar a tabela filtrada pela aba "emitida"
- [ ] A conta recém-criada deve aparecer visível na lista

### AC3 — Checkboxes na tabela
- [ ] Adicionar checkbox "select all" no header da tabela (`cp-select-all`)
- [ ] Adicionar checkbox individual em cada row (class `cp-check`, value=`item.id`)
- [ ] Criar variável `_selectedContaPagarIds = new Set()`
- [ ] Criar funções `toggleSelectAllContasPagar()` e `atualizarSelecaoContasPagar()`
- [ ] Checkbox do header reflete estado (checked, indeterminate, unchecked)

### AC4 — Barra de ações em lote (bulk actions)
- [ ] Criar barra `cp-bulk-actions` que aparece quando há contas selecionadas
- [ ] Exibir: "X conta(s) selecionada(s)"
- [ ] Botão **"Pagar (Baixar)"** — chama `registrarBaixaContaPagar()` para cada conta selecionada
- [ ] Botão **"Excluir"** — confirmação única, exclui todas selecionadas de uma vez
- [ ] Após ação em lote: limpar seleção, re-renderizar tabela, esconder barra

### AC5 — Menu contextual "..." por conta
- [ ] Adicionar botão "..." (três pontos / ellipsis) ao lado do checkbox de cada row
- [ ] Ao clicar, abrir menu dropdown com as opções:
  - **Pagar (Baixar)** → `registrarBaixaContaPagar(item.id)`
  - **Clonar** → `clonarContaPagar(item.id)`
  - **Excluir** → `excluirContaPagar(item.id)`
- [ ] Menu fecha ao clicar fora (click outside) ou ao selecionar uma opção
- [ ] Se status === "paga", opção "Pagar" desabilitada ou oculta

### AC6 — Remover coluna "Ação"
- [ ] Remover a coluna "Ação" do header da tabela
- [ ] Remover a `<td>` de ações do render de cada row (inline buttons)
- [ ] As ações agora vivem no menu "..." (AC5) e na barra bulk (AC4)

### AC7 — Detalhe da conta ao clicar na descrição
- [ ] Ao clicar na descrição da conta (td com `item.descricao`), abrir modal/painel de detalhes
- [ ] Exibir todos os campos: descrição, categoria, forma, valor, vencimento, data emissão, status, auditoria
- [ ] Botão **"Editar"** que habilita edição dos campos (descrição, categoria, forma, valor, vencimento, data emissão)
- [ ] Botão **"Salvar"** que persiste as alterações via `saveContasPagar()` e atualiza audit
- [ ] Botão **"Fechar"** para voltar à listagem
- [ ] Campos de status e auditoria são read-only (não editáveis)

## Technical Notes

### Padrão de checkbox (copiar de Pedidos)
```javascript
var _selectedContaPagarIds = new Set();

function toggleSelectAllContasPagar() {
  const checked = document.getElementById("cp-select-all").checked;
  document.querySelectorAll(".cp-check").forEach(cb => { cb.checked = checked; });
  atualizarSelecaoContasPagar();
}

function atualizarSelecaoContasPagar() {
  _selectedContaPagarIds.clear();
  document.querySelectorAll(".cp-check:checked").forEach(cb => { _selectedContaPagarIds.add(cb.value); });
  const bar = document.getElementById("cp-bulk-actions");
  bar.classList.toggle("hidden", _selectedContaPagarIds.size === 0);
  // Atualizar contador
}
```

### Menu "..." (dropdown contextual)
```html
<td style="position:relative">
  <input type="checkbox" class="cp-check" value="${item.id}" onchange="atualizarSelecaoContasPagar()">
  <button class="btn-icon" onclick="toggleCpMenu('${item.id}')">⋯</button>
  <div id="cp-menu-${item.id}" class="dropdown-menu hidden">
    <a onclick="registrarBaixaContaPagar('${item.id}')">Pagar (Baixar)</a>
    <a onclick="clonarContaPagar('${item.id}')">Clonar</a>
    <a onclick="excluirContaPagar('${item.id}')">Excluir</a>
  </div>
</td>
```
