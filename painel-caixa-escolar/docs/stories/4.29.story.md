## Story

**As a** operador que monta propostas para cotações da Caixa Escolar,
**I want** que o sistema preencha automaticamente preços, margens e marcas com base no banco inteligente,
**so that** eu consiga montar propostas competitivas em minutos, não em horas, com preços baseados em dados reais.

---

## Contexto / Motivação

Hoje o operador monta pré-orçamentos manualmente: olha cada item do orçamento, busca no banco de preços, ajusta margem, preenche marca. Isso é lento e propenso a erros. Com o Motor de Inteligência completo (4.26 + 4.27 + 4.28), temos os dados para automatizar:

- **Item mestre** resolve a correspondência entre item da licitação e item do banco
- **Histórico** dá o custo atualizado
- **Inteligência** sugere o preço ideal baseado em concorrência

### Funcionalidade existente

- `gerarPreOrcamento(orcId)` → cria pré-orçamento a partir de orçamento SGD
- `findBancoItem(nomeItem)` → busca básica no banco (3 níveis de matching)
- Preenchimento atual: custoBase, precoReferencia, unidade, marca do banco
- Margem padrão: 30% fixo (do perfil)

### Evolução desejada

- Matching via Item Mestre (muito mais preciso)
- Margem sugerida baseada em concorrência (não fixa)
- Indicador de confiança por item (dados frescos vs antigos)
- One-click para preencher proposta inteira
- Resumo com análise de competitividade ANTES de enviar

## Acceptance Criteria

### AC-1: Auto-match via Item Mestre

- [ ] Ao gerar pré-orçamento, cada item do orçamento SGD é matcheado contra itens mestres (não apenas banco flat)
- [ ] Usar aliases e similaridade (threshold >= 0.6 para sugerir, >= 0.8 para auto)
- [ ] Exibir indicador de confiança do match: verde (auto), amarelo (sugerido), vermelho (sem match)
- [ ] Itens sem match ganham destaque visual para preenchimento manual

### AC-2: Preço sugerido inteligente

- [ ] Para cada item com match, calcular preço sugerido baseado em:
  - Meu custo atual (último preço do fornecedor)
  - Menor preço da concorrência (do histórico de licitações)
  - Média do mercado (de todos os fornecedores no banco)
- [ ] Fórmula de sugestão: `max(meuCusto * (1 + margemMinima), menorConcorrente * 0.95)`
- [ ] Exibir como tooltip: "Sugestão: R$ X (custo R$ Y + margem Z%)"

### AC-3: Margem dinâmica por item

- [ ] Substituir margem fixa de 30% por margem calculada por item
- [ ] Se histórico de concorrência disponível → margem que garante competitividade
- [ ] Se sem histórico → usar margem padrão do perfil
- [ ] Exibir margem real vs margem sugerida na tabela do pré-orçamento
- [ ] Cor: verde se margem >= 15%, amarelo se 5-15%, vermelho se < 5%

### AC-4: Preenchimento one-click

- [ ] Botão "Preencher com Banco Inteligente" no topo do pré-orçamento
- [ ] Preenche automaticamente: preço, marca, margem para TODOS os itens com match
- [ ] Mostra resumo: "X de Y itens preenchidos automaticamente, Z precisam revisão manual"
- [ ] Itens sem match ficam com preço zero e destaque vermelho

### AC-5: Análise pré-envio

- [ ] Antes de enviar proposta, exibir card de resumo:
  - Valor total da proposta
  - Margem média ponderada
  - Competitividade estimada: "Sua proposta está X% abaixo/acima da média de mercado"
  - Itens de risco: onde seu preço está acima da concorrência
  - Itens de oportunidade: onde margem é muito alta (pode reduzir para ser mais competitivo)
- [ ] Sugestão: "Reduzindo {item} de R$ X para R$ Y, sua proposta fica Z% mais competitiva"

### AC-6: Aprendizado pós-resultado

- [ ] Quando resultado da licitação é conhecido (ganhou/perdeu), registrar no banco:
  - Itens que ganharam → meu preço era competitivo, reforçar
  - Itens que perderam → preço do vencedor como referência
- [ ] Input manual: botão "Registrar Resultado" no pré-orçamento enviado
- [ ] Dados alimentam o histórico e melhoram futuras sugestões

## Dev Notes

### Fórmula de preço sugerido

```javascript
function calcPrecoSugerido(mestreId) {
  const mestre = mestres.find(m => m.id === mestreId);
  if (!mestre) return null;

  const historico = getHistoricoMestre(mestreId);
  const meuCusto = historico.filter(h => h.fornecedor === "proprio").sort((a,b) => b.data.localeCompare(a.data))[0]?.preco;
  const menorConcorrente = Math.min(...historico.filter(h => h.tipo === "concorrente").map(h => h.preco));
  const mediaMercado = historico.reduce((s, h) => s + h.preco, 0) / historico.length;

  const margemMinima = 0.08; // 8% mínimo
  const precoMinimo = meuCusto ? meuCusto * (1 + margemMinima) : mediaMercado;
  const precoCompetitivo = menorConcorrente ? menorConcorrente * 0.97 : mediaMercado;

  return Math.max(precoMinimo, Math.min(precoCompetitivo, mediaMercado));
}
```
