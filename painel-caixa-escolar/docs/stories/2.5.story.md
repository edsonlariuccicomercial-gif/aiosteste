# Story 2.5 — Testes Automatizados (Vitest + Playwright)

**Epic:** 2 — Resolucao de Debito Tecnico
**Sprint:** 2 (Semana 4-5)
**Prioridade:** P2 — ALTO
**Estimativa:** 32 horas
**Debitos resolvidos:** SYS-2

---

## Descricao

Implementar cobertura de testes automatizados partindo de 0% para >= 60%. Usar Vitest para testes unitarios (logica de negocios, validacao, matching) e Playwright para testes e2e (fluxos criticos: login, scan SGD, envio de proposta, pedido). Esta story e a fundacao de qualidade que permite todas as refatoracoes futuras com seguranca.

---

## Criterios de Aceitacao

- [ ] Vitest configurado e funcional
- [ ] Playwright configurado e funcional
- [ ] Testes unitarios para: schemas Zod, matching de itens, safe-json, audit log
- [ ] Testes unitarios para: calculo de precos, filtros de orcamentos, formatacao
- [ ] Testes de integracao para: APIs Express (auth, submit, scan, order)
- [ ] Testes e2e para: fluxo de login, visualizacao de orcamentos, envio de proposta
- [ ] Testes e2e para: fluxo GDP portal (login escola, catalogo, pedido)
- [ ] Cobertura de testes >= 60% para codigo critico
- [ ] Scripts npm `test`, `test:unit`, `test:e2e`, `test:coverage` funcionais
- [ ] Testes rodam em < 2 minutos (unit) e < 5 minutos (e2e)

---

## Tarefas

- [ ] Instalar Vitest: `npm install -D vitest @vitest/coverage-v8`
- [ ] Criar `vitest.config.js` com configuracao para ESM
- [ ] Criar estrutura de diretorio `tests/unit/`, `tests/e2e/`, `tests/fixtures/`
- [ ] Escrever testes para schemas Zod (cada schema: valid + invalid)
- [ ] Escrever testes para `safe-json.js` (read, write, atomic, corrupt)
- [ ] Escrever testes para matching de itens (10+ cenarios)
- [ ] Escrever testes para calculo de precos (custo + margem)
- [ ] Escrever testes para filtros de orcamentos (escola, municipio, status)
- [ ] Escrever testes para `auth-middleware.js` (401, 403, 200)
- [ ] Escrever testes para `rate-limiter.js`
- [ ] Escrever testes para `audit-log.js`
- [ ] Configurar Playwright: `npx playwright install`
- [ ] Criar `playwright.config.js` com base URL localhost:8082
- [ ] Escrever e2e: fluxo de login fornecedor
- [ ] Escrever e2e: visualizacao de orcamentos + filtros
- [ ] Escrever e2e: geracao de pre-orcamento
- [ ] Escrever e2e: fluxo GDP portal (login escola → catalogo → pedido)
- [ ] Criar fixtures de dados para testes (orcamentos, precos, escolas)
- [ ] Adicionar scripts npm: `test`, `test:unit`, `test:e2e`, `test:coverage`
- [ ] Verificar cobertura >= 60%
