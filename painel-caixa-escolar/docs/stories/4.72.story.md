## Story

**As a** usuario do GDP,
**I want** que ao atualizar a pagina (F5 ou Ctrl+Shift+R) eu veja as funcionalidades mais recentes,
**so that** nao preciso limpar cache manualmente nem abrir aba anonima para usar o sistema.

---

## Acceptance Criteria

- [ ] **AC-1:** Desregistrar qualquer Service Worker no topo do gdp-contratos.html (antes de todos os scripts)
- [ ] **AC-2:** Verificar se existe arquivo de Service Worker (sw.js, service-worker.js) e remover
- [ ] **AC-3:** Adicionar meta tag no-cache no head do HTML
- [ ] **AC-4:** Apos deploy, usuario ve paginacao em Clientes e Central de Produtos com F5 simples

## Contexto tecnico

A paginacao (Story 4.70) esta implementada e funcional em producao — testada com Playwright mostrando 50 itens/pagina e controles [1] [2] [»].

O browser do usuario nao carrega os scripts novos apesar de:
- Cache bust v7 em todos os scripts
- Ctrl+Shift+R
- Deploy com --force (sem cache do edge)
- vercel.json com Cache-Control: no-cache

Causa provavel: **Service Worker** registrado anteriormente que intercepta requests e serve versoes cacheadas.

---

## Fix recomendado pelo @analyst

Adicionar no topo do `gdp-contratos.html`, ANTES de qualquer outro script:

```html
<script>
if ('serviceWorker' in navigator) {
  navigator.serviceWorker.getRegistrations().then(function(registrations) {
    registrations.forEach(function(registration) { registration.unregister(); });
  });
}
</script>
```

Tambem verificar se existem arquivos SW no projeto:
- `sw.js`
- `service-worker.js`
- Qualquer arquivo com `self.addEventListener('fetch'`
