## Story

**As a** fornecedor que recebe pedidos de escolas,
**I want** preencher os dados de pagamento (forma de recebimento, conta bancária, condição de pagamento) diretamente no detalhe do pedido,
**so that** ao gerar a nota fiscal o sistema já emita boleto via API bancária, registre a data de vencimento na NF e crie automaticamente a conta a receber com todos os dados preenchidos.

---

## Contexto / Motivação

Hoje o fornecedor gera a NF e o sistema cria a conta a receber com vencimento fixo de 28 dias e forma "boleto" padrão. O fornecedor não tem como definir:
- A forma de recebimento real (boleto, PIX, TED)
- Qual conta bancária usar
- O prazo de pagamento negociado (ex: 30/60/90 dias)

Isso causa retrabalho — o fornecedor precisa editar a conta a receber depois.

**Fluxo desejado:**
1. Pedido recebido → fornecedor abre detalhe
2. Preenche: forma de recebimento + conta bancária + prazo/vencimento
3. Gera NF → sistema usa esses dados para:
   - Incluir forma de pagamento e vencimento na NF
   - Gerar boleto/PIX via API bancária
   - Criar conta a receber com tudo preenchido

**Arquivo principal:** `painel-caixa-escolar/squads/caixa-escolar/dashboard/gdp-contratos.html`

## Acceptance Criteria

### AC1 — Seção "Dados de Pagamento" no detalhe do pedido
- [ ] Ao abrir `verPedidoDetalhe()`, exibir nova seção "Dados de Pagamento" abaixo da seção de Cadastro Fiscal
- [ ] Campos:
  - **Forma de Recebimento** (select): Boleto, PIX, TED, Dinheiro
  - **Conta Bancária** (select): popular a partir das contas cadastradas em configurações ou hardcoded inicial (Banco, Agência, Conta, PIX)
  - **Condição de Pagamento — Prazo** (select): À vista, 15 dias, 28 dias, 30 dias, 45 dias, 60 dias, 90 dias
  - **Condição de Pagamento — Vencimento** (date): auto-calculado a partir da data do pedido + prazo selecionado, mas editável
- [ ] Ao mudar o prazo, recalcular a data de vencimento automaticamente
- [ ] Dados salvos no objeto do pedido em `pedido.pagamento`

### AC2 — Persistência dos dados de pagamento no pedido
- [ ] Novo campo `pagamento` no objeto pedido:
  ```javascript
  pedido.pagamento = {
    forma: "boleto",           // boleto|pix|ted|dinheiro
    contaBancaria: { banco, agencia, conta, pix, apelido },
    condicao: "28",            // dias
    vencimento: "2026-04-26"   // data calculada
  }
  ```
- [ ] Ao salvar (botão "Salvar Dados de Pagamento"), persistir via `savePedidos()`
- [ ] Dados permanecem ao reabrir o detalhe do pedido

### AC3 — Integração com geração de NF
- [ ] Na função `buildInvoiceFromOrder()`, usar `pedido.pagamento` para:
  - Definir `invoice.cobranca.forma` a partir de `pedido.pagamento.forma`
  - Definir `invoice.vencimento` a partir de `pedido.pagamento.vencimento`
  - Incluir dados da conta bancária em `invoice.cobranca`
- [ ] Se `pedido.pagamento` não estiver preenchido, manter comportamento atual (fallback boleto + 28 dias)

### AC4 — Integração com Contas a Receber
- [ ] Na função `buildReceivableFromInvoice()`, usar dados de pagamento do pedido/NF:
  - `receivable.forma` = forma de recebimento do pedido
  - `receivable.vencimento` = vencimento definido pelo fornecedor
  - `receivable.cobranca.contaBancariaId` = conta selecionada
  - `receivable.contaBancaria` = dados completos da conta
- [ ] A conta a receber deve ser criada instantaneamente junto com a NF (já acontece, garantir que os dados fluem)

### AC5 — Indicação visual de status de pagamento
- [ ] No detalhe do pedido, se `pedido.pagamento` já estiver preenchido, mostrar badge verde "Pagamento configurado"
- [ ] Se não preenchido, mostrar badge amarelo "Pagamento pendente — preencha antes de gerar NF"
- [ ] Na tabela de pedidos, não bloquear NF sem pagamento (apenas avisar)

## Technical Notes

### Estrutura pagamento no pedido
```javascript
pedido.pagamento = {
  forma: "boleto",
  contaBancaria: {
    banco: "Sicoob",
    agencia: "3068",
    conta: "12345-6",
    pix: "36802147000142",
    apelido: "Conta Principal"
  },
  condicao: "28",
  vencimento: "2026-04-26"
}
```

### Cálculo automático de vencimento
```javascript
function calcularVencimento(dataPedido, prazoDias) {
  const d = new Date(dataPedido);
  d.setDate(d.getDate() + Number(prazoDias));
  return d.toISOString().split("T")[0];
}
```

### Fluxo na geração de NF
```
verPedidoDetalhe() → preenche pagamento → salva
    ↓
gerarNotaFiscalPedido() → buildInvoiceFromOrder()
    ↓ usa pedido.pagamento.forma, .vencimento, .contaBancaria
buildReceivableFromInvoice()
    ↓ usa invoice.cobranca.forma, .vencimento, .contaBancaria
Conta a Receber criada com dados corretos
```
