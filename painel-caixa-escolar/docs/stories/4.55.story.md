## Story

**As a** gestor do sistema GDP,
**I want** que a exclusão de contas a receber seja permanente, que extratos importados fiquem salvos em formato colapsado, e que eu possa selecionar contratos com checkbox para alterar status em massa,
**so that** o sistema pare de restaurar dados excluídos, a conciliação bancária siga o modelo visual aprovado, e a gestão de contratos seja mais ágil.

---

## Contexto / Histórico

A Story 4.51 tentou resolver a exclusão de contas a receber e a persistência de extratos, mas os bugs persistem. O usuário reportou o problema múltiplas vezes. A análise do @analyst (Atlas) identificou as causas raiz com precisão:

1. **CR:** `bulkExcluirContasReceber()` não sincroniza com Supabase nem rastreia IDs deletados
2. **Conciliação:** `renderConciliacao()` sempre mostra tabela de detalhes aberta, sem state tracking
3. **Contratos:** falta checkbox e ação bulk — funcionalidade nova solicitada

## Acceptance Criteria

### AC-1 — Exclusão definitiva de Contas a Receber (BUG FIX)

**Arquivo:** `gdp-init.js` — função `bulkExcluirContasReceber()` (linhas ~352-360)

- [x] `bulkExcluirContasReceber()` DEVE chamar `gdpApi.contas_receber.remove(contaId)` para CADA conta selecionada
- [x] `bulkExcluirContasReceber()` DEVE rastrear cada ID deletado em `gdp.contas-receber.deleted.v1` (localStorage)
- [x] Após exclusão bulk, as contas NÃO devem reaparecer após sync do Supabase (`syncFromCloud`)
- [x] Após exclusão bulk, as contas NÃO devem reaparecer após refresh da página (F5)
- [x] Manter a lógica de `excluirContaReceber()` (individual) como referência — ela já funciona corretamente
- [x] Toast deve confirmar quantidade excluída

**Lógica de referência (função individual que funciona):**
```javascript
// excluirContaReceber() em gdp-init.js:496-524
// 1. Remove do array in-memory
// 2. saveContasReceber() → localStorage
// 3. gdpApi.contas_receber.remove(contaId) → Supabase
// 4. Rastreia ID em gdp.contas-receber.deleted.v1
```

**Teste de validação:**
1. Criar 3 contas a receber de teste
2. Selecionar todas e excluir via bulk
3. Aguardar 10 segundos (sync cycle)
4. Refresh (F5) — contas NÃO devem reaparecer
5. Abrir em outra aba — contas NÃO devem aparecer

### AC-2 — Conciliação bancária: extrato colapsado após importação

**Arquivos:** `gdp-core.js` (renderConciliacao, reabrirExtrato, registrarExtrato), `gdp-contratos.html` (seção conciliação)

- [x] Após importar um extrato, ele DEVE ser salvo e exibido em formato **fechado/colapsado**
- [x] A lista de extratos deve exibir em formato tabela com colunas: checkbox | Data | Arquivo | Conta financeira | Conciliados/Total
- [x] Cada linha de extrato deve ter um checkbox para seleção (para exclusão)
- [x] Ao clicar sobre o nome do arquivo, o extrato deve **reabrir** mostrando os lançamentos
- [x] Quando há mais de 1 extrato, devem aparecer como lista (múltiplas linhas na tabela)
- [x] Ao reabrir um extrato parcialmente conciliado, informar quais lançamentos estão **pendentes de conciliação**
- [x] A tabela de detalhes (lançamentos) só deve aparecer quando um extrato estiver aberto
- [x] O extrato NÃO deve ficar aberto permanentemente na página

**Modelo visual de referência (conforme imagem do usuário):**
```
[x] Data         | Arquivo                                    | Conta financeira | Conciliados/Total
[ ] 20/05/2026   | Extrato-15-05-2026-a-19-05-2026-OFX.ofx   | Banco Inter      | (61/61)
```

**Implementação técnica:**
- Adicionar `isOpen: false` como default em `registrarExtrato()`
- `renderConciliacao()` deve verificar `extrato.isOpen` antes de renderizar tabela de detalhes
- `reabrirExtrato(idx)` deve setar `extratos[idx].isOpen = true` e fechar os outros
- Vincular cada item de conciliação ao `extratoId` correspondente para filtrar corretamente

**Teste de validação (Playwright):**
1. Importar extrato OFX
2. Verificar que extrato aparece na tabela em formato colapsado (sem detalhes visíveis)
3. Clicar no arquivo → detalhes devem expandir
4. Importar segundo extrato → deve aparecer como segunda linha, colapsado
5. Refresh (F5) → extratos devem continuar salvos e colapsados

## Technical Notes

### Causa raiz AC-1 (análise @analyst)
`bulkExcluirContasReceber()` em `gdp-init.js` remove do array in-memory e salva no localStorage, mas:
- **NÃO** chama `gdpApi.contas_receber.remove()` → Supabase mantém os registros
- **NÃO** rastreia IDs em `gdp.contas-receber.deleted.v1` → `syncFromCloud()` re-importa
- A função individual `excluirContaReceber()` faz ambos corretamente — basta replicar a lógica

### Causa raiz AC-2 (análise @analyst)
`renderConciliacao()` em `gdp-core.js:2006-2043` SEMPRE renderiza a tabela de detalhes. Não existe state tracking (`isOpen`) nos extratos. `reabrirExtrato()` não filtra items por extrato.

### AC-3 (funcionalidade nova)
`toggleStatusContrato()` já existe e funciona individualmente. Schema Supabase já tem campo `status TEXT`. Basta adicionar checkbox no template do card e lógica de seleção bulk.
