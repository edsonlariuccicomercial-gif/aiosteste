## Story

**As a** fornecedor que envia pedidos ao Olist/Tiny,
**I want** que o endpoint `order.js` envie pedidos corretamente ao Tiny com o SKU real do produto,
**so that** os pedidos chegam ao ERP com os itens corretos e as notas fiscais podem ser emitidas.

---

## Root Cause Analysis

A edição anterior do `order.js` (commit `e267a69`) introduziu 4 bugs:

### Bug 1: `skuNumerico` undefined (linha 416)
A variável `skuNumerico` foi removida na edição (substituída por `item.sku = generateSku(...)`), mas a referência `codigo: skuNumerico` na criação do produto permaneceu. Resultado: produto criado no Tiny com `codigo: undefined`.

### Bug 2: Braces desalinhados (linhas 405-441)
O `else` interno (`} else {` na linha 405) não fecha corretamente. O bloco de criação de produto (linhas 409-440) escapa do escopo do else, podendo executar em contextos incorretos.

### Bug 3: Step 2 (criar pedido) fica inalcançável (BLOCKER)
O `}` na linha 441 fecha apenas o else interno, mas o else externo (linha 396) nunca fecha antes do rate limit e do Step 2. Resultado: se TODOS os itens têm SKU (caso normal após auto-cadastro da 4.9), o fluxo entra no `if` (linha 390), o `else` nunca executa, e o rate limit + Step 2 ficam dentro do else → **pedido NUNCA é enviado ao Tiny**.

### Bug 4: `normalizeDescription` encurta descrição
O `buildTinyPayload` usa `normalizeDescription(item.description)` que mapeia "MAÇÃ FUJI TIPO EXPORTAÇÃO 1KG" para "Maca". Se o produto no Tiny está cadastrado com a descrição completa, o match falha.

## Acceptance Criteria

### AC-1: Reescrever Step 1 do order.js com estrutura limpa
- [ ] Reescrever o bloco Step 1 completo (linhas 383-447) com braces corretos
- [ ] Lógica: se item tem SKU → usar direto, NÃO criar produto no Tiny (já existe)
- [ ] Se item NÃO tem SKU → buscar no Tiny por nome → se não encontrar, gerar SKU e criar produto
- [ ] Variável `skuNumerico` substituída por `item.sku` em toda referência
- [ ] Rate limit FORA de qualquer bloco condicional (sempre executa entre items)

### AC-2: Step 2 (criar pedido) sempre executa
- [ ] O bloco Step 2 (`buildTinyPayload` + POST ao Tiny) deve estar FORA do loop de Step 1
- [ ] Confirmar que o pedido é criado independentemente de os itens terem ou não SKU

### AC-3: buildTinyPayload usa descrição original
- [ ] Na criação do pedido (buildTinyPayload), usar `item.description` como descricao do item ao invés de `normalizeDescription(item.description)`
- [ ] `normalizeDescription` deve ser usada APENAS ao cadastrar produtos novos (não ao criar pedido)

### AC-4: Dados do cliente extraídos corretamente
- [ ] Confirmar que o objeto `cliente` enviado pelo frontend é lido e usado no `buildTinyPayload`
- [ ] Campos: nome, cpf_cnpj, endereco, numero, bairro, cep, cidade, uf, ie, fone, email

### AC-5: Teste end-to-end
- [ ] Enviar pedido com itens que TÊM SKU → pedido chega ao Tiny com SKU correto
- [ ] Enviar pedido com itens SEM SKU → produto criado no Tiny, pedido enviado
- [ ] Verificar que descrição no pedido é a original, não a normalizada
- [ ] Verificar dados do cliente na NF-e

  ```javascript
  // ─── Step 1: Ensure all items have a valid SKU ───
  const ncmAlerts = [];
  for (let i = 0; i < order.items.length; i++) {
    const item = order.items[i];
    const ncmMatch = findNcm(item.description);
    const ncmCode = ncmMatch ? ncmMatch.ncm : "";
    if (!ncmCode) {
      ncmAlerts.push({ item: item.itemNum || i + 1, description: item.description });
    }

    // If item already has SKU, skip Tiny product search/creation
    if (item.sku && String(item.sku).trim()) {
      continue;
    }

    // No SKU — search Tiny for existing product
    const existing = await searchTinyProduct(token, item.description);
    if (existing && existing.codigo) {
      item.sku = existing.codigo;
      if (!ncmMatch && existing.ncm) item._tinyNcm = existing.ncm;
    } else {
      // Not found — generate SKU and create product in Tiny
      item.sku = generateSku(item, i, arp || orderId);
      const produto = {
        produto: {
          codigo: item.sku,
          nome: normalizeDescription(item.description),
          ncm: ncmCode,
          unidade: "UN",
          preco: Number(item.unitPrice || 0),
          origem: "0",
          tipo: "P",
          situacao: "A",
        },
      };
      const prodForm = new URLSearchParams();
      prodForm.set("token", token);
      prodForm.set("formato", "json");
      prodForm.set("produto", JSON.stringify(produto));
      try {
        await fetch("https://api.tiny.com.br/api2/produto.incluir.php", {
          method: "POST",
          headers: { "Content-Type": "application/x-www-form-urlencoded" },
          body: prodForm.toString(),
        });
      } catch (_e) { /* best-effort */ }
    }

    // Tiny rate limit
    if (i < order.items.length - 1) {
      await new Promise(r => setTimeout(r, 1500));
    }
  }
  ```
