## Story

**As a** operador financeiro que gerencia contas a receber no GDP,
**I want** checkboxes para ações em lote, menu contextual "..." por conta com opções de receber/cobrar/clonar/excluir, campo de data de emissão no formulário e tela de detalhes com edição,
**so that** eu consiga gerenciar contas a receber de forma mais eficiente, com operações em lote e menos cliques.

---

## Contexto / Motivação

A seção Contas a Receber tem 7 botões de ação inline (Cobrar, Sync, Baixar, Clonar, Conciliar, Diverg., Excluir) que ocupam muito espaço e confundem o operador. Os botões Sync, Conciliar e Diverg. devem ser removidos — são operações de integração bancária que não fazem sentido no fluxo manual atual. O formulário não possui campo de data de emissão. Não existe tela de detalhes.

**Lógica espelhada da Story 4.44** (Contas a Pagar), com diferenças:
- Bulk actions: **Receber**, **Imprimir Boletos**, **Excluir** (em vez de Pagar/Excluir)
- Menu "...": **Receber (Baixar)**, **Cobrar**, **Clonar**, **Excluir** (mantém Cobrar para atrasos)
- Remover botões: Sync, Conciliar, Diverg. (não mais necessários)

**Arquivo principal:** `painel-caixa-escolar/squads/caixa-escolar/dashboard/gdp-contratos.html`

## Acceptance Criteria

### AC1 — Campo "Data de Emissão" no formulário
- [ ] Adicionar campo `cr-data-emissao` (type=date) no formulário de nova conta a receber (modal `cr-form-card`)
- [ ] Campo posicionado no grid do formulário, antes do campo Vencimento
- [ ] Ao salvar (`registrarContaReceber()`), gravar `dataEmissao` no objeto da conta
- [ ] Campo obrigatório — validar junto com os demais campos
- [ ] Default: data de hoje (`new Date().toISOString().split('T')[0]`)

### AC2 — Redirect para aba "Emitida" após salvar
- [ ] Após `registrarContaReceber()` salvar com sucesso, mudar a tab ativa para "emitida" (`CONTAS_RECEBER_STATUS_TABS[0]`)
- [ ] Re-renderizar a tabela filtrada pela aba "emitida"
- [ ] A conta recém-criada deve aparecer visível na lista

### AC3 — Checkboxes na tabela
- [ ] Adicionar checkbox "select all" no header da tabela (`cr-select-all`)
- [ ] Adicionar checkbox individual em cada row (class `cr-check`, value=`item.id`)
- [ ] Criar variável `_selectedContaReceberIds = new Set()`
- [ ] Criar funções `toggleSelectAllContasReceber()` e `atualizarSelecaoContasReceber()`
- [ ] Checkbox do header reflete estado (checked, indeterminate, unchecked)

### AC4 — Barra de ações em lote (bulk actions)
- [ ] Criar barra `cr-bulk-actions` que aparece quando há contas selecionadas
- [ ] Exibir: "X conta(s) selecionada(s)"
- [ ] Botão **"Receber"** — chama `registrarBaixaRecebimento()` para cada conta selecionada
- [ ] Botão **"Imprimir Boletos"** — gera impressão dos boletos das contas selecionadas (que tenham forma === "boleto")
- [ ] Botão **"Excluir"** — confirmação única, exclui todas selecionadas de uma vez
- [ ] Após ação em lote: limpar seleção, re-renderizar tabela, esconder barra

### AC5 — Menu contextual "..." por conta
- [ ] Adicionar botão "..." (três pontos / ellipsis) ao lado do checkbox de cada row
- [ ] Ao clicar, abrir menu dropdown com as opções:
  - **Receber (Baixar)** → `registrarBaixaRecebimento(item.id)`
  - **Cobrar** → `dispararCobrancaAutomatica(item.id)` (para cobranças de atrasos)
  - **Clonar** → `clonarContaReceber(item.id)`
  - **Excluir** → `excluirContaReceber(item.id)`
- [ ] Menu fecha ao clicar fora (click outside) ou ao selecionar uma opção
- [ ] Se status === "recebida", opção "Receber" desabilitada ou oculta

### AC6 — Remover coluna "Ação" e botões obsoletos
- [ ] Remover a coluna "Ação" do header da tabela
- [ ] Remover a `<td>` de ações do render de cada row (todos os inline buttons)
- [ ] Remover botões **Sync** (`sincronizarCobrancaProvider`), **Conciliar** (`atualizarStatusConciliacao(...,'conciliado_api_bancaria')`), **Diverg.** (`atualizarStatusConciliacao(...,'divergencia_api_bancaria')`) — não mais necessários
- [ ] As ações agora vivem no menu "..." (AC5) e na barra bulk (AC4)

### AC7 — Detalhe da conta ao clicar na descrição
- [ ] Ao clicar na descrição da conta (td com `item.descricao`), abrir modal/painel de detalhes
- [ ] Exibir todos os campos: descrição, categoria, cliente, forma, valor, vencimento, data emissão, status, automação, cobrança, auditoria
- [ ] Botão **"Editar"** que habilita edição dos campos (descrição, categoria, cliente, forma, valor, vencimento, data emissão)
- [ ] Botão **"Salvar"** que persiste as alterações via `saveContasReceber()` e atualiza audit
- [ ] Botão **"Fechar"** para voltar à listagem
- [ ] Campos de status, automação, cobrança e auditoria são read-only (não editáveis)

## Technical Notes

### Padrão de checkbox (espelhar CP da Story 4.44)
```javascript
var _selectedContaReceberIds = new Set();

function toggleSelectAllContasReceber() {
  const checked = document.getElementById("cr-select-all").checked;
  document.querySelectorAll(".cr-check").forEach(cb => { cb.checked = checked; });
  atualizarSelecaoContasReceber();
}

function atualizarSelecaoContasReceber() {
  _selectedContaReceberIds.clear();
  document.querySelectorAll(".cr-check:checked").forEach(cb => { _selectedContaReceberIds.add(cb.value); });
  const bar = document.getElementById("cr-bulk-actions");
  bar.classList.toggle("hidden", _selectedContaReceberIds.size === 0);
}
```
