# Story 2.11 — Polish e Melhorias de UX

**Epic:** 2 — Resolucao de Debito Tecnico
**Sprint:** 5 (Semana 10-12)
**Prioridade:** P4 — BAIXO
**Estimativa:** 42 horas
**Debitos resolvidos:** UX-07, UX-08, UX-09, UX-10, UX-14, UX-15, UX-16, UX-17, UX-22, UX-24, D-16, D-17, D-20, SYS-10

---

## Descricao

Implementar melhorias de usabilidade e polish final: loading states, error boundaries, PWA completa do entregador, undo basico, validacao inline de formularios, animacoes de transicao, e outras melhorias incrementais que elevam a qualidade da experiencia.

---

## Criterios de Aceitacao

- [ ] Loading spinner global visivel durante operacoes SGD (scan, submit)
- [ ] Error boundaries com UI amigavel para falhas de rede e API
- [ ] PWA entregador completa: manifest, icons, SW registrado, installable
- [ ] Undo basico funcional (ultimas 5 acoes, timer 10 segundos)
- [ ] Validacao inline em formularios (feedback em tempo real)
- [ ] Animacoes de transicao suaves entre abas (CSS transitions)
- [ ] Confirmacao antes de acoes destrutivas (dialog modal)
- [ ] Modal customizado para credenciais SGD (substituindo prompt())
- [ ] Dados demo removidos do gdp-entregador.html
- [ ] Feedback haptico (Vibration API) no app entregador em acoes criticas
- [ ] Emails pessoais anonimizados em dados commitados
- [ ] Map em memoria para lookups frequentes (indices)

---

## Tarefas

### Loading States e Error Handling

- [ ] Criar componente `<ce-loading>` — spinner/overlay para operacoes async
- [ ] Integrar loading em: scan SGD, submit proposta, sync pedidos
- [ ] Criar componente `<ce-error>` — error boundary com retry
- [ ] Integrar error boundary em: falha de rede, API timeout, parsing error
- [ ] Substituir `alert()` por toast ou error component

### PWA Entregador

- [ ] Criar `manifest-entregador.json` com metadados corretos
- [ ] Criar icons (192px e 512px) para o app
- [ ] Registrar Service Worker no HTML: `navigator.serviceWorker.register()`
- [ ] Configurar SW para cache de dados offline (pedidos, entregas)
- [ ] Testar install prompt no Chrome Android
- [ ] Implementar Vibration API para confirmacao de entrega e assinatura
- [ ] Remover dados demo hardcoded, substituir por estado vazio + fetch

### Melhorias de Formularios

- [ ] Criar funcao `validateField(input, rules)` — validacao em tempo real
- [ ] Adicionar indicadores visuais de campo obrigatorio (*)
- [ ] Adicionar feedback inline: borda verde (valido) / vermelha (invalido)
- [ ] Aplicar em: login, credenciais SGD, banco de precos, pre-orcamento
- [ ] Substituir `prompt()` por modal customizado para credenciais SGD

### Undo e Confirmacao

- [ ] Criar modulo `undo-stack.js` — pilha de acoes reversiveis
- [ ] Implementar undo para: aprovar/recusar pre-orcamento, editar banco de precos
- [ ] Mostrar toast com botao "Desfazer" por 10 segundos
- [ ] Implementar dialog de confirmacao antes de: limpar banco, recusar proposta

### Animacoes e Polish

- [ ] Adicionar CSS transitions entre abas (opacity + transform)
- [ ] Adicionar transition em modals (fade + scale)
- [ ] Respeitar `prefers-reduced-motion` — desativar animacoes se usuario preferir

### Dados e Performance

- [ ] Anonimizar emails pessoais em `lariucci-arp-2025.json` (manter dominio, anonimizar usuario)
- [ ] Implementar Map em memoria para lookups de escolas e municipios (D-17)
- [ ] Avaliar viabilidade de ARP dinamica (D-16) — scraping CatalogoMobile
- [ ] Se viavel, implementar scraping basico do CatalogoMobile para atualizar ARP
