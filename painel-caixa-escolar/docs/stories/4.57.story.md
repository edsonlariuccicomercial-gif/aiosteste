## Story

**As a** gestor do sistema GDP,
**I want** que o extrato bancário importado apareça corretamente na conciliação e que o PDF da NF-e autorizada chegue com dados preenchidos no email,
**so that** a conciliação funcione conforme o modelo e os clientes recebam a nota fiscal correta.

---

## Contexto / Histórico

Stories 4.55 e 4.56 tentaram resolver estes bugs mas falharam por:
1. **Extrato:** A migração de `extratoId` roda dentro de `renderConciliacao()`, que chama `saveConciliacao()` + `saveExtratos()` a cada render — causando loop de re-save e stats inconsistentes. A migração precisa rodar UMA VEZ no boot.
2. **PDF:** `gerarDanfeHtmlCompleto()` tenta `transmissao.parsed.emitente` que NÃO existe (parsed só tem protocolo/cStat). O fallback para `preview.emitente` deveria funcionar, mas `nf.sefaz.preview` pode não estar sendo preservado. Os dados estão em `nf.sefaz.preview` (linha 938) e NUNCA em `transmissao.parsed`.

## Acceptance Criteria

### AC-1 — Migração extratoId no boot (NÃO no render)

**Arquivo:** `gdp-core.js`

- [ ] Mover a migração de itens órfãos (sem `extratoId`) de `renderConciliacao()` para `initializeData()` (boot sequence, ~linha 800)
- [ ] A migração deve rodar UMA VEZ após `loadConciliacao()` e `loadExtratos()` no boot
- [ ] Após migração, chamar `saveConciliacao()` para persistir — nunca mais precisará migrar
- [ ] REMOVER a migração de dentro de `renderConciliacao()` — render NÃO deve modificar dados
- [ ] REMOVER `saveExtratos()` de dentro de `renderConciliacao()` — render é somente leitura
- [ ] O recalculo de stats deve ser feito em `atualizarExtratoStats()` apenas, NÃO em `renderConciliacao()`
- [ ] `renderConciliacao()` deve apenas LER os dados e renderizar HTML — zero saves

**Lógica da migração (no boot):**
```javascript
// Após carregar conciliação e extratos no initializeData():
const _concItems = loadConciliacao();
const _extratos = loadExtratos();
const _orphans = _concItems.filter(i => !i.extratoId);
if (_orphans.length > 0 && _extratos.length > 0) {
  const target = _extratos[0]; // vincular ao primeiro (mais antigo)
  _orphans.forEach(i => { i.extratoId = target.id; });
  saveConciliacao(_concItems);
  // Recalcular stats
  _extratos.forEach(ext => {
    const extItems = _concItems.filter(i => i.extratoId === ext.id);
    ext.conciliados = extItems.filter(i => i.conciliado).length;
    ext.total = extItems.length;
  });
  saveExtratos(_extratos);
  gdpLog('[boot] Migrated ' + _orphans.length + ' orphan conciliacao items to extrato ' + target.id);
}
```

**Refactor de `renderConciliacao()`:**
- Carregar extratos e items (READ ONLY)
- Renderizar tabela de extratos
- Se algum extrato isOpen, renderizar detalhes filtrados
- NÃO chamar `saveConciliacao()` nem `saveExtratos()`

**Refactor de `atualizarExtratoStats()`:**
- Remover migração de órfãos (já feita no boot)
- Apenas recalcular conciliados/total por extrato e salvar

### AC-2 — PDF DANFE com dados reais do emitente/destinatário

**Arquivo:** `gdp-notas-fiscais.js` — `gerarDanfeHtmlCompleto()` (linha ~1591)

- [ ] Reverter a tentativa de usar `transmissao.parsed` — NÃO tem emitente/destinatario
- [ ] Usar SEMPRE `nf.sefaz.preview.emitente` e `nf.sefaz.preview.destinatario` como fonte primária
- [ ] Estes dados são populados na linha 938: `invoice.sefaz.preview = data.payload` (durante preparação do XML)
- [ ] Se `preview` estiver vazio, usar dados de `nf.cliente` como fallback para destinatário
- [ ] Verificar que `transmitirHomologacaoNota()` (linha 1219) NÃO sobrescreve `nf.sefaz.preview`
- [ ] Após transmissão autorizada, preservar `nf.sefaz.preview` explicitamente

**Código correto para gerarDanfeHtmlCompleto:**
```javascript
function gerarDanfeHtmlCompleto(nf) {
  if (!nf) return "";
  const sefazData = nf.sefaz || {};
  // Preview contém emitente/destinatario (set na linha 938 durante preparação XML)
  const previewData = sefazData.preview || {};
  const emit = previewData.emitente || {};
  const dest = previewData.destinatario || nf.cliente || {};
  // ... resto usa emit e dest normalmente
}
```

- [ ] Garantir que após `transmitirHomologacaoNota()` (linha 1219), o preview é preservado adicionando: `if (!nf.sefaz.preview && result.preview) nf.sefaz.preview = result.preview;`
