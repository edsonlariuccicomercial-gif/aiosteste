## Bug Report

| Campo | Detalhe |
|-------|---------|
| **Erro** | Excluir extrato apagava todos os dados do Caixa |
| **Severidade** | P0 — Critica — perda de dados financeiros |
| **Ocorrencias** | 2x no mesmo dia (13:57 e 13:21 UTC) |
| **Causa raiz** | Itens de conciliacao SAO os dados do Caixa (loadConciliacao() e fonte de verdade). Excluir extrato removia esses itens. |

### Historico de tentativas de fix

1. **Tentativa 1 (Story 4.68):** `if (remaining.length === 0) saveConciliacao([])` — apagava tudo quando ultimo extrato era excluido
2. **Tentativa 2 (Story 4.68):** Filtrar por `extratoId` — falhou porque TODOS os itens tinham o mesmo extratoId do extrato excluido
3. **Fix definitivo (Story 4.69):** Excluir extrato NUNCA toca nos itens de conciliacao — apenas remove o registro da lista

### Solucao definitiva

A funcao `excluirExtratosSelecionados()` agora:
- Remove apenas o registro do extrato de `gdp.extratos.v1`
- NAO chama `saveConciliacao()` em nenhuma circunstancia
- Itens de conciliacao ficam orfaos (sem extrato pai) mas preservados no Caixa
