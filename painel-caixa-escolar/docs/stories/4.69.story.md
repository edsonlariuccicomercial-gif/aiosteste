## Story

**I want** que ao excluir um extrato bancario, apenas os lancamentos conciliados permanecam no Caixa e lancamentos nao conciliados sejam removidos, sem que o sync restaure dados excluidos,

## Bug Report — Historico completo de tentativas

### Problema original
Excluir extrato apagava TODOS os dados do Caixa (loadConciliacao() e a fonte de verdade do Caixa).

### Tentativa 1 — FALHOU
`if (remaining.length === 0) saveConciliacao([])` — apagava tudo quando ultimo extrato era excluido.

### Tentativa 2 — FALHOU
Filtrar por `deletedExtIds.has(item.extratoId)` — falhou porque TODOS os itens tinham o mesmo extratoId do extrato excluido.

### Tentativa 3 — FALHOU
Excluir extrato NUNCA toca nos itens de conciliacao — itens nao conciliados ficavam como fantasmas no Caixa.

### Tentativa 4 — FALHOU
Filtrar itens por `deletedExtIds` da exclusao atual — itens orfaos de exclusoes anteriores escapavam do filtro.

### Tentativa 5 — FALHOU (sync)
Delete tracking implementado mas: (a) app-sync.js delKeyMap nao tinha gdp.extratos.v1, e (b) forcarSyncCompleto() apagava gdp.conciliacao.deleted.v1 antes do sync.

### Fix DEFINITIVO (commit 5561eaf) — FUNCIONOU

Logica final da `excluirExtratosSelecionados()`:

```javascript
// Verificar contra lista de extratos RESTANTES (nao excluidos)
const remainingExtIds = new Set(remaining.map(e => e.id));
const kept = allItems.filter(item => {
  if (item.conciliado === true) return true;           // conciliado = sempre manter
  if (item.extratoId && remainingExtIds.has(item.extratoId)) return true; // tem extrato ativo = manter
  return false;                                         // fantasma = remover
});
```

**Regra simples — NUNCA ALTERAR:**
| Condicao | Resultado |
|----------|-----------|
| Conciliado | SEMPRE fica no Caixa |
| Nao conciliado + extrato ativo | Fica |
| Nao conciliado + sem extrato ou extrato excluido | REMOVIDO (fantasma) |
