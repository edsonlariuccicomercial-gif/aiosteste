## Story

**As a** fornecedor que importa contratos com dezenas de itens,
**I want** que todos os produtos e o cliente vinculado sejam automaticamente cadastrados no Olist/Tiny no momento da importação,
**so that** quando eu criar pedidos, os SKUs e dados do cliente já estejam corretos no ERP e as notas fiscais saiam sem divergência.

---

## Design Decision

**Princípio: "Cadastra primeiro, usa depois"**

- Ao importar/criar contrato → batch register de todos os itens no Tiny automaticamente
- Ao vincular escola ao contrato → cadastrar cliente no Olist se ainda não existe
- Banco de Produtos ERP passa a ser cache read-only (populado pelo retorno do Tiny, não pelo usuário)
- Elimina necessidade de sync manual e propagação de SKU em múltiplas funções

## Acceptance Criteria

### AC-1: Auto-cadastro de itens no Tiny ao criar contrato
- [ ] Após qualquer fluxo de criação de contrato (import mapa, import docx/pdf, import cronograma, criação manual), disparar automaticamente o envio dos itens ao Tiny via `/api/tiny-produtos`
- [ ] Usar a mesma lógica de `enviarTiny()` mas em background (sem modal de preview NCM)
- [ ] Exibir toast de progresso: "Cadastrando X itens no ERP..." → "X itens cadastrados no ERP, Y SKUs obtidos"
- [ ] Se o endpoint falhar (offline, erro de API), salvar flag `pendingTinySync: true` no contrato para retry posterior
- [ ] NÃO bloquear a criação do contrato — o cadastro no ERP é assíncrono

### AC-2: SKUs do Tiny salvos automaticamente no contrato + Banco
- [ ] Quando o Tiny retorna os SKUs (response da API), salvar `item.sku = response.sku` em cada item do contrato
- [ ] Propagar cada SKU ao Banco de Produtos via `adicionarAoBancoProdutos(item)` (já implementado na 4.8)
- [ ] Chamar `saveContratos()` após todos os SKUs serem aplicados
- [ ] Logar no console: `[ERP Auto-Sync] Contrato ${id}: ${count} SKUs obtidos`

### AC-3: Retry de cadastro pendente
- [ ] Ao abrir detalhes de um contrato (`abrirContrato`): se `c.pendingTinySync === true`, exibir banner amarelo: "⚠️ Itens pendentes de cadastro no ERP" com botão "Cadastrar Agora"
- [ ] O botão dispara o mesmo fluxo do AC-1
- [ ] Ao abrir GDP Contratos (boot): verificar se há contratos com `pendingTinySync: true` e exibir badge no KPI ou toast informativo
- [ ] Após cadastro bem-sucedido, remover flag `pendingTinySync`

### AC-4: Auto-cadastro de cliente no Olist ao vincular escola
- [ ] Ao vincular escola ao contrato (`vincularEscolaContrato` → salvar): para cada escola recém-vinculada, verificar se já tem campo `olistClienteId`
- [ ] Se não tem: disparar POST ao endpoint de cadastro de clientes no Olist (nova rota ou adaptar existente) com os dados: nome, cnpj, indicador_contribuinte: 9, ie, cep, cidade, uf, logradouro, bairro, numero, telefone, email
- [ ] Salvar `olistClienteId` retornado no objeto do usuario
- [ ] Se falhar: não bloquear vinculação, logar warning e permitir retry manual
- [ ] Em `enviarPedidoOlist`: se `cliente.olistClienteId` existe, enviar junto no payload

### AC-5: Indicador visual de status ERP nos itens do contrato
- [ ] Na tabela de itens do contrato (detalhes): exibir badge ao lado de cada item:
  - `🟢` SKU presente (cadastrado no ERP)
  - `🟡` Sem SKU (pendente de cadastro)
- [ ] No header do contrato: exibir contagem "X/Y itens com SKU" ao lado do saldo

## Tasks / Subtasks

### Task 1: Função de auto-cadastro batch no Tiny (AC-1, AC-2)

- [ ] 1.1 Criar função `autoCadastrarTiny(contratoId)`:
  ```javascript
  async function autoCadastrarTiny(contratoId) {
    const c = contratos.find(x => x.id === contratoId);
    if (!c || !c.itens || c.itens.length === 0) return;

    showToast(`Cadastrando ${c.itens.length} itens no ERP...`, 3000);

    const itens = c.itens.map(item => ({
      num: item.num,
      descricao: item.descricao,
      descricaoCompleta: item.descricao,
      unidade: item.unidade,
      precoUnitario: item.precoUnitario,
      ncm: item.ncm || (findNcmLocal(item.descricao) || {}).ncm || '',
      sku: item.sku || ''
    }));

    try {
      const resp = await fetch("/api/tiny-produtos", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({ itens, contractId: contratoId, action: "cadastrar" })
      });
      const data = await resp.json();

      if (data.success && data.results) {
        let skuCount = 0;
        data.results.forEach(r => {
          if (r.sku) {
            const item = c.itens.find(it => it.num === r.num);
            if (item) {
              item.sku = r.sku;
              skuCount++;
              adicionarAoBancoProdutos(item);
            }
          }
        });
        delete c.pendingTinySync;
        saveContratos();
        console.log(`[ERP Auto-Sync] Contrato ${contratoId}: ${skuCount} SKUs obtidos`);
        showToast(`${skuCount}/${c.itens.length} itens cadastrados no ERP com SKU`, 4000);
      } else {
        c.pendingTinySync = true;
        saveContratos();
        showToast("Erro ao cadastrar no ERP. Retry disponível nos detalhes do contrato.", 5000);
      }
    } catch(err) {
      c.pendingTinySync = true;
      saveContratos();
      console.warn("[ERP Auto-Sync] Falha:", err.message);
      showToast("ERP offline. Itens serão cadastrados quando disponível.", 4000);
    }
  }
  ```

- [ ] 3.1 Criar função `autoCadastrarClienteOlist(usuario)`:

  ```javascript
  async function autoCadastrarClienteOlist(usuario) {
    if (usuario.olistClienteId) return; // já cadastrado
    try {
      const resp = await fetch("/api/olist/order", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify({
          action: "cadastrar-cliente",
          cliente: {
            nome: usuario.nome,
            cnpj: usuario.cnpj || '',
            indicador_contribuinte: '9',
            ie: usuario.ie || 'ISENTO',
            cep: usuario.cep || '',
            cidade: usuario.municipio || '',
            uf: usuario.uf || 'MG',
            logradouro: usuario.logradouro || '',
            bairro: usuario.bairro || '',
            numero: usuario.numero || '',
            telefone: usuario.telefone || '',
            email: usuario.email || ''
          }
        })
      });
      const data = await resp.json();
      if (data.success && data.clienteId) {
        usuario.olistClienteId = data.clienteId;
        saveUsuarios();
        showToast(`Cliente "${usuario.nome.slice(0,30)}" cadastrado no Olist`);
      }
    } catch(err) {
      console.warn("[Olist] Falha ao cadastrar cliente:", err.message);
    }
  }
  ```
