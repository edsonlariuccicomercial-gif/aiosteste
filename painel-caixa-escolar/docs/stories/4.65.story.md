## Root Cause (Analise @analyst)

### Bug 1 — Exclusao conciliados ainda falha

**Hipotese:** As 7 regras da Story 4.64 foram implementadas mas alguma pode ter regredido ou ha um cenario nao coberto. Necessario audit completo.

**7 invariantes a auditar:**
1. `syncToCloud` bloqueia arrays puros para conciliacao/extratos (app-sync.js:222-224, gdp-core.js:393-395)
2. `syncFromCloud` converte arrays puros para formato wrapped (app-sync.js:186-188)
3. Conciliacao usa timestamp, NAO isSharedKey (app-sync.js:150-154)
4. `saveConciliacao()` gera IDs e usa formato wrapped (gdp-core.js:1994-2002)
5. `excluirCaixaLancamentos()` rastreia IDs deletados (gdp-pedidos.js:1934-1961)
6. `DELETED_KEY_MAP` inclui conciliacao (gdp-api.js:143, app-sync.js:168)
7. Forcar Sync limpa `gdp.conciliacao.deleted.v1` (gdp-core.js:430)

### Bug 2 — Pedidos revertendo status (NOVO)

**Causa raiz confirmada:** Race condition em `alterarStatusPedido()` (gdp-pedidos.js:2314-2323):

```
Usuario muda status para "faturado"
→ p.status = "faturado" (memoria)
→ savePedidos() dispara (ASYNC, sem await — 3 caminhos paralelos)
→ renderAll() executa imediatamente ✓
→ 30s depois: syncFromCloud() carrega dados do Supabase
→ Se cloud tem dados antigos (status="em_aberto") → sobrescreve local
→ renderAll() chamado de novo → pedido volta para "em_aberto" ✗
```

**3 caminhos async de savePedidos() (gdp-core.js:750-761):**
1. `window.gdpApi[table].saveAll(items)` — Supabase direto
2. `cloudSave(key, wrapped)` — legacy cloud (sync_data)
3. `schedulCloudSync()` — timer 30s

**Diferenca para conciliacao (que funciona):**
- Conciliacao tem protecao por timestamp (nao isSharedKey)
- Conciliacao tem delete tracking dedicado
- Conciliacao bloqueia formato legacy
- Pedidos NAO tem nenhuma dessas protecoes contra race condition

### Diagrama do race condition (pedidos)

```
[T=0s] Usuario: alterarStatusPedido("PED-001", "faturado")
  → p.status = "faturado" (memoria)
  → savePedidos() → saveWrappedArray() dispara:
    Path A: gdpApi.pedidos.saveAll([...]) → Supabase (async, ~200ms)
    Path B: cloudSave(key, wrapped) → sync_data (async, ~300ms)
    Path C: schedulCloudSync() → agenda sync em 30s
  → renderAll() → pedido aparece em "faturado" ✓

[T=0.5s] Path A completa → Supabase atualizado ✓

[T=28s] syncFromCloud() dispara (timer anterior)
  → Le sync_data do Supabase (que AINDA pode ter dados antigos de Path B anterior)
  → cloudTime > localTime? Se sim → sobrescreve localStorage
  → renderAll() → pedido volta para "em_aberto" ✗
```
