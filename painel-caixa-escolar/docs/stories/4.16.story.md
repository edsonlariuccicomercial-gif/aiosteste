## Story

**As a** operador que cria contratos a partir de mapas importados,
**I want** uma tela de comparacao entre os itens do contrato e os produtos do banco antes de finalizar,
**so that** cada item do contrato ja tenha SKU, NCM e unidade corretos do Tiny, evitando divergencias na emissao de NF-e.

---

## Contexto / Motivacao

Com o banco de produtos limpo e sincronizado com o Tiny (Story 4.15), o proximo passo e garantir que ao criar um contrato, os itens importados do mapa sejam vinculados aos produtos corretos do banco. Isso elimina o problema de itens com descricoes ligeiramente diferentes (ex: "ARROZ TIPO 1 5KG" vs "ARROZ T1 5KG") que geram cadastros duplicados ou SKUs errados no Tiny.

## Acceptance Criteria

### AC-1: Tela de Sincronizacao na Criacao de Contrato

- [x] Ao criar um novo contrato (apos importar mapa e antes de inserir itens), abrir automaticamente uma tela/modal de "Sincronizacao de Itens"
- [x] A tela mostra uma tabela com 2 colunas principais:
  - **Coluna Esquerda:** Item do contrato (descricao, unidade, preco do mapa importado)
  - **Coluna Direita:** Produto similar encontrado no Banco de Produtos (descricao, SKU, NCM, unidade)
- [x] O matching e feito por similaridade de nome (normalizado: sem acentos, uppercase, keywords):
  - Match exato: highlight verde
  - Match parcial (>60% similaridade): highlight amarelo com opcao de selecionar
  - Sem match: highlight vermelho com opcao de buscar manualmente ou criar novo
- [x] Para cada item, o usuario pode:
  - **Vincular:** Aceitar o match sugerido — o item do contrato herda SKU, NCM e unidade do banco
  - **Buscar:** Digitar para buscar outro produto no banco
  - **Criar:** Abrir modal de novo produto (Story 4.15 AC-3) para cadastrar e vincular
  - **Ignorar:** Manter o item do contrato como esta (sem vinculo ao banco)
- [x] Botao "Sincronizar Selecionados" aplica todos os vinculos aceitos aos itens do contrato
- [x] Botao "Pular" fecha a tela sem alterar nada (fluxo antigo)
- [x] Apos sincronizacao, os itens do contrato ja terao SKU, NCM e unidade corretos do Tiny

### Task 2: Algoritmo de Similaridade

- [x] 2.1 Implementar `calcularSimilaridade(strA, strB)`:
  ```javascript
  function calcularSimilaridade(a, b) {
    const normalize = s => s.normalize('NFD').replace(/[\u0300-\u036f]/g,'').toUpperCase().trim();
    const keywords = s => normalize(s).split(/\s+/).filter(w => w.length >= 3);
    const kwA = keywords(a), kwB = keywords(b);
    if (normalize(a) === normalize(b)) return { score: 100, matchType: 'exato' };
    const common = kwA.filter(w => kwB.includes(w)).length;
    const total = Math.max(kwA.length, kwB.length, 1);
    const score = Math.round(common / total * 100);
    return { score, matchType: score >= 60 ? 'parcial' : 'sem-match' };
  }
  ```
- [x] 2.2 Para cada item do contrato, buscar o melhor match no banco (score mais alto) e ate 3 alternativas parciais
