## Story

**As a** operador que atende escolas de múltiplas regionais (SREs),
**I want** que o sistema suporte SRE Uberlândia e SRE Passos além da SRE Uberaba,
**so that** eu possa varrer orçamentos e fazer pré-orçamentos para todas as regionais onde atuo, com capacidade de filtrar por SRE na visualização.

---

## Contexto / Motivação

Atualmente o sistema só suporta SRE Uberaba: o arquivo `data/sre-uberaba.json` contém a lista de escolas e a função `varrerSgd()` está hardcoded para varrer apenas esta SRE.

A expansão para SRE Uberlândia e SRE Passos requer:

1. **Dados**: criar arquivos JSON com a lista de escolas de cada SRE (códigos e nomes conforme a SEE-MG). Os dados precisam ser obtidos em https://www.educacao.mg.gov.br/ — os arquivos serão criados com dados placeholder e o usuário deve preenchê-los com os dados reais.
2. **Lógica de varredura**: `varrerSgd()` deve iterar sobre TODAS as SREs configuradas, não apenas Uberaba.
3. **Filtro na UI**: a aba Oportunidades deve ter um dropdown "Filtrar por SRE" que filtra os orçamentos exibidos.
4. **KPIs e Radar**: exibir dados agrupados por SRE quando mais de uma SRE tiver dados.

Cada orçamento já possui um campo `sre` que identifica a SRE de origem — este campo é a base do filtro.

## Acceptance Criteria

### AC-1: Arquivos de dados das novas SREs

- [ ] `data/sre-uberlandia.json` criado com estrutura idêntica ao `data/sre-uberaba.json`
- [ ] `data/sre-passos.json` criado com estrutura idêntica ao `data/sre-uberaba.json`
- [ ] Ambos os arquivos contêm placeholder de ao menos 5 escolas com a nota de que os dados reais devem ser obtidos em https://www.educacao.mg.gov.br/
- [ ] O formato de cada escola é: `{ "codigo": "...", "nome": "...", "municipio": "..." }`

### AC-2: Configuração multi-SRE em `app.js`

- [ ] `sreData` deixa de ser um objeto único e passa a ser um array de configurações:
  ```
  [ { id: "uberaba", nome: "SRE Uberaba", arquivo: "sre-uberaba.json" },
    { id: "uberlandia", nome: "SRE Uberlândia", arquivo: "sre-uberlandia.json" },
    { id: "passos", nome: "SRE Passos", arquivo: "sre-passos.json" } ]
  ```
- [ ] As escolas de cada SRE são carregadas dinamicamente ao inicializar
- [ ] Código legado que referenciava `sreData` como objeto único é atualizado sem quebrar

### AC-3: `varrerSgd()` suporta múltiplas SREs

- [ ] `varrerSgd()` itera sobre cada SRE configurada e varre os orçamentos de todas
- [ ] O campo `sre` de cada orçamento capturado recebe o ID da SRE correspondente
- [ ] A varredura pode ser iniciada para todas as SREs de uma vez (botão "Varrer Todas") ou individualmente por SRE
- [ ] Progress/feedback indica qual SRE está sendo varrida no momento

### AC-4: Filtro por SRE na aba Oportunidades

- [ ] Dropdown "SRE" adicionado na área de filtros da aba Oportunidades
- [ ] Opções: "Todas", "SRE Uberaba", "SRE Uberlândia", "SRE Passos"
- [ ] Filtrar por SRE restringe os orçamentos exibidos àquela regional
- [ ] Filtro persiste na sessão (não precisa persistir no localStorage)
- [ ] Filtro funciona em combinação com os outros filtros existentes (município, status, etc.)

### AC-5: KPIs e Radar por SRE

- [ ] Se orçamentos de mais de uma SRE estiverem carregados, os KPIs principais mostram totais globais
- [ ] Um breakdown secundário exibe contagem por SRE (ex: "Uberaba: 12 | Uberlândia: 8 | Passos: 4")
- [ ] O Radar de Oportunidades pode ser filtrado pelo mesmo dropdown de SRE

### AC-6: Sem regressão na SRE Uberaba

- [ ] Todos os fluxos que funcionavam antes (varredura Uberaba, pré-orçamentos, envio SGD) continuam funcionando
- [ ] Dados já salvos com SRE Uberaba são reconhecidos corretamente

```json
{
  "sre": "SRE Uberlândia",
  "id": "uberlandia",
  "uf": "MG",
  "municipioSede": "Uberlândia",
  "escolas": [
    { "codigo": "1234567", "nome": "EE PLACEHOLDER ESCOLA 1", "municipio": "Uberlândia" },
    { "codigo": "1234568", "nome": "EE PLACEHOLDER ESCOLA 2", "municipio": "Uberlândia" },
    { "codigo": "1234569", "nome": "EE PLACEHOLDER ESCOLA 3", "municipio": "Araguari" },
    { "codigo": "1234570", "nome": "EE PLACEHOLDER ESCOLA 4", "municipio": "Monte Alegre de Minas" },
    { "codigo": "1234571", "nome": "EE PLACEHOLDER ESCOLA 5", "municipio": "Tupaciguara" }
  ],
  "_nota": "DADOS PLACEHOLDER — preencher com dados reais de https://www.educacao.mg.gov.br/regionais/"
}
```

### Configuração multi-SRE (app.js)

```javascript
const SRE_CONFIGS = [
  { id: 'uberaba',    nome: 'SRE Uberaba',     arquivo: 'data/sre-uberaba.json' },
  { id: 'uberlandia', nome: 'SRE Uberlândia',  arquivo: 'data/sre-uberlandia.json' },
  { id: 'passos',     nome: 'SRE Passos',      arquivo: 'data/sre-passos.json' }
];
```

### Varredura multi-SRE (app.js)

```javascript
async function varrerTodasSres() {
  for (const sreConfig of SRE_CONFIGS) {
    atualizarStatus(`Varrendo ${sreConfig.nome}...`);
    await varrerSgdParaSre(sreConfig);
  }
  atualizarStatus('Varredura completa.');
}
```

### Filtro na renderização

```javascript
function getOrcamentosFiltrados() {
  const sreFiltro = document.getElementById('filtro-sre')?.value || 'todas';
  return orcamentos.filter(orc => {
    if (sreFiltro !== 'todas' && orc.sre !== sreFiltro) return false;
    // demais filtros existentes...
    return true;
  });
}
```

### Fonte dos dados das escolas

- https://www.educacao.mg.gov.br/regionais/sre-uberlandia/
- https://www.educacao.mg.gov.br/regionais/sre-passos/

O usuário deve consultar esses URLs e preencher os arquivos JSON com os códigos e nomes reais das escolas antes de usar as novas SREs em produção.
