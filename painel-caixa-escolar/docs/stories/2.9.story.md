# Story 2.9 — Acessibilidade WCAG 2.1 AA

**Epic:** 2 — Resolucao de Debito Tecnico
**Sprint:** 4 (Semana 8-9)
**Prioridade:** P2 — ALTO
**Estimativa:** 32 horas
**Debitos resolvidos:** UX-04, UX-20, UX-21

---

## Descricao

Implementar conformidade WCAG 2.1 Nivel AA em todas as 7 paginas do sistema. Isso e uma **obrigacao legal** para sistemas educacionais publicos brasileiros (Lei 13.146/2015, Decreto 5.296/2004, eMAG 3.1). O sistema atualmente tem zero suporte a acessibilidade — nenhum atributo aria-*, nenhum role semantico, nenhum focus management.

---

## Criterios de Aceitacao

- [ ] Lighthouse Accessibility score >= 90 em todas as 7 paginas
- [ ] Todos os componentes customizados tem roles semanticos (tab, dialog, alert, etc.)
- [ ] Todos os controles interativos tem `aria-label` ou `aria-labelledby`
- [ ] Todos os inputs tem `<label>` associado (for/id)
- [ ] Todos os modals tem focus trap e `aria-modal="true"`
- [ ] Skip links implementados em todas as paginas
- [ ] Navegacao por teclado funcional em todas as paginas (Tab, Enter, Escape, Arrow keys)
- [ ] Contraste minimo 4.5:1 para texto normal em todas as paginas
- [ ] Contraste minimo 3:1 para texto grande e elementos graficos
- [ ] Alt text em todas as imagens e icones informativos
- [ ] Toast notifications usam `role="alert"` com `aria-live="polite"`
- [ ] Tabelas tem headers marcados com `<th>` e `scope`
- [ ] Tabelas tem versao responsiva (card view) para mobile

## Tarefas

### Fundacao

- [ ] Instalar axe-core para auditoria automatizada
- [ ] Executar audit baseline em todas as 7 paginas (registrar score inicial)
- [ ] Criar checklist WCAG AA por pagina

### index.html (Painel Fornecedor)

- [ ] Adicionar skip link para conteudo principal
- [ ] Adicionar `role="tablist"`, `role="tab"`, `role="tabpanel"` nas abas
- [ ] Adicionar `aria-selected` na aba ativa
- [ ] Adicionar `aria-controls` e `aria-labelledby` nas tab panels
- [ ] Associar labels aos filtros (escola, municipio, grupo, status, texto)
- [ ] Modal banco de precos: `role="dialog"`, `aria-modal="true"`, focus trap
- [ ] Modal import: `role="dialog"`, `aria-modal="true"`, focus trap
- [ ] Toast: `role="alert"`, `aria-live="polite"`
- [ ] Tabela de orcamentos: `scope="col"` nos headers, `aria-sort` se ordenavel
- [ ] KPI cards: `role="status"` ou `aria-label` descritivo

### Paginas GDP

- [ ] gdp-dashboard.html: skip link, cards com `aria-label`, KPIs semanticos
- [ ] gdp-contratos.html: upload zone com `aria-label`, progress bar com `aria-valuenow`
- [ ] gdp-gestao.html: pipeline com `role="list"` e `role="listitem"`, checklist semantica
- [ ] gdp-portal.html: catalogo com `aria-label` nos produtos, carrinho com `aria-live`
- [ ] gdp-entregador.html: delivery cards com `role="button"` e `tabindex="0"`, canvas com alternativa textual

### Navegacao por Teclado

- [ ] Implementar focus visible (`:focus-visible`) em todos os interativos
- [ ] Implementar focus trap em todos os modals
- [ ] Tecla Escape fecha modals e menus
- [ ] Arrow keys navegam entre tabs
- [ ] Enter/Space ativa botoes e links

### Contraste e Visual

- [ ] Verificar e corrigir contraste de `--muted` sobre `--bg` (ajustar se necessario)
- [ ] Verificar contraste de badges e status indicators
- [ ] Adicionar `prefers-reduced-motion` para animacoes
- [ ] Testar com modo alto contraste do Windows

### Tabelas Responsivas

- [ ] Criar CSS para tabela responsiva (card view em <= 768px)
- [ ] Aplicar em tabela de orcamentos (index.html)
- [ ] Aplicar em tabelas de contratos (gdp-contratos.html)
- [ ] Aplicar em tabelas de pedidos (gdp-gestao.html)

### Verificacao

- [ ] Executar Lighthouse a11y em todas as 7 paginas — score >= 90
- [ ] Testar com screen reader (NVDA ou VoiceOver)
- [ ] Testar navegacao completa apenas com teclado
- [ ] Testar com modo alto contraste
