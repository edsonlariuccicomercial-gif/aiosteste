## Story

**As a** operador de prospecção que monitora oportunidades no Radar,
**I want** poder descartar processos (orçamentos) que não tenho interesse de participar,
**so that** minha área de trabalho fique limpa e eu consiga focar apenas nos processos relevantes.

---

## Contexto / Motivação

A aba "Oportunidades — SRE Uberaba" lista TODOS os orçamentos varridos do SGD. Com a expansão da cobertura (Story 4.18), o volume de processos aumentou. O operador precisa de uma forma rápida de descartar processos que não são do interesse da empresa (itens fora do catálogo, valores muito baixos, prazos inviáveis, etc.) sem perdê-los permanentemente — apenas ocultá-los da visualização principal.

### Conceito existente similar

O sistema já tem `isGrupoExcluido()` que marca **grupos inteiros** como "Fora do escopo" via `perfil.config.gruposExcluidos`. Aqui queremos exclusão **individual** por orçamento, controlada pelo operador.

## Acceptance Criteria

### AC-1: Botão de descartar individual na tabela

- [ ] Cada linha da tabela de orçamentos com status "aberto" deve ter um botão de descartar (ícone X ou lixeira) na coluna de ação
- [ ] O botão NÃO deve aparecer em linhas com pré-orçamento já criado (status aprovado/pendente/enviado)
- [ ] Ao clicar, exibir confirmação rápida (ex: `confirm()`) antes de descartar
- [ ] Após descartar, a linha deve sumir da tabela imediatamente (sem reload)

### AC-2: Persistência dos descartados

- [ ] Os IDs dos processos descartados devem ser salvos no `localStorage` com chave `caixaescolar.descartados`
- [ ] O array deve ser um Set de IDs de orçamentos descartados
- [ ] Os dados do orçamento descartado NÃO devem ser apagados do array `orcamentos` — apenas marcados/filtrados
- [ ] Os descartados devem persistir entre sessões (localStorage) e sincronizar com cloud se cloudSync ativo

### AC-3: Filtro para ocultar descartados

- [ ] A função `filteredOrcamentos()` deve excluir processos descartados por padrão
- [ ] O filtro de status deve ganhar uma opção "Descartados" para ver os processos removidos
- [ ] Quando o filtro "Descartados" está ativo, exibir os processos descartados com visual diferenciado (ex: opacidade reduzida, badge "Descartado")

### AC-4: Restaurar processo descartado

- [ ] Na visualização de descartados, cada linha deve ter um botão "Restaurar"
- [ ] Ao restaurar, o processo volta a aparecer na lista principal
- [ ] O ID é removido do array de descartados no localStorage

### AC-5: Descarte em lote

- [ ] Os checkboxes existentes (batch selection) devem permitir descarte em lote
- [ ] Adicionar botão "Descartar Selecionados" na barra de ações em lote (`batch-bar`)
- [ ] Confirmação antes de descartar em lote: "Descartar X processos?"

### AC-6: KPIs atualizados

- [ ] Os KPIs do Radar (abertos, urgentes, etc.) NÃO devem contar processos descartados
- [ ] O toast/resumo da varredura SGD deve informar quantos processos descartados já estavam no sistema

### Padrão de persistência (igual a preOrcamentos)

```javascript
const DESCARTADOS_KEY = "caixaescolar.descartados";
let descartados = new Set();

function loadDescartados() {
  try {
    const raw = localStorage.getItem(DESCARTADOS_KEY);
    descartados = new Set(raw ? JSON.parse(raw) : []);
  } catch (_) { descartados = new Set(); }
}

function saveDescartados() {
  localStorage.setItem(DESCARTADOS_KEY, JSON.stringify([...descartados]));
}

function isDescartado(id) { return descartados.has(id); }

function descartarOrcamento(id) {
  descartados.add(id);
  saveDescartados();
}

function restaurarOrcamento(id) {
  descartados.delete(id);
  saveDescartados();
}
```

### Modificação em `filteredOrcamentos()`

```javascript
// Adicionar no início da chain de filtros:
.filter((o) => {
  if (status === "descartados") return isDescartado(o.id);

```
