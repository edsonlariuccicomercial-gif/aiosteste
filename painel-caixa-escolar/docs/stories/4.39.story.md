## Story

**As a** operador que monitora oportunidades no Radar,
**I want** que editais prestes a vencer tenham destaque visual (cor de alerta) e que processos já enviados para pré-orçamento saiam da minha tela,
**so that** eu foque apenas no que precisa de ação imediata.

---

## Contexto / Motivação

O Radar mostra todos os orçamentos juntos. Editais com prazo de 1-3 dias se misturam com os de 30 dias, e processos que já têm pré-orçamento continuam ocupando espaço visualmente. O operador precisa de dois comportamentos:

1. **Alerta visual por prazo:** destaque vermelho para prazo ≤ 3 dias, amarelo para 4-7 dias, opacidade reduzida para vencidos
2. **Ocultação de pré-orçados:** por padrão, processos que já têm pré-orçamento (pendente/aprovado/enviado/ganho) devem sumir da view, com opção de reexibir via filtro

## Dev Notes

### Cálculo de prazo

```javascript
function calcDaysTo(prazoStr) {
  if (!prazoStr) return null;
  const diff = new Date(prazoStr) - new Date();
  return Math.floor(diff / (1000 * 60 * 60 * 24));
}
```

### Aplicação de classe por prazo

```javascript
function getRowClassByPrazo(daysTo) {
  if (daysTo === null) return '';
  if (daysTo < 0)  return 'row-vencido';
  if (daysTo <= 3) return 'row-urgente';
  if (daysTo <= 7) return 'row-vencendo';
  return '';
}
```

### Filtro de pré-orçados

```javascript
function filteredOrcamentos() {
  const statusFiltro = document.getElementById('filtro-status')?.value;
  const incluirPreOrcados = statusFiltro === 'todos' || statusFiltro === 'com-pre-orcamento';

  return orcamentos.filter(o => {
    const temPreOrcamento = !!(preOrcamentos && preOrcamentos[o.id]);
    if (statusFiltro === 'com-pre-orcamento') return temPreOrcamento;
    if (!incluirPreOrcados && temPreOrcamento) return false;
    // ... restante dos filtros existentes
  });
}
```
