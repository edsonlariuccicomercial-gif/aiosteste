# Story 5.3 — Auth middleware nas Vercel Functions

**Epic:** 5 (Technical Debt v2)
**Sprint:** 1 (Segurança)
**Status:** Draft
**Agente:** @dev
**Débitos:** TD-A5
**Estimativa:** 4-8h
**Depende de:** 5.1

---

## Objetivo

Adicionar verificação de JWT (Supabase Auth) em todas as Vercel Functions que manipulam dados sensíveis.

## Acceptance Criteria

- [x] AC1: Criar middleware `api/lib/auth.js` que valida JWT do header Authorization
- [x] AC2: Aplicar middleware em: gdp-integrations, pedidos, produtos, fornecedores, movimentacoes, estoque
- [ ] AC3: Endpoints públicos (caixa-proxy, send-order-email) devem ter rate limiting ao invés de auth
- [x] AC4: Retornar 401 Unauthorized com mensagem clara quando token inválido
- [ ] AC5: Testes para middleware de auth

## Arquivos Impactados

- `api/lib/auth.js` (novo)
- `api/gdp-integrations.js`, `api/sync-pedidos.js`, `api/sync-entregas.js`, etc. (adicionar middleware)
