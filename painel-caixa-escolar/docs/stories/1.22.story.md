# Story 1.22 - Controle de Concorrencia no Refresh do Dashboard

## Status
Done

## Objetivo
Evitar chamadas concorrentes de refresh (timer + clique manual) para manter consistencia visual e reduzir risco de race condition.

## Criterios de Aceite

- [x] Refresh ignora nova execucao enquanto uma atualizacao estiver em andamento.
- [x] Botao `Atualizar agora` fica desabilitado durante refresh.
- [x] Indicador exibe estado `Atualizando...` durante ciclo de recarga.
- [x] Fluxo continua funcional no auto-refresh periodico.

## Tarefas

- [x] Adicionar flag de lock de refresh.
- [x] Atualizar botao para estado disabled/loading.
- [x] Ajustar indicador de status durante atualizacao.
- [x] Validar sintaxe e registrar evidencias.

## Definition of Done

- [x] Criterios de aceite validados.
- [x] Story atualizada com evidencias reais.
- [x] File list atualizada.

## Evidencias

- Refresh manual agora respeita lock e nao dispara chamadas simultaneas.
- Botao muda para estado desabilitado enquanto dados estao sendo recarregados.
