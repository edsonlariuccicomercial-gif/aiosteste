## Story

**As a** operador que analisa orçamentos na aba Oportunidades,
**I want** poder editar o texto do campo "Objeto" diretamente na tabela, resumindo descrições longas,
**so that** a visualização fique mais limpa e eu consiga identificar rapidamente o propósito de cada orçamento sem ler textos enormes.

---

## Contexto / Motivação

O campo "Objeto" dos orçamentos SGD contém a descrição oficial do processo licitatório, que muitas vezes é um texto longo e burocrático (ex: "AQUISIÇÃO DE GÊNEROS ALIMENTÍCIOS PARA ATENDER AS NECESSIDADES DA UNIDADE ESCOLAR DURANTE O ANO LETIVO DE 2025, CONFORME ANEXO I DO EDITAL"). Isso torna a tabela difícil de escanear.

O operador precisa de uma forma rápida de criar um resumo/apelido interno sem perder o texto original do SGD. A edição deve ser:

- **Inline**: sem abrir modais separados — clicar no texto para editar diretamente
- **Persistente**: salvo no localStorage junto ao orçamento
- **Não destrutiva**: o texto original (`objetoOriginal`) é preservado sempre
- **Reversível**: botão ou duplo-clique para restaurar o original

## Acceptance Criteria

### AC-1: Edição inline do campo Objeto

- [x] Na tabela de orçamentos (aba Oportunidades), a célula da coluna "Objeto" exibe um ícone de lápis (lapiz) ao hover
- [x] Ao clicar na célula (ou no ícone), o texto se transforma em um `<textarea>` editável in-place
- [x] O textarea é pré-preenchido com o texto atual (editado ou original)
- [x] Pressionar Enter ou clicar fora (`blur`) salva a edição
- [x] Pressionar Escape cancela e restaura o texto anterior sem salvar

### AC-2: Persistência no localStorage

- [x] Ao salvar, o campo `objeto` do orçamento é atualizado para o texto editado
- [x] O campo `objetoOriginal` é criado (se ainda não existir) com o valor do objeto antes da primeira edição
- [x] O array de orçamentos é salvo no localStorage com a chave correta (`caixaescolar.orcamentos` ou equivalente)
- [x] Recarregar a página preserva o texto editado

### AC-3: Indicador visual de objeto editado

- [x] Células com objeto editado exibem um indicador visual discreto (ex: asterisco, badge "editado", ou cor diferente)
- [x] Tooltip no indicador mostra o texto original: "Original: ..."

### AC-4: Restaurar objeto original

- [x] Um botão ou link "Restaurar original" fica visível na célula quando o objeto foi editado
- [x] Clicar em "Restaurar original" define `objeto = objetoOriginal` e remove a flag de editado
- [x] O campo `objetoOriginal` é removido (ou mantido) após restauração — opcional, mas o campo `objeto` volta ao original

### AC-5: Sem quebra de funcionalidades existentes

- [x] A edição do objeto não afeta outros campos do orçamento (escola, SRE, valor, itens, etc.)
- [x] O orçamento ainda funciona normalmente em pré-orçamento e envio SGD após edição do objeto
- [x] Filtros e ordenações da tabela continuam funcionando com o texto editado

## Dev Notes

### Estrutura do objeto orçamento após edição

```javascript
// Antes da primeira edição:
{ id: "...", objeto: "AQUISIÇÃO DE GÊNEROS ALIMENTÍCIOS...", escola: "...", ... }

// Após edição:
{
  id: "...",
  objeto: "Gêneros Alimentícios 2025",       // texto editado pelo operador
  objetoOriginal: "AQUISIÇÃO DE GÊNEROS ALIMENTÍCIOS...",  // preservado
  escola: "...",
  ...
}
```

### Renderização da célula (exemplo)

```javascript
function renderCelulaObjeto(orc) {
  const editado = !!orc.objetoOriginal;
  const indicador = editado
    ? `<span class="badge-editado" title="Original: ${orc.objetoOriginal}">✎</span>
       <button onclick="restaurarObjetoOriginal('${orc.id}')" class="btn-restaurar">↩</button>`
    : '';
  return `
    <td class="cel-objeto ${editado ? 'editado' : ''}"
        onclick="ativarEdicaoObjeto(this, '${orc.id}')">
      <span class="objeto-texto">${orc.objeto}</span>
      <span class="icone-editar">✏</span>
      ${indicador}
    </td>`;
}
```
