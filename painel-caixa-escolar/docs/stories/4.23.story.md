## Story

**As a** operador que cria contratos por lista de produtos,  
**I want** importar PDFs textuais e PDFs escaneados sem erro de carregamento da biblioteca,  
**so that** eu consiga usar arquivos PDF no fluxo de `Contratos` mesmo quando o arquivo vier como imagem digitalizada.

---

## Contexto / Motivacao

O usuario reportou na producao que, ao tentar importar um PDF na sessao `Contratos`, a tela mostra `Erro ao ler PDF: 404`.

O fluxo atual depende de `pdf.js` via CDN no HTML principal:

- `pdf.min.js`
- `pdf.worker.min.js`

O erro observado indica que o problema acontece antes ou durante o bootstrap da biblioteca, nao necessariamente no parser de itens em si. Isso bloqueia completamente o uso de PDFs, mesmo quando o arquivo e valido.

O usuario deixou explicito que o fluxo tambem precisa ler PDF escaneado. Portanto, esta story cobre:

- restauracao do bootstrap do PDF.js
- importacao de PDF textual
- tentativa de importacao via OCR quando o PDF vier escaneado/imagem

## Acceptance Criteria

### AC-1: Bootstrap de PDF deixa de quebrar o fluxo

- [ ] Ao abrir `gdp-contratos.html` e importar um PDF, o usuario nao deve receber `Erro ao ler PDF: 404`
- [ ] O fluxo nao pode depender exclusivamente do `pdf.worker.min.js` remoto atual para continuar operando

### AC-2: PDF textual gera pre-visualizacao verificavel

- [ ] Dado um PDF textual com tabela de itens, a importacao deve gerar pre-visualizacao com ao menos `descricao`, `quantidade`, `unidade` e total quando houver valores
- [ ] O mesmo PDF textual nao deve cair no fallback de OCR quando o texto extraivel for suficiente

### AC-3: PDF escaneado usa o pipeline OCR ja existente no produto

- [ ] Quando `parsePDF()` detectar texto insuficiente, o sistema deve usar o fallback ja existente `parsePdfViaOCR()`
- [ ] O OCR deve continuar usando a infraestrutura ja existente do dashboard para imagens, via endpoint `dashboard/api/ai-parse-price.js`
- [ ] Dado um PDF escaneado com itens legiveis, o fluxo deve montar a pre-visualizacao da lista a partir da saida do OCR
- [ ] A tela deve informar ao usuario que a importacao foi processada via OCR

### AC-4: Falha funcional clara

- [ ] Dado um PDF sem texto extraivel e sem OCR aproveitavel, a mensagem de erro deve ser funcional, sem mencionar `404`
- [ ] Em falha, a tela nao pode manter pre-visualizacao residual da importacao anterior

### AC-5: Regressao objetiva

- [ ] Reimportar um CSV valido da story 4.22 deve continuar gerando a lista corretamente
- [ ] Importar `doc` ou `docx` com tabela valida deve continuar chamando `loadListaFromTables()` sem alteracao de fluxo

---

## Non-Goals

- Nao buscar perfeicao de OCR para qualquer layout arbitrario
- Nao reescrever o parser inteiro de PDF alem do necessario para restaurar o fluxo
- Nao alterar o modelo de dados do contrato

## Tasks / Subtasks

### Task 1: Diagnosticar a origem do 404

- [ ] 1.1 Revisar os scripts e configuracao de `pdf.js` no HTML
- [ ] 1.2 Confirmar se o `404` vem de `pdf.min.js`, `pdf.worker.min.js` ou de outro recurso encadeado
- [ ] 1.3 Definir estrategia de correcao resiliente para producao

### Task 2: Corrigir bootstrap do PDF.js

- [ ] 2.1 Eliminar a dependencia quebradica que gera `404` na producao
- [ ] 2.2 Garantir inicializacao segura do parser de PDF antes de `parsePDF()`
- [ ] 2.3 Garantir que falhas de biblioteca gerem mensagem funcional e nao erro cru de rede

### Task 3: Conectar o fallback OCR ao fluxo de lista

- [ ] 3.1 Detectar quando o PDF nao possui texto extraivel suficiente
- [ ] 3.2 Reutilizar `parsePdfViaOCR()` ou extrair dele o necessario sem duplicar pipeline
- [ ] 3.3 Adaptar a saida do OCR para o fluxo de lista de produtos
- [ ] 3.4 Sinalizar quando a importacao foi feita por OCR

### Task 4: Validar fluxo de importacao

- [ ] 4.1 Validar importacao de um fixture de PDF textual com sucesso
- [ ] 4.2 Validar importacao de um fixture de PDF escaneado com OCR
- [ ] 4.3 Validar erro funcional para PDF sem estrutura reconhecivel
- [ ] 4.4 Revalidar importacao por CSV/planilha apos a correcao
- [ ] 4.5 Revalidar importacao por `doc`/`docx`

- se a correcao depender de CDN externa instavel, o problema pode reaparecer
- OCR em PDF escaneado aumenta custo, tempo de processamento e variabilidade de resultado, especialmente em tabelas ruins
