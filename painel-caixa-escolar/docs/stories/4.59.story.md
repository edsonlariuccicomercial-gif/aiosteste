## Story

**so that** os clientes recebam a nota fiscal correta e legível por email.

## Contexto / Histórico

Teste Playwright comprovou:
- `gerarDanfeHtmlCompleto(nf)` gera HTML de 13.457 chars com TODOS os dados corretos (emitente, destinatário, itens, chave, protocolo)
- Screenshot do HTML renderizado no browser mostra DANFE perfeito
- Porém `gerarDanfePdfBase64(nf)` gera PDF de apenas 3KB (vazio)

**Causa raiz:** O container é criado com `position:absolute;left:-9999px` (offscreen). O `html2canvas` (usado pelo `html2pdf`) não captura elementos posicionados fora da viewport, resultando em canvas vazio → PDF vazio.

---

## Acceptance Criteria

### AC-1 — Corrigir gerarDanfePdfBase64 para gerar PDF com conteúdo

**Arquivo:** `gdp-notas-fiscais.js` — `gerarDanfePdfBase64()` (linha ~1580)

- [ ] Alterar posicionamento do container de `left:-9999px` para `position:fixed;left:0;top:0;z-index:-1;opacity:0;pointer-events:none`
- [ ] Isto mantém o container invisível para o usuário mas renderizável pelo html2canvas
- [ ] Se `opacity:0` não funcionar com html2canvas, usar alternativa: `position:fixed;left:0;top:0;z-index:99999` visível brevemente, gerar PDF, depois remover
- [ ] PDF gerado deve ter >= 20KB (um DANFE real tem 50-150KB)
- [ ] Validar que o PDF contém: dados emitente, destinatário, itens, chave de acesso, protocolo

**Código correto:**
```javascript
async function gerarDanfePdfBase64(nf) {
  const danfeHtml = gerarDanfeHtmlCompleto(nf);
  const container = document.createElement("div");
  // CRITICAL: html2canvas não captura elementos offscreen (left:-9999px)
  // Usar position:fixed visível mas sob z-index baixo
  container.style.cssText = "position:fixed;left:0;top:0;width:210mm;background:#fff;z-index:-1;pointer-events:none";
  container.innerHTML = danfeHtml;
  document.body.appendChild(container);
  try {
    await new Promise(r => setTimeout(r, 200)); // aguardar render
    const opt = { margin: [6, 6, 6, 6], filename: "DANFE_" + (nf.numero || nf.id) + ".pdf", image: { type: "jpeg", quality: 0.95 }, html2canvas: { scale: 2, useCORS: true }, jsPDF: { unit: "mm", format: "a4", orientation: "portrait" } };
    const pdfBlob = await html2pdf().set(opt).from(container).outputPdf("blob");
    const reader = new FileReader();
    return new Promise((resolve) => {
      reader.onloadend = () => resolve(reader.result.split(",")[1]);
      reader.readAsDataURL(pdfBlob);
    });
  } finally {
    document.body.removeChild(container);
  }
}
```

**Teste Playwright:**
1. Chamar `gerarDanfePdfBase64(nf)` para NF autorizada
2. Verificar que base64 length > 20000 (>15KB)
3. Decodificar e verificar que começa com %PDF-1

## Technical Notes

### Evidência do bug (Playwright):
- Container offscreen (`left:-9999px`): PDF = 4080 bytes base64 (~3KB) → VAZIO
- Container on-screen (`left:0;top:0`): PDF = 4080 bytes base64 (~3KB) → AINDA VAZIO (opacity issue?)
- HTML renderizado no browser (screenshot): DANFE PERFEITO com todos os dados

### html2canvas known issue:
`html2canvas` tem limitações conhecidas com elementos fora da viewport ou com `opacity:0`. A solução mais confiável é renderizar visível brevemente e usar `z-index:-1` para ficar atrás do conteúdo.

Se `z-index:-1` não funcionar, usar `overflow:hidden` no body temporariamente com o container visible.
