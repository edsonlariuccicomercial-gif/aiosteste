## Story

**As a** gestor do sistema GDP,
**I want** que as unidades dos produtos nos pedidos respeitem SEMPRE o cadastro da Central de Produtos, e que lançamentos excluídos no Caixa não reapareçam,
**so that** os pedidos tenham dados corretos e o Caixa reflita exclusões permanentes.

---

## Acceptance Criteria

### AC-1 — Remover parseUnidadeFromName do portal (eliminar inferência)

**Arquivo:** `gdp-portal.html`

O fix anterior (linhas 976-978 e 1019-1020) não resolveu porque o portal da escola pode ter cache antigo. Além disso, a função `parseUnidadeFromName()` é perigosa — ela adivinha unidade pelo nome e causa erros.

- [ ] Remover completamente a função `parseUnidadeFromName()` do `gdp-portal.html`
- [ ] Nas linhas 976-978: substituir por `const unidade = rawUnidade || 'UN';` (usa cadastro ou default UN)
- [ ] Nas linhas 1019-1020: substituir por `const unidade = rawUnidade || 'UN';`
- [ ] Sem inferência de unidade pelo nome — NUNCA. Se não tem unidade cadastrada, é UN.
- [ ] Adicionar cache-busting no script tag do portal para forçar reload: `?v=462`

### AC-2 — Corrigir pedido PED-20260522-0807 no Supabase

- [ ] Atualizar unidade de "GR" para "UN" no pedido PED-20260522-0807 via Supabase
- [ ] Verificar se há outros pedidos recentes com unidade inferida incorretamente e corrigir

### AC-3 — Exclusão no Caixa persiste no cloud (não reaparece)

**Arquivo:** `gdp-pedidos.js` — `excluirCaixaLancamentos()`

O problema: `saveConciliacao()` chama `cloudSave()` mas o `syncToCloud()` roda depois (no visibilitychange) e reenvia dados antigos.

- [ ] Em `excluirCaixaLancamentos()`: após `saveConciliacao(remaining)`, desabilitar o próximo `syncToCloud` para a chave `gdp.conciliacao.v1`
- [ ] OU: adicionar um debounce/flag que impede `syncToCloud` de sobrescrever dados que acabaram de ser salvos pelo `cloudSave`
- [ ] OU (mais simples): no `syncToCloud()`, verificar se a chave já foi salva pelo `cloudSave` nos últimos 5 segundos e pular

### AC-4 — Cache-busting no portal da escola

**Arquivo:** `gdp-contratos.html` ou onde o portal é linkado

- [ ] Adicionar versão no link do portal: `gdp-portal.html?v=462`
- [ ] Garantir que scripts JS do portal carregam com cache-busting

## Technical Notes

### Bug unidade — cadeia completa verificada via Playwright:
- Central de Produtos: "Café Vasconcelos 500 gr" → `unidade_base: "UN"` ✓
- Portal cria pedido → `parseUnidadeFromName("Café Vasconcelos 500 gr")` → retorna `"GR"` ✗
- Supabase tabela pedidos: `unidade: "GR"` ✗
- Fix anterior no portal não pegou porque browser da escola tinha cache antigo

### Bug Caixa — cadeia verificada:
- Cloud `sync_data` gdp.conciliacao.v1 = 0 itens (limpo)
- localStorage do browser = 10 itens (persistem)
- `visibilitychange` → `syncToCloud()` reenvia 10 itens para o cloud
- Próximo `syncFromCloud()` restaura os 10 itens
