## Story

**As a** operador que cota produtos no Radar,
**I want** ver a unidade de cada item do orçamento (UN, KG, CX, PCT) diretamente na tabela,
**so that** eu não erre cotando como pacote algo que é kg.

---

## Contexto / Motivação

O operador está cotando errado porque não vê a unidade na tela do Radar. Pensa que é pacote de 500 mas na verdade é kg. A informação de unidade existe nos itens do orçamento (`o.itens[].unidade`) mas não é exibida na tabela principal.

Sem esta informação visível, o erro de cotação é previsível: o operador precifica por pacote quando deveria precificar por kg, gerando discrepância entre o proposto e o real.

## Acceptance Criteria

### AC-1: Exibir unidade na tabela do Radar

- [x] Na tabela de orçamentos do Radar, a coluna de itens/objeto inclui a unidade de cada item
- [x] A informação é extraída de `o.itens[].unidade`

### AC-2: Múltiplos itens com unidades distintas

- [x] Quando o orçamento tem múltiplos itens com unidades diferentes, são exibidas as unidades distintas separadas por vírgula (ex: "UN, KG, CX")
- [x] Unidades repetidas são deduplicadas (não mostrar "UN, UN, KG" — mostrar "UN, KG")

### AC-3: Orçamento com item único

- [x] Quando o orçamento tem apenas 1 item, a unidade é exibida diretamente (ex: "KG")

### AC-4: Orçamento sem itens

- [x] Se `o.itens` está vazio ou indefinido, exibir "—" no lugar da unidade

## Dev Notes

### Lógica de extração de unidades

```javascript
function getUnidadesOrcamento(orcamento) {
  if (!orcamento.itens || orcamento.itens.length === 0) return '—';
  const unidades = [...new Set(
    orcamento.itens.map(i => i.unidade).filter(Boolean)
  )];
  return unidades.length > 0 ? unidades.join(', ') : '—';
}
```

### Integração em renderOrcamentos()

```javascript
// Dentro do loop de renderização de cada linha:
const unidades = getUnidadesOrcamento(o);
// Adicionar na célula de itens/objeto ou em <td> próprio
```
