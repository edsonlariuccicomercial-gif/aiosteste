## Story

**As a** operador de prospecção que monitora oportunidades no Radar,
**I want** que a varredura SGD capture orçamentos de TODOS os 25 municípios da SRE Uberaba,
**so that** eu não perca cotações de escolas estaduais legítimas por falta de county IDs no whitelist.

---

## Contexto / Motivação

O Radar não está puxando cotações de escolas da SRE de Uberaba. A função `varrerSgd()` em `app.js` usa uma estratégia "county-first" para filtrar orçamentos, mas o mapa `sreCountyMap` só cobre **8 dos 25 municípios** da SRE Uberaba.

### Causa raiz identificada

1. **`sreCountyMap` incompleto** (`app.js:3428-3433`): Só tem IDs para Uberaba, Araxa, Sacramento, Iturama, Pirajuba, Santa Juliana, Frutal e Campos Altos. Faltam 17 municípios.

2. **Fallback rejeita matches válidos** (`app.js:3495-3501`): Quando o nome da escola bate com `sre-uberaba.json` mas o county ID não está no whitelist, o código **rejeita** a escola como "falso positivo" — quando na verdade é um match legítimo.

3. **networkId nulo na primeira chamada** (`app.js:3469`): `BrowserSgdClient.listBudgets()` é chamado com `networkId: null` no login inicial. Sem o header `x-network-being-managed-id`, a API pode retornar orçamentos de outra SRE ou dar 422.

### Municípios sem county ID (17 de 25)

Agua Comprida, Campo Florido, Carneirinho, Comendador Gomes, Conceicao das Alagoas, Conquista, Delta, Fronteira, Itapagipe, Limeira do Oeste, Pedrinopolis, Planura, Pratinha, Sao Francisco de Sales, Tapira, Uniao de Minas, Verissimo.

## Acceptance Criteria

### AC-1: Fallback por nome aceita escolas da SRE Uberaba

- [ ] Quando `findSreMatch()` retorna match positivo contra `sre-uberaba.json`, o orçamento deve ser **aceito** independente do county ID
- [ ] Log de debug deve indicar `via: "name-only"` para matches por nome sem county conhecido
- [ ] Zero escolas legítimas da SRE Uberaba sendo rejeitadas

### AC-2: Descoberta automática de county IDs faltantes

- [ ] Durante a varredura, quando um match por nome é aceito com county desconhecido, logar o county ID no console: `[Varrer] Novo county descoberto: {id} → {municipio}`
- [ ] Exibir resumo no console ao final: lista de county IDs novos para atualização futura do `sreCountyMap`

### AC-3: NetworkId garantido antes da varredura

- [ ] Antes de chamar `listBudgets()` no `varrerSgd()`, garantir que `BrowserSgdClient.networkId` está definido (fazer uma chamada `listBudgets(1, 1)` de warm-up se necessário)
- [ ] Header `x-network-being-managed-id` deve estar presente em TODAS as chamadas à API SGD

### AC-4: Validação no Radar

- [ ] Após varredura, o Radar deve exibir orçamentos de municípios que antes não apareciam (ex: Fronteira, Conceicao das Alagoas, Delta)
- [ ] O total de orçamentos aceitos deve ser >= ao total anterior (nunca perder cobertura)
- [ ] Console deve mostrar breakdown: `[Varrer] X county-match, Y name-match, Z total aceitos de W varridos`

### Mudança secundária (Task 2) — REVISADA

**Causa raiz real:** `BrowserSgdClient` NÃO tinha `getUser()` — diferente do `SgdClient` (server.js) que resolve `networkId` via `/auth/user` após login. Sem `networkId`, o warm-up `listBudgets(1,1)` falhava com 422 (header `x-network-being-managed-id` obrigatório).

```javascript
// ANTES — warm-up sem networkId → 422
await BrowserSgdClient.login();
if (!BrowserSgdClient.networkId) {
  await BrowserSgdClient.listBudgets(1, 1); // FALHA: API requer header
}

// DEPOIS — getUser() resolve networkId antes de qualquer chamada
await BrowserSgdClient.login();
if (!BrowserSgdClient.networkId) {
  try {
    await BrowserSgdClient.getUser(); // /auth/user → retorna idNetwork
  } catch (e) { /* fallback abaixo */ }
}
if (!BrowserSgdClient.networkId) {
  await BrowserSgdClient.listBudgets(1, 1); // fallback com warm-up
}
```

### Correções adicionais

1. **Proxy `get-user` action** — `api/sgd-proxy.js` agora suporta action `get-user` que chama `/auth/user`
2. **Login aceita 302** — `handleLogin` agora aceita status 3xx (redirect após set-cookie)
3. **Error handling melhorado** — mensagens específicas para 422 (networkId) e 401 (credenciais)
4. **maxDuration 60s** — `vercel.json` agora configura timeout para `sgd-proxy.js`
