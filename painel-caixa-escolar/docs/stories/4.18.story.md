## Story

**As a** operador de pos-licitacao,  
**I want** emitir nota fiscal diretamente do pedido dentro do GDP,  
**so that** o faturamento aconteca no proprio sistema sem depender de Tiny/Olist.

---

## Acceptance Criteria

### AC-1: Pedido possui base fiscal obrigatoria
- [ ] O pedido deve conter nome, CNPJ/CPF, IE/indicador, logradouro, numero, bairro, CEP, cidade e UF
- [ ] Cada item do pedido deve conter descricao, quantidade, unidade, preco unitario e NCM
- [ ] O sistema deve bloquear a emissao de NF quando houver campo fiscal obrigatorio ausente

### AC-2: Geracao de NF no pedido
- [ ] Cada pedido deve ter uma acao explicita `Gerar Nota Fiscal`
- [ ] Ao gerar, o sistema cria um registro de NF vinculado ao `pedido_id`
- [ ] A NF deve nascer com status `pendente_autorizacao`
- [ ] O pedido deve passar a exibir referencia da NF gerada

### AC-3: Rastreabilidade fiscal
- [ ] Toda NF deve manter vinculo com `pedido_id`, `contrato_id` e `cliente`
- [ ] O sistema deve impedir NF duplicada para o mesmo pedido, exceto fluxo futuro de reemissao controlada

### AC-4: Persistencia
- [ ] As notas fiscais devem ser persistidas em chave dedicada do GDP
- [ ] O fluxo deve sobreviver a reload/browser novo via mecanismo de sync ja existente
