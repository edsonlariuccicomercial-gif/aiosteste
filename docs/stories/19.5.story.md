## Story
**Como** stakeholder do caixa,
**Quero** que o saldo no banco reflita o valor real do extrato Inter (1.174,25),
**Para** que todos os PCs mostrem o saldo correto de forma definitiva.

## Contexto / Problema
Saldo base era 10.949,40. Após a conciliação a partir de 02/06/2026, o saldo real do banco Inter é **1.174,25** (confirmado pelo stakeholder como correto e conferido com o extrato real). Hoje o Supabase calcula **-3.494,29** porque há exclusões/ajustes feitos no PC do stakeholder que **não propagaram**.

Cálculo de referência:
- saldo_inicial: 10.949,40
- conciliações válidas atuais: entradas 14.946,92 − saídas 29.390,61 = saldo -3.494,29
- delta a remover (exclusões não propagadas): +4.668,54 para chegar a 1.174,25

**Fonte da verdade:** backup do localStorage do PC do stakeholder (já exportado antes de qualquer correção).

## Critérios de Aceitação
- **AC1:** *Given* a reconciliação aplicada, *When* qualquer PC abre o caixa, *Then* o saldo é 1.174,25 para todos.
- **AC2:** O saldo confere com o extrato bancário real do Inter (02/06 → hoje).
- **AC3:** Snapshot pré-reconciliação disponível para rollback.
- **AC4:** Validação aritmética: saldo_inicial (10.949,40) + soma conciliações válidas = 1.174,25.

## Escopo
- **IN:** comparar backup do PC × banco; identificar lançamentos a marcar `deleted_at`; aplicar soft-delete controlado; validar saldo.
- **OUT:** mudança de código (já feita na 19.2).

## Requisitos Funcionais
- **FR-19.5.1:** Comparar backup do PC (estado correto) com o banco; identificar lançamentos que devem ser marcados `deleted_at` (exclusões que não propagaram).
- **FR-19.5.2:** Aplicar soft-delete dos lançamentos identificados de forma controlada (snapshot antes).
- **FR-19.5.3:** Validar: saldo_inicial + soma conciliações válidas = 1.174,25.
