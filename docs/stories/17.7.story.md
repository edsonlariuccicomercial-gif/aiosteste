# Story 17.7 — Convergência de Cache no Boot (Fonte Única Confiável)

**Epic:** EPIC-17 — Sincronização do Caixa Multiusuário (FASE 1 — Persistência)
**Status:** Ready for Review
**Prioridade:** P0
**Risco:** MÉDIO | **Complexidade:** M
**Executor:** @dev
**Resolve:** FR-C — "dados somem sozinhos" / divergência histórica

## Story
**Como** usuário do caixa,
**Quero** que meu caixa sempre convirja para o estado real do servidor,
**Para** que os dados não se percam nem divirjam entre navegadores ao longo do tempo.

## Contexto / Problema
O estado do caixa evolui diferente por navegador porque:
- O merge de boot (`gdp-init.js` `_mergeTable`, ~L2895-2930) faz `[...mergedRemote, ...localOnly]` com `preferLocal < 5s` — itens local-only antigos podem ressuscitar e o cache nunca converge totalmente para o servidor.
- A SAFETY de `gdpApi...list` ("nunca sobrescrever cache com vazio") protege contra apagão, mas pode perpetuar um cache local divergente. (Causa-raiz: brief.)

## Escopo SIMPLIFICADO (decisão do stakeholder)
Como o reset (Fase 2) limpa o histórico, o foco é **garantir convergência dali pra frente**: novos lançamentos sincronizam e o cache converge para o Supabase. A complexidade de "não ressuscitar itens locais antigos" é mitigada pelo reset (que limpa o cache de todos) — manter como salvaguarda, não como foco principal.

## Critérios de Aceitação
- **AC1:** *Given* um navegador com cache divergente, *When* faz boot/reload, *Then* o caixa converge para o estado do Supabase (fonte única). (Pós-reset, todos os caches partem limpos.)
- **AC2:** *Given* um lançamento criado em um navegador e sincronizado, *When* outro navegador recarrega, *Then* vê o mesmo lançamento (persistência confirmada).
- **AC3:** *Given* a SAFETY anti-vazio, *When* o Supabase tem dados, *Then* o cache converge para eles (não mantém divergência permanente).
- **AC4 (não-regressão):** a proteção contra apagar tudo quando Supabase está offline é preservada.

## Escopo
- **IN:** lógica de convergência no boot; revisão do merge `mergedRemote+localOnly`.
- **OUT:** saldo inicial (17.5), tombstone (17.6).

## Definition of Done
- AC1–AC4 validados; um lançamento sobrevive a reload e aparece em outro navegador.

## Dev Notes (Dex — integração)
- `_mergeTable` (gdp-init.js): para a tabela `gdp.conciliacao.v1` (caixa), o Supabase passa a ser FONTE ÚNICA — `localOnly` forçado a `[]` (não ressuscita lançamentos órfãos/antigos que causavam a divergência histórica). Demais tabelas mantêm o comportamento anterior (sem regressão em pedidos/contratos/NFs).
- **SAFETY anti-vazio PRESERVADA** (AC4): se o Supabase retorna vazio/offline, `_mergeTable` retorna `false` antes do merge — não apaga o cache.
- Backfill de conciliação já existia no boot (`_tableMap`); esta mudança garante convergência sem lixo local.
