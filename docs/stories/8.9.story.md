# Story 8.9 — Populacao Sistematica de preco_historico — 6 Pontos de Insercao

**Epic:** Epic 8 — Consolidacao GDP e Fechamento de Gaps Operacionais
**Status:** Ready
**Priority:** P2
**Story Points:** 3

---

## Descricao

A tabela `preco_historico` no Supabase (migration 005) existe mas so e populada a partir de NFs de entrada (`gdp-init.js:1682-1683`). O operador nao consegue saber preco medio por cidade, evolucao de margem por SRE, ou quais produtos perde por preco.

Esta story adiciona insercoes em `preco_historico` em 5 pontos adicionais do fluxo operacional (alem do ponto ja existente), totalizando 6 fontes de dados:

1. Proposta enviada ao SGD (tipo `proposta`)
2. Resultado ganho (tipo `ganho`)
3. Resultado perdido (tipo `perdido`)
4. Contrato criado (tipo `contrato`)
5. NF saida emitida (tipo `nf_saida`)
6. NF entrada processada (tipo `nf_entrada`) — **JA EXISTE**

**Referencia SOP:** Etapa 14 — Serie Historica

---

## Criterios de Aceitacao

- [ ] AC-1: Proposta enviada ao SGD gera registro `tipo = 'proposta'` em `preco_historico` com SKU, valor, escola, SRE, custo_base, margem
- [ ] AC-2: Resultado ganho gera registro `tipo = 'ganho'` com SKU, valor praticado, escola, SRE
- [ ] AC-3: Resultado perdido gera registro `tipo = 'perdido'` com SKU, valor proposto, valor vencedor (metadata), escola, SRE

- [ ] AC-6: Todas as insercoes sao async e NAO bloqueiam a UI (NFR-G4-001)
- [ ] AC-7: Falha no Supabase NAO impede a operacao principal (fire-and-forget com log)
- [ ] AC-8: Campo `metadata` JSONB contem dados contextuais conforme FR-G4-002 (orcamento_id, municipio, grupo_despesa, marca, etc.)
- [ ] AC-9: Cada registro inclui `empresa_id` para RLS

## Notas Tecnicas

### Helper Central

Criar funcao `inserirPrecoHistorico()` reutilizavel:

```javascript
async function inserirPrecoHistorico({
  empresa_id,
  sku,
  escola,
  sre,
  tipo,          // 'proposta' | 'ganho' | 'perdido' | 'contrato' | 'nf_saida' | 'nf_entrada'
  valor,
  custo_base,
  margem_pct,
  fonte,
  metadata = {},
}) {
  try {
    const { error } = await supabase.from('preco_historico').insert({
      empresa_id,
      sku,
      escola,
      sre,
      tipo,
      valor,
      custo_base,
      margem_pct,
      fonte: fonte || `gdp-${tipo}`,
      metadata,
    });
    if (error) console.warn(`preco_historico insert (${tipo}):`, error);
  } catch (err) {
    console.error(`preco_historico insert falhou (${tipo}):`, err);
    // Fire-and-forget — nao impede operacao principal
  }
}
```

### Dados por Item

Cada item de um orcamento/contrato gera um registro SEPARADO em `preco_historico`. Se um orcamento tem 8 itens e e marcado como "ganho", sao 8 registros tipo `ganho`.

### ISS-3 (Critique)

O ponto de insercao exato em `salvarResultado()` deve ser APOS `alimentarBancoComResultado()` (~linha 199, nao 168), para nao interferir com a logica de resultado existente.

## Riscos

| Risco | Probabilidade | Mitigacao |
|-------|:-------------:|-----------|
| Volume alto de inserts em orcamentos com muitos itens | Media | Insercoes async; batch se necessario; monitorar com `supabase db size` |
| Dados de escola/SRE/municipio nao disponiveis em todos os pontos | Media | Campos opcionais; preencher o que estiver disponivel; metadata como fallback |
| Supabase free tier atinge limite de storage | Media | Monitorar tamanho; retention policy futura (>12 meses agregar) |
