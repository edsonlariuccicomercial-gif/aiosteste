# Story 8.21 — Melhorias DRE (estrutura completa) + Fluxo de Caixa (visualizacoes) + Fix impressao

- **Epic:** EPIC-8 — GDP Gaps
- **Status:** Ready
- **Complexity:** M (Medium)
- **Agent:** @dev
- **Depends on:** Story 8.20

## Descricao

Aprimorar o relatorio DRE para seguir a estrutura profissional completa (modelo Olist/ERP), adicionar opcoes de fonte de dados (NFs ou Pedidos) e custo (compra ou medio), corrigir impressao da DRE, e adicionar visualizacao por dia/semana/mes no Fluxo de Caixa.

## Acceptance Criteria

### AC1: DRE — Estrutura completa conforme modelo de referencia
- **Given** o usuario gera a DRE
- **Then** a tabela deve conter TODAS as linhas na seguinte ordem:
  1. **RECEITA BRUTA** (bold, destaque)
  2. Notas Fiscais (sub-item)
  3. Notas Servico (sub-item, valor 0,00 se nao houver dados)
  4. Quantidade
  5. Ticket medio
  6. **(-)DEDUCOES** (bold)
  7. Devolucoes
  8. Impostos (sub-header)
  9. ICMS, IR, IPI, ISS, INSS, CSLL, PIS, COFINS (sub-itens, valor 0,00 se nao discriminados)
  10. Estorno de impostos de devolucoes (sub-header)
  11. ICMS, IPI (sub-itens, valor 0,00)
  12. **(=)RECEITA LIQUIDA** (bold) = Receita Bruta - Deducoes
  13. **(-)CUSTOS** (bold)
  14. CMV (Custo das Mercadorias Vendidas)
  15. Estorno de CMV de devolucoes
  16. **(=)LUCRO BRUTO** (bold) = Receita Liquida - Custos
  17. **(-)DESPESAS OPERACIONAIS** (bold)
  18. (categorias dinamicas: Comissoes, Taxas e tarifas, Agua/luz, Telecomunicacao, Fornecedores, Combustivel, Salarios, Funcionario, Pejotizado, etc.)
  19. **(=)RESULTADO OPERACIONAL** (bold) = Lucro Bruto - Despesas Operacionais
  20. **(+)OUTRAS RECEITAS** (bold)
  21. (categorias dinamicas: Emprestimo, Vendas Internet, etc.)
  22. **(-)OUTRAS DESPESAS** (bold)
  23. (categorias dinamicas: Servicos gerais, Veiculos/transportes, Frete, Despesas de Viagem, etc.)
  24. **(-)TRIBUTOS** (bold)
  25. Impostos, taxas
  26. **(=)RESULTADO LIQUIDO** (bold)
  27. **(=)RESULTADO / LUCRO BRUTO** (percentual por mes)

### AC2: DRE — Opcao fonte de dados (NFs ou Pedidos)
- **Given** a tela de configuracao da DRE
- **Then** o dropdown "Baseado em" deve ter opcoes:
  - "Notas fiscais" (default) — usa `notasFiscais` para receita
  - "Pedidos" — usa `pedidos` (nao cancelados) para receita
- **Then** ao gerar, a receita bruta deve vir da fonte selecionada

### AC3: DRE — Opcao custo baseado em (compra ou medio)
- **Given** a tela de configuracao da DRE
- **Then** deve haver dropdown "Custo baseado em:" com opcoes:
  - "Custo da compra" — usa `custoBase` do produto na Central de Produtos
  - "Custo medio" — calcula media de `custosFornecedor[].valor` do produto
- **Then** o CMV deve ser calculado como: soma(qtd_vendida * custo_unitario) por mes
- **Then** se nenhum produto tem custo cadastrado, CMV fica 0,00

### AC4: DRE — Fix impressao
- **Given** o usuario clica em "Imprimir" no relatorio DRE
- **Then** a impressao deve:
  - Usar orientacao landscape (via CSS `@page { size: landscape }`)
  - Remover `position:sticky` das colunas
  - Usar fonte 9px para caber 14 colunas
  - Titulo centralizado: "Demonstracao do resultado do exercicio" + ano
  - Bordas e linhas visiveis para separar secoes
  - Valores alinhados a direita com formato brasileiro (pontos como milhar, virgula decimal)

### AC5: Fluxo de Caixa — Visualizacao por dia/semana/mes
- **Given** a tela de configuracao do Fluxo de Caixa
- **Then** deve haver seletor "Visualizacao:" com opcoes:
  - "Por dia" (default) — uma linha por data
  - "Por semana" — agrupa por semana do mes (Semana 1, 2, 3, 4, 5)
  - "Por mes" — agrupa por mes (mostra 12 meses do ano selecionado)
- **Then** ao gerar, as entradas/saidas devem ser agrupadas conforme a visualizacao
- **Then** saldo acumulado calculado progressivamente

## Scope

### IN
- Reestruturar linhas da DRE conforme modelo de referencia completo
- Adicionar dropdown "Baseado em" (NFs / Pedidos)
- Adicionar dropdown "Custo baseado em" (compra / medio)
- Calcular CMV quando dados de custo disponiveis
- Fix CSS impressao DRE (landscape, sem sticky, fonte ajustada)
- Adicionar linha RECEITA LIQUIDA e RESULTADO / LUCRO BRUTO %
- Adicionar linhas de impostos detalhados (zerados se sem dados)
- Seletor de visualizacao no Fluxo de Caixa (dia/semana/mes)
- Deploy em producao

### OUT
- Discriminacao real de impostos por NF (depende de integracao SEFAZ — fase futura)
- Graficos no Fluxo de Caixa (fase futura)

## Riscos
- CMV depende de `custoBase` estar populado nos produtos — mitigacao: mostrar 0,00 quando nao disponivel
- Impostos detalhados (ICMS, PIS, etc.) nao estao discriminados nas NFs do GDP — mitigacao: linhas existem com 0,00
