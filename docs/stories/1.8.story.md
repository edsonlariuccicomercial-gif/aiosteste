## Acceptance Criteria

### AC-1: Validação de itens antes de transmitir NF-e
- [ ] Em `gerarNotaFiscalPedido()`, antes de enviar à API, validar que `pedido.itens.length > 0`
- [ ] Exibir alert com contagem: "Gerando NF-e com X itens. Confirma?"
- [ ] Se pedido tem 0 itens, bloquear com mensagem de erro
- [ ] Log no console: quantidade de itens enviados à API

### AC-2: Numeração NF-e persistida no Supabase
- [ ] Ao gerar número (`getProximoNumeroNf`), salvar no Supabase key `gdp.nf-counter.v1`
- [ ] Ao carregar, buscar do Supabase primeiro, localStorage como fallback
- [ ] Garantir que dois browsers não gerem mesmo número (read-then-increment atômico)

### AC-3: Tabela completa de municípios de MG
- [ ] Carregar tabela IBGE de MG (~853 municípios) como JSON estático
- [ ] Resolver `cMun` do destinatário dinamicamente pela cidade+UF
- [ ] Se município não encontrado, usar fallback 3106200 (BH) e logar warning

### AC-4: Teste controlado end-to-end
- [ ] Criar pedido de teste com 5+ itens via Playwright
- [ ] Gerar preview → verificar que XML tem todos os itens
- [ ] Transmitir à SEFAZ → verificar cStat 100
- [ ] Cancelar nota de teste
- [ ] Documentar resultado
