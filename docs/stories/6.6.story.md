# Story 6.6 — Hook NF Saída + Contrato → Registro de Preço Real no Histórico

**Epic:** Epic 6 — Inteligência de Preços: Fechamento de Loops
**Status:** Done
**Priority:** P2
**Story Points:** 2

---

## Problema

Quando uma NF-e é emitida (saída) ou um contrato é criado/registrado, o preço real faturado/contratado NÃO é registrado na tabela `preco_historico` do Supabase. Isso significa que:

1. O histórico de preços não tem dados de "preço real faturado" — só tem propostas e custos
2. A margem real (preço NF saída - custo NF entrada) nunca é calculada automaticamente
3. Contratos criados via `criarContratoGdp()` registram itens com preços, mas não alimentam o banco de preços com tipo="contrato"

O `seedFromContratos()` no radar-matcher já lê contratos para popular o dicionário de equivalências, mas não registra o PREÇO do contrato no histórico.

## Scope

### IN
- Hook na emissão de NF-e: registrar cada item em `preco_historico` com tipo="nf_saida"
- Hook na criação de contrato GDP: registrar cada item com tipo="contrato"
- Calcular `margem_real` = (precoNF - custoBase) / precoNF quando possível

### OUT
- Alteração na lógica de emissão NF-e (já funciona)
- Alteração na criação de contratos (já funciona)
- Dashboard visual (futuro)

### DEPENDS ON
- **Story 6.2** — Tabela `preco_historico` deve existir em Supabase

---

## Acceptance Criteria

### AC-1: Hook NF Saída → preco_historico
- [ ] Após emissão bem-sucedida de NF-e (cStat=100), para cada item da nota:
  - Inserir em `preco_historico`: tipo="nf_saida", valor=valorUnitario, sku, escola
  - Calcular `margem_pct` se custoBase disponível no banco
  - metadata: {nf_numero, chave_acesso, cnpj_dest}
- [ ] Graceful degradation: se Supabase falhar, logar e continuar

### AC-2: Hook Contrato GDP → preco_historico
- [ ] Após `criarContratoGdp()` salvar contrato, para cada item:
  - Inserir em `preco_historico`: tipo="contrato", valor=precoUnitario, sku, escola
  - metadata: {contrato_id, edital, fornecedor}
- [ ] Também executar quando contrato é criado via `gerarContratoUnificado()`

### AC-3: Margem real calculada
- [ ] Quando tipo="nf_saida" e custoBase existe no banco para o SKU:
  - `margem_pct = ((valor - custo_base) / valor) * 100`
- [ ] Quando tipo="contrato":
  - Mesmo cálculo se custoBase disponível
- [ ] Se custoBase=0 ou não encontrado → margem_pct=null (não inventar)
