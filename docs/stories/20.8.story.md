# Story 20.8 — Centralizar prazo padrão (remover hardcode de 28 dias)

**Epic:** EPIC-20 — Ajustes Financeiro GDP
**Status:** Ready for Review
**Prioridade:** P2
**Risco:** BAIXO | **Complexidade:** S
**Onda:** 2
**Executor:** @dev
**Resolve:** FIN-5 — prazo padrão de 28 dias hardcoded em 6+ pontos; sempre é preciso trocar manualmente.
**ADR:** `docs/architecture/ADR-epic20-financas-config-vencimento.md`
**Depende de:** 20.6 (config `getFinancasConfig()` precisa existir)

## Story
**Como** dono do sistema,
**Quero** que o prazo padrão usado nos pedidos venha da config Finanças,
**Para** não ter que trocar 28→5 dias manualmente e para que cada empresa (SaaS) use seu próprio prazo.

## Contexto / Problema
O prazo padrão de 28 dias está hardcoded em 6+ pontos de `gdp-pedidos.js` (`:775, 964, 1096, 1171, 1178, 1429`) e em `gdp-notas-fiscais.js:823` (este último já tratado na 20.7). Centralizar a leitura na config Finanças.

## Decisões do ADR (obrigatórias)
- Usar `getFinancasConfig().prazoRecebimentoDias` como fonte do prazo padrão, com **constante de fallback única** por página (não reespalhar "5" como novo número mágico).
- Espelhar `getFinancasConfig()` em `gdp-pedidos.js` se ainda não presente (padrão multi-page).

## Critérios de Aceitação
- **AC1:** *Given* prazo configurado = 5, *When* crio um pedido sem alterar os dados de pagamento, *Then* a condição padrão usada é 5 dias (não 28).
- **AC2:** *Given* a config vazia, *When* o sistema lê o prazo, *Then* usa um fallback são (sem quebrar) via constante única.
- **AC3:** Nenhum dos pontos identificados mantém o `28`/`"28"` hardcoded como única fonte do prazo padrão.
- **AC4 (não-regressão):** o `<select>` de condição de pagamento continua exibindo as opções (28 dias permanece como uma opção válida selecionável, mas não como default fixo no código).

## Escopo
- **IN:** substituir os defaults `28`/`"28"` nos pontos de `gdp-pedidos.js` pela leitura de `getFinancasConfig().prazoRecebimentoDias` com fallback.
- **OUT:** UI de config (20.6); o ponto de NF (20.7).

## Requisitos Funcionais
- **FR-20.8.1:** Ponto único de leitura do prazo padrão a partir de `getFinancasConfig()` por página.
- **FR-20.8.2:** Substituir os defaults `28` nos pontos identificados, mantendo fallback seguro.
