# Story 4.73 — Cleanup dead code associação de produtos (Radar/Intel Preços)

- **Epic:** EPIC-4 — GDP Caixa Escolar
- **Status:** Ready
- **Complexity:** S (Small)
- **Agent:** @dev
- **Depends on:** None
- **Priority:** Medium (manutenibilidade — código morto confunde devs)

## Descrição

O módulo de associação de produtos possui código morto e campo legado que confundem a manutenção:
1. Bloco de modal hooking via MutationObserver em `pricing-intel.js:457-491` — nunca completado, não funciona
2. Campo `skuVinculado` em `app-results.js` é fallback legado do campo padrão `skuBanco` — duplicação desnecessária

**Identificado por @analyst** via análise estática do codebase.

## Acceptance Criteria

### AC1: Remover dead code modal hooking
- **Given** `pricing-intel.js` contém linhas 457-491 com MutationObserver incompleto
- **When** o código é removido
- **Then** nenhuma funcionalidade quebra (bloco nunca era executado)
- **And** o arquivo fica mais limpo e compreensível

### AC2: Consolidar skuVinculado → skuBanco
- **Given** `app-results.js` usa `skuVinculado` como fallback em 3 locais (linhas ~690, ~713, ~741)
- **When** as referências são consolidadas para usar apenas `skuBanco`
- **Then** o campo padrão `skuBanco` é usado uniformemente
- **And** leitura de dados existentes que tenham `skuVinculado` é migrada: `item.skuBanco || item.skuVinculado`

## Scope

### IN
- Remover bloco morto pricing-intel.js (modal hooking MutationObserver)
- Substituir `skuVinculado` por `skuBanco` em app-results.js (com fallback de leitura)

### OUT
- Mudanças no RadarMatcher (está OK)
- Mudanças no pendAssociar/renderPendentes (está OK)
- Migração de dados existentes no Supabase/localStorage
