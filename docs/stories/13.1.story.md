## Story

**Como** operador do Licit-AIX,
**Quero** configurar quais SREs de MG a varredura SGD deve cobrir,
**Para que** eu possa expandir minha cobertura para todas as 47 SREs do estado sem perder performance, filtrando apenas as de interesse.

## Dev Notes

### Arquivos a modificar:
- `app-state.js` — expandir `SRE_CONFIGS` para 47 SREs, adicionar `sresAtivas` ao perfil
- `app.js` — UI de configuração de SREs, indicador de progresso, cache 24h
- `app-sync.js` — persistir `sresAtivas` no cloud sync
- `index.html` — modal/seção de configuração de SREs

### Dados de referência:
- Lista completa das 47 SREs no SOP-INTEL-PRECOS-V2-002 §5.1
- Cada SRE precisa de: `{ id: "slug", nome: "Nome Completo", arquivo: "data/sre-slug.json" }`
- Os arquivos JSON de SRE (`data/sre-*.json`) precisam ser criados ou detectados dinamicamente da API SGD

### Cache strategy:
- Key: `radar.cache.sre.{sreId}` com timestamp
- Expiração: 24h (configurável)
- Invalidação: botão "Forçar Revarredura" limpa cache da SRE selecionada

### Compatibilidade:
- Operadores existentes que usam 3 SREs devem continuar funcionando sem configuração
- Default: as 3 SREs atuais (Uberaba, Uberlândia, Passos) pré-selecionadas
