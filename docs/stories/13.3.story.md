## Story

**Como** operador do Licit-AIX,
**Quero** que o batch pré-orçar me permita criar produtos na hora quando não há match, e que os botões de download payload sejam removidos,
**Para que** eu tenha celeridade no processo de cotação sem sair do fluxo, e a interface fique limpa sem features sem uso.

---

## Acceptance Criteria

### Criar Produto Inline
- [ ] **AC1:** Quando batch pré-orçar encontra item "sem_match", exibe modal de criação rápida
- [ ] **AC2:** Modal tem campos mínimos: Nome (pré-preenchido do SGD), Unidade (inferida), Categoria (do grupo)
- [ ] **AC3:** Ao salvar, produto é criado na `central_produtos` com flag `origem: "batch-pre-orcamento"`
- [ ] **AC4:** Produto recém-criado é automaticamente vinculado ao item SGD no pré-orçamento
- [ ] **AC5:** Fluxo continua para o próximo item sem match (loop)
- [ ] **AC6:** Resumo final do batch mostra: "X produtos criados, Y associados, Z sem custo"
- [ ] **AC7:** Opção "Pular" permite ignorar o item e continuar (não obriga a criar)

### Remover Payload Download
- [ ] **AC8:** Botão "Baixar Payload" removido da UI (aba Envio SGD, individual)
- [ ] **AC9:** Botão "Baixar Todos" removido da UI (aba Envio SGD, batch)
- [ ] **AC10:** Funções `sgdBaixarPayload()`, `sgdBaixarTodos()` removidas do JS
- [ ] **AC11:** Função `sgdBaixarTodosPdf()` mantida (se existir uso de PDF export)
- [ ] **AC12:** Fluxo de envio ao SGD inalterado: Aprovar → Enviar via API

## Dev Notes

### Criar Produto Inline — Arquitetura:

```
batchPreOrcar() loop:
  Para cada item sem match:
    1. Pausar loop
    2. Abrir modal "Criar Produto Rápido"
       ├─ Nome: [pré-preenchido do item.nome SGD]
       ├─ Unidade: [inferida de item.unidade ou "UN"]
       ├─ Categoria: [do orc.grupo]
       └─ [Salvar] [Pular]
    3a. Salvar → criar produto + vincular + continuar loop
    3b. Pular → deixar sem match + continuar loop
  Fim loop → resumo
```

### Arquivos a modificar:
- `app.js` — `batchPreOrcar()`: adicionar loop com modal para itens sem match
- `app.js` — criar `abrirCriarProdutoRapidoBatch(item, callback)` 
- `app-sgd-integration.js` — remover `sgdBaixarPayload()`, `sgdBaixarTodos()`
- `index.html` — remover botões de download payload, adicionar modal de criação rápida

### Produto criado inline — dados mínimos:
```javascript
{
  id: "PROD-" + Date.now() + "-" + random,
  nome: item.nome,           // do SGD
  unidade_base: item.unidade || "UN",
  categoria: orc.grupo || "Geral",
  origem: "batch-pre-orcamento",  // rastreabilidade
  ativo: true,
  criadoEm: new Date().toISOString()
}
```

### Nota sobre payload download:
- `sgdBaixarPayload()` já foi corrigido no commit d827748 para não mudar status
- Agora remover completamente — feature sem utilidade operacional confirmada pelo operador
- `downloadSgdPayload()` helper function pode ser mantida se usada em outros contextos
