# Story 4.75 — Fix email NF-e: PDF DANFE sombreado + remover HTML inline do corpo

- **Epic:** EPIC-4 — GDP Caixa Escolar
- **Status:** Done
- **Complexity:** S (Small)
- **Agent:** @dev
- **Depends on:** Story 4.74
- **Priority:** High (email chega ao cliente com DANFE ilegível)

## Descrição

2 problemas no email de confirmação de NF-e reportados pelo usuário:

1. **PDF DANFE sombreado/manchado** — O PDF anexo ao email é gerado por `gerarDanfePdfBase64()` que usa `gerarDanfeHtmlCompleto()` (versão compacta com CSS diferente). O resultado sai com sombras e manchas. O "Visualizar DANFE" (`abrirDanfeNotaFiscal()`) gera um HTML perfeito — o PDF deve ser idêntico a esse.

2. **HTML da DANFE embutido no corpo do email** — O `danfeHtml` completo (com `<style>` tags, flexbox, classes CSS) é injetado inline no email em `send-order-email.js:223-231`. Clientes de email (Gmail, Outlook) stripam `<style>` e não suportam flexbox, resultando em layout totalmente quebrado. A DANFE já vai como PDF em anexo — remover o HTML inline e manter só o resumo de texto.

## Acceptance Criteria

### AC1: PDF DANFE idêntico ao "Visualizar DANFE"
- **Given** uma NF-e autorizada com o botão "Enviar Email"
- **When** o email é enviado
- **Then** o PDF DANFE anexo é visualmente idêntico ao que se vê ao clicar "Visualizar DANFE"
- **And** o PDF tem fundo branco, sem sombras, sem manchas, sem artefatos de tema dark

### AC2: Remover HTML da DANFE do corpo do email
- **Given** uma NF-e com `danfeHtml` no payload
- **When** o servidor monta o HTML do email
- **Then** o bloco `danfeHtml` NÃO é incluído no corpo do email
- **And** o resumo de texto (protocolo, escola, itens, total, dados NF-e, chave de acesso) permanece intacto

### AC3: Email limpo conforme exemplo do usuário
- **Given** o email é enviado
- **When** o destinatário abre no Gmail/Outlook
- **Then** o corpo mostra apenas:
  - Header "GDP — Confirmacao de Pedido / Lariucci & Ribeiro Pereira"
  - "Pedido registrado com sucesso!" + Protocolo
  - Tabela escola (nome, CNPJ, responsável, SRE, data)
  - Tabela itens (produto, qtd, unit, subtotal) + TOTAL
  - Bloco NF-e (número, série, protocolo, valor, chave de acesso)
  - Link consulta SEFAZ
- **And** a DANFE vai SOMENTE como anexo PDF

## Análise Técnica (@analyst)

### Problema 1: PDF sombreado

**Causa raiz:** `gerarDanfePdfBase64()` (linha 1594) chama `gerarDanfeHtmlCompleto()` (linha 1640) — uma versão compacta do DANFE com CSS diferente (classes `.danfe-wrap`, `.bx`, `.row`, `.cell`, etc.). Essa versão renderiza com artefatos no iframe do `html2pdf.js`.

O "Visualizar DANFE" usa `abrirDanfeNotaFiscal()` (linha 1910) que gera um HTML mais completo e limpo com CSS dedicado (`@media print`, `@page`, classes `.wrap`, `.rec-body`, `.hdr-logo`, etc.) e abre em `window.open()`.

**Fix:** Extrair o HTML de `abrirDanfeNotaFiscal()` (linhas 1970-2100+) como string e usá-lo no iframe de `gerarDanfePdfBase64()` em vez de `gerarDanfeHtmlCompleto()`.

### Problema 2: HTML inline no email

**Causa raiz:** `send-order-email.js:223-231`:
```javascript
${nfe?.danfeHtml ? `<div>...${nfe.danfeHtml}...</div>` : ''}
```
O `danfeHtml` usa `<style>` tags, flexbox, classes CSS — nada suportado por clientes de email.

**Fix:** 
1. Client-side: remover `danfeHtml` do payload (`gdp-notas-fiscais.js:1547` e `2532`)
2. Server-side: remover bloco condicional `nfe?.danfeHtml` (`send-order-email.js:223-231`)
