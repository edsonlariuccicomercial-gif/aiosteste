# Story 8.2 — Migrar Dados do Banco de Precos para Central de Produtos

**Epic:** Epic 8 — Consolidacao GDP e Fechamento de Gaps Operacionais
**Status:** Ready
**Priority:** P1
**Story Points:** 2

---

## Descricao

Apos a unificacao do schema (Story 8.1), esta story implementa o script de migracao one-time que copia todos os dados de `caixaescolar.banco.v1` (Banco de Precos) para `gdp.produtos.v1` (Central de Produtos). A migracao executa automaticamente na inicializacao do sistema (`gdp-init.js`), faz matching por nome/SKU, e preserva o backup original por 30 dias.

**Referencia SOP:** Etapas 6-7 (Unificacao de repositorios de produto)

---

## Criterios de Aceitacao

- [ ] AC-1: Migracao automatica executa no boot (`gdp-init.js`) sem intervencao do usuario
- [ ] AC-2: Cada item de `caixaescolar.banco.v1` e pareado com produto em `gdp.produtos.v1` por nome/SKU
- [ ] AC-3: Se encontrar correspondencia: merge dos campos `custoBase`, `precoReferencia`, `custosFornecedor`, `concorrentes`, `propostas`, `historicoResultados`, `grupo` no produto Central
- [ ] AC-4: Se NAO encontrar correspondencia: cria novo produto na Central com todos os campos
- [ ] AC-5: Flag `localStorage.setItem('banco-precos-migrated.v1', 'true')` setada apos migracao bem-sucedida
- [ ] AC-6: Migracao e idempotente — executar 2x nao duplica produtos nem sobrescreve dados ja mergeados
- [ ] AC-7: Backup original `caixaescolar.banco.v1` NAO e removido (preservar por 30 dias conforme CON-G2-001)
- [ ] AC-8: Log de migracao no console com contadores: `{total, merged, created, skipped}`

## Notas Tecnicas

### Ponto de Insercao

- **`js/gdp-init.js`** — Adicionar funcao `migrarBancoPrecos()` na inicializacao, APOS a carga da Central de Produtos e ANTES do render principal
- Verificar `localStorage.getItem('banco-precos-migrated.v1')` — se `true`, pular migracao

### Algoritmo de Matching

```javascript
function encontrarCorrespondencia(itemBanco, produtosCentral) {
  // 1. Match exato por nome normalizado
  const matchNome = produtosCentral.find(p =>
    normalizarNome(p.descricao) === normalizarNome(itemBanco.item)
  );
  if (matchNome) return matchNome;

  // 2. Match por SKU (se disponivel)
  if (itemBanco.sku) {
    const matchSku = produtosCentral.find(p => p.sku === itemBanco.sku);
    if (matchSku) return matchSku;
  }

  // 3. Nao encontrou — sera criado como novo
  return null;
}
```

### Merge Seguro

- Campos do Banco de Precos so sobrescrevem se o campo na Central esta `null`, `undefined` ou vazio
- Arrays (`custosFornecedor`, `concorrentes`, `propostas`) sao CONCATENADOS (sem duplicar por `data + valor`)
- Campo `fonte` do produto mergeado recebe `'migracao-banco-precos'`

### Dados de Origem

- **`caixaescolar.banco.v1`** — localStorage com estrutura `{ itens: [...], meta: {...} }`
- Campos por item: `item` (nome), `custoBase`, `precoReferencia`, `custosFornecedor[]`, `concorrentes[]`, `propostas[]`

## Riscos

| Risco | Probabilidade | Mitigacao |
|-------|:-------------:|-----------|
| Matching impreciso perde correspondencias | Media | Log de itens nao pareados; operador pode corrigir manualmente na Central |
| Migracao demora em base grande (500+ itens) | Baixa | Operacao sincrona em localStorage e rapida (<100ms para 500 itens) |
| Dados corrompidos em `caixaescolar.banco.v1` | Baixa | Try-catch com log de erro; itens invalidos sao pulados |
