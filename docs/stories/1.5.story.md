# Story 1.5 — GDP Pedidos: Listas de Compras, Impressão, Fixes & UX

**Epic:** GDP — Gestão de Pedidos
**Status:** Ready for Review
**Branch:** `feat/caixa-escolar-fase3-sgd`
**Arquivo principal:** `painel-caixa-escolar/squads/caixa-escolar/dashboard/gdp-contratos.html`

---

## Contexto

O sistema GDP precisa de relatórios de lista de compras agrupados por Cliente×Produto e Fornecedor×Produto (baseados nos modelos Olist da Lariucci), impressão formal de pedidos, e uma série de correções e melhorias de UX nas seções Pedidos, Contratos, Inteligência e Entregas.

---

## Acceptance Criteria

### AC-1: Lista de Compras — Agrupada por Cliente × Produto
- [x] Na seção Pedidos, ao selecionar pedidos e clicar "Gerar Lista de Compras", mostrar filtros de agrupamento: **Clientes + Produtos** e **Fornecedores + Produtos**
- [x] Ao selecionar **Clientes + Produtos**, abrir página de relatório estilo Olist com:
  - Header: logo Lariucci, título "Relatório de Vendas"
  - Subtítulo: Período (data inicial a data final) + Situação do pedido
  - Tabela agrupada por **Cliente** (nome + CNPJ em negrito como linha de grupo)
  - Colunas: Produto, Código (SKU), Quantidade, Valor, Frete, Total
  - Subtotal por cliente (quantidade, valor, frete, total)
  - Linha de **Totais** gerais no rodapé
- [x] Layout para impressão (print-friendly, sem scroll, página inteira)

### AC-2: Lista de Compras — Agrupada por Fornecedor × Produto
- [x] Ao selecionar **Fornecedores + Produtos**, abrir página de relatório com:
  - Mesmo header e subtítulo do AC-1
  - Tabela agrupada por **Fornecedor** (nome + CNPJ em negrito como linha de grupo)
  - Itens sem fornecedor agrupados sob "Sem fornecedor"
  - Colunas: Produto, Código (SKU), Quantidade, Valor, Frete, Total
  - Subtotal por fornecedor
  - Linha de **Totais** gerais no rodapé
- [x] Layout para impressão

### AC-3: Impressão de Pedidos — Modelo Formal "Pedido de Fornecimento"
- [x] Ao clicar "Imprimir Pedidos" na seção Pedidos, gerar documento formal com:
  - **Header:** "LARIUCCI & RIBEIRO PEREIRA LTDA" (dados da empresa do localStorage)
  - **Canto superior direito:** "PEDIDO DE FORNECIMENTO" + código do pedido (PED-YYYYMMDD-XXXX) + data
  - **Bloco info:** ESCOLA/ÓRGÃO (nome + CNPJ), RESPONSÁVEL, STATUS, QTD ITENS
  - **Bloco contrato:** Processo, Observações
  - **Tabela itens:** #, PRODUTO/DESCRIÇÃO, UNID., QTD, PREÇO UNIT., SUBTOTAL
  - **VALOR TOTAL** destacado em azul (R$ formato BRL)
  - **TERMOS DE RECEBIMENTO:** declaração padrão, campos para Data, Hora, Nota Fiscal
  - **Assinaturas:** Recebedor(a) | Entregador
  - **Rodapé:** "Documento gerado automaticamente pelo sistema GDP — Gestão de Pedidos" + dados CNPJ empresa
- [x] Cada pedido selecionado gera uma página separada (page-break)
- [x] Acionar `window.print()` automaticamente

### AC-4: Fix — Filtro "Data Prevista" na Lista de Compras
- [x] O filtro de data prevista na tela da lista gerada deve comparar corretamente as datas
- [x] Usar comparação `<=` com normalização de formato (YYYY-MM-DD)
- [x] Verificar que `dataPrevista`, `dataEntrega` e `data` são todos considerados no filtro
- [x] Filtro deve atualizar a lista em tempo real ao alterar a data

### AC-5: Botão "Voltar" na Tela de Importar Mapas (Contratos)
- [x] Na seção Contratos, após importar cronograma/mapa, adicionar botão "← Voltar" na parte superior da tela
- [x] Botão deve retornar à lista de contratos (tab principal)
- [x] Botão visível durante e após o processo de importação

### AC-6: Excluir Pedido Remove Demanda Vinculada
- [x] Ao excluir um pedido, verificar se existe demanda vinculada (por `pedidoId` ou `origem_pedido_id`)
- [x] Se existir, excluir também a demanda do array `estoqueIntelPedidos`
- [x] Exibir no confirm: "Excluir X pedido(s) e suas demandas vinculadas?"
- [x] Salvar ambos os arrays após exclusão

### AC-7: Fix — Pedidos do Catálogo Devem Ir para "Em Aberto" (não "Agendado")
- [x] Pedidos criados pelo fornecedor via catálogo na página de detalhes do contrato devem receber status `pendente` ("Em Aberto"), **não** `agendado`
- [x] Status `agendado` reservado apenas para pedidos criados via importação de cronograma
- [x] Verificar a função que cria pedidos a partir do catálogo e corrigir o status inicial

### AC-8: Trocar "Baixar" por "Excluir" no Card Demandas Geradas (Inteligência)
- [x] No card "Demandas Geradas" da seção Inteligência, remover o botão verde "Baixar" (se existir)
- [x] Substituir por botão vermelho "🗑️ Excluir" que remove a demanda selecionada
- [x] Confirmar que "Cancelar Reserva" no protocolo individual da demanda é equivalente a "Baixar"
- [x] Se equivalente, manter "Cancelar Reserva" no protocolo e usar "Excluir" no card geral
- [x] Excluir demanda deve também limpar referências em pedidos vinculados

### AC-9: Mostrar Nome do Cliente no Protocolo da Demanda (Protocolo Fechado)
- [x] No card do protocolo da demanda (quando clicado), exibir o nome do cliente/escola
- [x] Buscar nome via `pedido.escola` ou `contrato.escola` vinculado
- [x] Exibir em destaque abaixo do número do protocolo

### AC-10: Botão "Reverter" na Seção Entregas
- [x] Adicionar botão "↩️ Reverter" nas linhas de entregas com status "concluído"
- [x] Ao clicar, confirmar: "Reverter baixa manual de [protocolo]? O status voltará para pendente."
- [x] Reverter deve:
  - Mudar status da entrega de "concluido" para "pendente"
  - Restaurar reserva da demanda vinculada se aplicável
  - Atualizar contadores de entrega no contrato
- [x] Botão visível apenas para entregas com baixa manual (não automáticas)

## Technical Notes

- **Arquivo principal:** `gdp-contratos.html` (monolito HTML+JS+CSS)
- **Funções-chave a modificar:**
  - `gerarListaComprasDePedidos()` → AC-1, AC-2 (adicionar agrupamento)
  - `imprimirPedidosSelecionados()` → AC-3 (reescrever template)
  - `consolidarItensParaLista()` / filtro data → AC-4
  - `importarCronograma()` → AC-5 (botão voltar)
  - `excluirPedidosSelecionados()` → AC-6 (cascade delete demanda)
  - Criação pedido via catálogo → AC-7 (status fix)
  - Card Demandas Geradas render → AC-8 (trocar botão)
  - Protocolo demanda render → AC-9 (nome cliente)
  - `_renderEntregasUI()` → AC-10 (botão reverter)
- **Storage keys:** `gdp.pedidos.v1`, `gdp.contratos.v1`, `estoque-intel.pedidos.v1`, `gdp.entregas.provas.v1`
