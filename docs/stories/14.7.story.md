# Story 14.7: Pedidos — editar quantidade + inserir produto na aba Pedidos

## Status: Ready for Review
## Epic: EPIC-14-SYNC-PERFORMANCE
## Priority: P0-Critical
## Agent: @dev (Dex)

## Problema

Na aba Pedidos do GDP, ao visualizar um pedido existente:
1. A quantidade dos itens é exibida como texto estático — não é possível editar
2. O botão "+ Adicionar Produto" abre campo de busca, mas a função `buscarProdutoDetalhe()` não existe — busca não funciona

## Acceptance Criteria

- [ ] AC1: Na visualização de detalhe do pedido, quantidade de cada item deve ser um campo `<input type="number">` editável
- [ ] AC2: Ao alterar a quantidade, o valor total do item e do pedido recalcula automaticamente
- [ ] AC3: Ao alterar a quantidade, o pedido é salvo automaticamente (gdpApi.pedidos.save)
- [ ] AC4: O botão "+ Adicionar Produto" funciona — campo de busca pesquisa no contrato vinculado e estoque-intel
- [ ] AC5: Ao selecionar produto da busca, item é adicionado ao pedido com qtd=1 e editável
- [ ] AC6: Produtos adicionados são persistidos no Supabase via gdpApi

## Tasks

- [x] T1: Qty agora é `<input type="number">` editável em verPedidoDetalhe() (era texto estático)
- [x] T2: `recalcSubtotalDetalhe()` atualizado — lê qty e preço dos inputs, recalcula total e salva
- [x] T3: `buscarProdutoDetalhe()` já existia — busca no contrato vinculado + estoque-intel (confirmado funcional)
- [x] T4: `adicionarProdutoDetalhe()` já existia — adiciona item e re-renderiza (confirmado funcional)
- [ ] T5: Testar em produção

## Arquivos Afetados

## Notas Técnicas

- `verPedidoDetalhe()` está em `gdp-pedidos.js` ~linha 1091-1227
- Itens do pedido: `p.itens[]` com campos `itemNum, descricao, qtd, precoUnitario, unidade, sku, ncm`
- Contrato vinculado: `p.contratoId` → buscar em `contratos[]`
- Estoque-intel: `localStorage gdp.estoque-intel.produtos.v1`
- Save: `gdpApi.pedidos.save(pedido)` com retry automático
