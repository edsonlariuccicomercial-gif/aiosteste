## Story

**Como** operador do Licit-AIX,
**Quero** um normalizador multi-camada que associe descrições SGD a produtos da central com maior precisão, e uma classificação Kraljic em cada produto,
**Para que** o matching automático seja mais confiável e a estratégia de precificação seja adaptada por tipo de produto.

---

## Acceptance Criteria

### Normalizador Multi-Camada
- [ ] **AC1:** Camada N1 — Equivalência confirmada (dicionário): score 1.0, match definitivo
- [ ] **AC2:** Camada N2 — Similaridade tokens (Jaccard): threshold ≥ 0.7 (era 0.6), sugestão com revisão
- [ ] **AC3:** Camada N3 — Regras manuais via regex patterns configuráveis
- [ ] **AC4:** Camada N4 — Fallback: criar produto inline (já implementado na 13.3)
- [ ] **AC5:** Log de qual camada fez o match (para analytics de qualidade do normalizador)

### Classificação Kraljic
- [ ] **AC6:** Campo `classificacao_kraljic` no formulário de produto (dropdown: não-crítico / alavancagem / gargalo / estratégico)
- [ ] **AC7:** Descrição de cada quadrante visível como tooltip/help no dropdown
- [ ] **AC8:** Classificação exibida na tabela da Central de Preços como badge colorido
- [ ] **AC9:** Classificação persistida na `central_produtos` + cloud sync
- [ ] **AC10:** Classificação Kraljic padrão = "alavancagem" (mais comum para licitações escolares)
