# Story 5.1 — Migração Supabase-First: Eliminar localStorage como fonte primária

**Epic:** 5 - Estabilidade e Comercialização
**Status:** Draft
**Priority:** Critical
**Branch:** `feat/supabase-first`

---

## Contexto

O sistema GDP usa localStorage como banco de dados principal e Supabase como backup via sync bidirecional. Isso causa perda de dados em quedas de energia, desvinculação de contratos entre máquinas, e NFs irrecuperáveis. A migração inverte a arquitetura: Supabase vira fonte única, localStorage vira cache.

## Objetivo

Toda leitura e escrita de dados do GDP passa pelo Supabase. O localStorage é usado apenas como cache para renderização imediata enquanto o fetch do Supabase carrega.

---

## Acceptance Criteria

### AC-1: Data Access Layer (gdpApi)
- [ ] Criar módulo `gdpApi` com CRUD para cada entidade (contratos, pedidos, nfs, clientes, contas)
- [ ] Toda chamada `localStorage.getItem/setItem` para dados GDP substituída por `gdpApi`
- [ ] Fallback para localStorage quando offline (navigator.onLine === false)

### AC-2: Tabelas Supabase reais
- [ ] Criar tabelas: `contratos`, `pedidos`, `notas_fiscais`, `clientes`, `contas_pagar`, `contas_receber`
- [ ] Migrar dados de `sync_data` (key-value) para tabelas reais
- [ ] Índices em `empresa_id`, `status`, `created_at`

### AC-3: Boot sequence cloud-first
- [ ] `loadData()` busca do Supabase primeiro, fallback localStorage
- [ ] Eliminar `syncToCloud()` e `syncFromCloud()` do boot
- [ ] Renderização com dados do Supabase, não do localStorage

### AC-4: Escrita direto no Supabase
- [ ] `saveContrato()`, `saveNF()`, `savePedido()` gravam no Supabase
- [ ] localStorage atualizado como cache após confirmação do Supabase
- [ ] Erro de rede mostra toast e mantém dado local para retry

### AC-5: Mesma tela em todas as máquinas
- [ ] Abrir gdp-contratos.html em 2 browsers diferentes mostra mesmos dados
- [ ] Criar contrato em browser A aparece em browser B após reload
- [ ] NF emitida fica disponível em qualquer máquina imediatamente

### AC-6: Proteção contra perda de dados
- [ ] NF-e salva chave de acesso e XML no Supabase ANTES de retornar ao browser
- [ ] Queda de energia após emissão NF não perde dados fiscais
- [ ] Vinculação contrato-escola é foreign key, não array local
