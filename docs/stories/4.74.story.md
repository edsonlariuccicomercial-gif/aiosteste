# Story 4.74 — Fix 4 bugs GDP: filtro emissão, número NF, sync notas entrada, lentidão boot

- **Epic:** EPIC-4 — GDP Caixa Escolar
- **Status:** Ready
- **Complexity:** M (Medium)
- **Agent:** @dev
- **Depends on:** Story 4.72 (parcialmente — lentidão não resolvida)
- **Priority:** Critical (lentidão afeta todos os usuários em todos os computadores)

## Descrição

4 bugs reportados pelo usuário, todos confirmados por @analyst:

1. **Filtro Contas a Receber** — filtra por categorias mas deveria filtrar por data de emissão
2. **Número NF na página pedidos** — mostra "próxima nota: 1461" mas gera 1460 (off-by-one)
3. **Notas de entrada excluídas reaparecem** cross-machine — key de exclusão não sincroniza
4. **Lentidão persiste** — 6 chamadas `await gdpApi[table].list()` bloqueiam ANTES do render

## Acceptance Criteria

### AC1: Filtro Contas a Receber por emissão
- **Given** o usuário está na aba Financeiro > Contas a Receber
- **When** usa o filtro de categorias
- **Then** o filtro agrupa/filtra por data de emissão (`dataEmissao`) em vez de `categoria`
- **And** o label do dropdown muda de "Categorias" para "Por Emissão" ou similar

### AC2: Número NF correto na página pedidos
- **Given** o usuário clica "Gerar Nota Fiscal" em um pedido
- **When** a mensagem de preview mostra "próxima nota: X"
- **Then** a nota gerada é exatamente X (não X-1)
- **And** `peekProximoNumeroNf()` e `consumirProximoNumeroNf()` retornam o mesmo número

### AC3: Exclusão de notas de entrada sincroniza cross-machine
- **Given** o usuário exclui uma nota de entrada no Computador A
- **When** acessa o sistema no Computador B
- **Then** a nota excluída NÃO reaparece
- **And** a key `gdp.notas-entrada.deleted.v1` é incluída no `SYNC_KEYS`

### AC4: Boot GDP non-blocking (resolver lentidão real)
- **Given** o usuário abre a página GDP
- **When** o boot inicia
- **Then** o bloco `gdpApi[table].list()` (6 chamadas Supabase) roda em background APÓS renderAll()
- **And** a UI fica interativa em < 3s (localStorage-first, Supabase em background)

## Scope

### IN
- Trocar filtro categorias → emissão em `gdp-pedidos.js:1685-1692`
- Corrigir off-by-one em `gdp-notas-fiscais.js:525-564` (peek vs consumir)
- Adicionar `gdp.notas-entrada.deleted.v1` ao `SYNC_KEYS` em `app-state.js:682-703`
- Mover bloco `gdpApi[table].list()` em `gdp-init.js:2823-2890` para background

### OUT
- Mudanças no schema Supabase
- Refatoração do gdpApi

## Análise Técnica (@analyst)

### Bug 1: `gdp-pedidos.js:1685-1692`
- Linha 1692: `filtered = filtered.filter(item => item.categoria === filtroCategoria)` → mudar para filtro por `dataEmissao`

### Bug 2: `gdp-notas-fiscais.js:525-564`
- `peekProximoNumeroNf()` (linha 525) retorna N
- `consumirProximoNumeroNf()` (linha 536) retorna N-1 e depois incrementa para N
- Fix: alinhar para que ambos retornem o mesmo número

### Bug 3: `app-state.js:682-703`
- Adicionar `"gdp.notas-entrada.deleted.v1"` ao array `SYNC_KEYS`

### Bug 4: `gdp-init.js:2823-2890`
- Mover todo o bloco Supabase-First para `requestIdleCallback` ou `setTimeout(fn, 0)`
- Padrão: render do localStorage primeiro → merge Supabase em background → re-render se houver dados novos
