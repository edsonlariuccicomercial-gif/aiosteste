## Story

**Como** operador do Licit-AIX,
**Quero** uma Central de Produtos única com custos de fornecedores em tabela separada,
**Para que** eu tenha 1 cadastro mestre de produtos com múltiplos registros de custo por fornecedor, eliminando bancos fragmentados.

---

## Acceptance Criteria

- [ ] **AC1:** Tabela `central_produtos` com campos: id, nome, unidade_base, categoria, sku, ncm, origem, produto_critico, ativo, criadoEm, atualizadoEm
- [ ] **AC2:** Tabela `custos_fornecedores` com campos: id, produto_id, fornecedor, custo, data_coleta, validade, regiao, origem, confiabilidade, arquivo_id
- [ ] **AC3:** CRUD completo na Central de Produtos (criar, editar, excluir com soft delete)
- [ ] **AC4:** Ao visualizar um produto, exibir lista de custos de todos os fornecedores com datas
- [ ] **AC5:** Custo mais recente e menor custo destacados na tabela de produtos
- [ ] **AC6:** Campo `origem` do custo identifica fonte: excel, pdf, pdf-ocr, nf, api, b2b, marketplace, manual
- [ ] **AC7:** Campo `confiabilidade` (0-1) atribuído automaticamente conforme origem (NF=1.0, manual=0.9, excel=0.95, OCR=0.5)
- [ ] **AC8:** Migração automática: dados existentes em `gdp.estoque-intel.produtos.v1` e `caixaescolar.banco.v1` migrados para nova estrutura no boot
- [ ] **AC9:** Sync cloud (Supabase) para `central_produtos` e `custos_fornecedores`
- [ ] **AC10:** Importação Excel/PDF grava em `custos_fornecedores` (não sobrescreve produto)

## Dev Notes

### Modelo de dados — REGRA FUNDAMENTAL:

```
PROIBIDO:                    CORRETO:
├─ banco fornecedor A        ├─ 1 central_produtos (base fixa)
├─ banco fornecedor B        ├─ N custos_fornecedores (valores dinâmicos)
└─ banco fornecedor C        └─ Decisão baseada em dados

PRODUTO → MÚLTIPLOS CUSTOS → DECISÃO
```

### Arquivos a modificar:
- `app-state.js` — novas constantes de storage keys
- `app.js` — `renderCentralPrecos()` refatorar para exibir custos por fornecedor
- `app-import.js` — `mergeImportIntoBanco()` gravar em `custos_fornecedores` em vez de sobrescrever
- `js/gdp-core.js` — funções save/load para novas tabelas
- `js/gdp-banco-produtos.js` — adaptar CRUD para novo modelo
- `index.html` — UI expandida com sub-tabela de custos por produto

### Migração:
- No boot, detectar formato antigo (array plano com `custoBase` embutido)
- Extrair `custosFornecedor[]` de cada produto → mover para `custos_fornecedores`
- Manter produto limpo na `central_produtos` (sem preços)
- Flag `_migrated: true` para não re-migrar

### Confiabilidade por origem:

| Fonte | Tag | Score |
|-------|-----|-------|
| Nota Fiscal XML | nf | 1.00 |
| Excel importado | excel | 0.95 |
| Digitação manual | manual | 0.90 |
| B2B/Marketplace | b2b | 0.85 |
| PDF texto | pdf | 0.85 |
| PDF OCR | pdf-ocr | 0.50 |
| API externa | api | 0.90 |
