# Story 8.5 — Modal de Revisao Pre-Envio com Deteccao de Divergencia de Unidade

**Epic:** Epic 8 — Consolidacao GDP e Fechamento de Gaps Operacionais
**Status:** Ready
**Priority:** P0
**Story Points:** 3

---

## Descricao

Atualmente, quando o operador envia a proposta ao SGD (funcao `enviarParaSgd()` em `app-sgd-integration.js:120`), nao existe nenhuma etapa de validacao que compare a unidade do SGD com a unidade da Central de Produtos. Isso causa perda de licitacoes quando o operador cota "R$ 3,50 por unidade" mas o SGD espera "R$ 42,00 por caixa com 12".

Esta story implementa um modal de revisao que intercepta o fluxo de envio, lista todos os itens com suas unidades, detecta divergencias, e bloqueia o envio ate o operador confirmar ciencia.

**Referencia SOP:** Etapa 5 — Revisao da Cotacao

### Ponto de Insercao

- **Arquivo:** `app-sgd-integration.js`
- **Funcao:** `enviarParaSgd()` — linha 120
- **Fluxo atual:** Clique "Enviar" -> `buildSgdPayload()` (linha 58) -> envio ao proxy
- **Fluxo novo:** Clique "Enviar" -> `abrirModalRevisao()` -> operador confirma -> `buildSgdPayload()` -> envio ao proxy

### Dados de Entrada

```javascript
// Itens do pre-orcamento (app-state)
const itens = preOrcamentos[preOrcId].itens;
// Cada item tem: nome, preco, quantidade

// Itens do orcamento SGD
const itensSgd = orcamentos[orcId].itens;
// Cada item tem: nome, unit (unidade SGD), quantidade

// Matching com Central
const produtoCentral = RadarMatcher.match(item.nome);
// Retorna: descricao, unidade, sku, ncm, marca, custoBase, precoReferencia
```

### Deteccao de Divergencia

```javascript
function detectarDivergenciaUnidade(unidadeSgd, unidadeCentral) {
  const normSgd = normalizarUnidade(unidadeSgd);   // "CX C/ 12" -> "CX12"
  const normCentral = normalizarUnidade(unidadeCentral); // "UN" -> "UN"
  return normSgd !== normCentral;
}
```

### Restricao CON-G1-002

O matching entre item SGD e produto Central DEVE usar `RadarMatcher.match()` existente em `radar-matcher.js`. NAO criar mecanismo de matching novo.
