# Story 5.17 — Entrega Parcial de Pedidos

**Epic:** 5 — Modernização GDP
**Status:** Ready
**Prioridade:** Alta
**Estimativa:** M (médio)
**Agente:** @dev

---

## Contexto

Faturamento e entrega são processos independentes no fluxo do GDP. Atualmente não há controle de entrega por item — quando um pedido é marcado como "entregue", é tudo ou nada. Na prática:

- Entregas podem ser fracionadas (ex: 5 KG pedidos, entregam 2 KG hoje e 3 KG semana que vem)
- Faturamento pode ocorrer antes, depois ou independente da entrega
- Não há visibilidade de saldo pendente por item

Referência visual: sistema Tiny ERP (imagens fornecidas pelo PO).

---

## Acceptance Criteria

### AC1 — Botão "Entregar" no detalhe do pedido
- [ ] Adicionar botão "Entregar" na topbar do detalhe do pedido (`gdp-pedidos.js`)
- [ ] Botão visível para pedidos com status "recebido", "faturado" ou "em separação"
- [ ] Estilo: botão verde, alinhado ao lado dos botões existentes

### AC2 — Modal "Entregar Pedido"
- [ ] Modal abre ao clicar "Entregar"
- [ ] Campos do header:
  - Data de entrega (default: hoje)
  - Observações (textarea)
- [ ] Dados fiscais opcionais (colapsável):
  - N NF-e vinculada (input texto)
  - Série (input texto)
  - Data de emissão (date picker)
- [ ] Tabela de itens com colunas:
  - Produto (nome/descrição)
  - Qtd Pedida
  - Qtd Já Entregue
  - Saldo Pendente
  - Input "Entregar" (numérico, max = saldo pendente)
  - Unidade (KG, UN, etc.)
- [ ] Botões: "Fechar" | "Entregar Pedido" (verde)

### AC3 — Controle de saldo por item
- [ ] Cada item do pedido ganha campos: `qtdEntregue` (default 0), computado `saldoPendente = qtd - qtdEntregue`
- [ ] Ao confirmar entrega, `qtdEntregue` incrementa com a quantidade informada
- [ ] Validação: não permitir entregar mais que o saldo pendente
- [ ] Dados persistem em `pedido.dados_extras.entregas[]` (JSONB no Supabase)

### AC4 — Histórico de entregas por pedido
- [ ] Array `pedido.dados_extras.entregas[]` armazena cada remessa:
  ```json
  {
    "id": "ent-xxx",
    "data": "2026-06-02",
    "obs": "Entrega parcial - 1a remessa",
    "nfNumero": "",
    "nfSerie": "",
    "nfDataEmissao": "",
    "itens": [
      { "itemIdx": 0, "descricao": "Filé de tilápia", "qtdEntregue": 2, "unidade": "KG" }
    ],
    "criadoEm": "2026-06-02T14:30:00Z",
    "criadoPor": "lariucci"
  }
  ```
- [ ] Histórico visível no detalhe do pedido (seção colapsável abaixo dos itens)

### AC5 — Status de entrega automático
- [ ] Status calculado automaticamente baseado no saldo:

| Status | Condição | Badge |
|--------|----------|-------|
| `pendente` | Nenhum item entregue (`totalEntregue = 0`) | `badge-vencido` (amarelo) |
| `parcial` | Algum item entregue, mas há saldo | `badge-pendente` (laranja) |
| `entregue` | Todos os itens com saldo = 0 | `badge-aprovado` (verde) |
| `atrasado` | Data prevista passou + saldo > 0 | `badge-vermelho` (vermelho) |

- [ ] Status salvo em `pedido.dados_extras.statusEntrega`

### AC6 — Exibição na lista de pedidos
- [ ] Coluna "Entrega" na listagem de pedidos (ao lado de Status e Valor)
- [ ] Mostra: badge de status + data da última entrega
- [ ] Formato: "ENTREGUE PARCIALMENTE" com ícone, ou "ENTREGUE" verde, ou "PENDENTE", ou "ATRASADO" vermelho

### AC7 — Saldo pendente nos itens do pedido
- [ ] Na tabela de itens do detalhe do pedido, exibir coluna "Entrega"
- [ ] Formato: "02 PENDENTE(S)" em badge laranja quando há saldo, ou "OK" verde quando entregue

### AC8 — Persistência Supabase
- [ ] `savePedidos()` já persiste no Supabase (fix anterior desta sessão)
- [ ] Verificar que `dados_extras` com entregas é persistido corretamente
- [ ] Sync cross-machine via Realtime funciona para atualizações de entrega
