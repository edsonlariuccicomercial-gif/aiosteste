# Story 8.4 — Enriquecer UI Central de Produtos com Colunas de Custo, Margem e Competitividade

**Epic:** Epic 8 — Consolidacao GDP e Fechamento de Gaps Operacionais
**Status:** Ready
**Priority:** P1
**Story Points:** 2

---

## Descricao

Com o schema unificado (Story 8.1), a Central de Produtos agora contem campos de pricing que antes so existiam no Banco de Precos. Esta story enriquece a UI de `renderBancoProdutos()` para exibir essas informacoes diretamente na tabela, portando as colunas de custo/margem/competitividade que antes existiam em `renderBanco()` do `app-banco.js`.

**Referencia SOP:** Etapa 7 (Central de Produtos com visao integrada)

---

## Criterios de Aceitacao

- [ ] AC-1: Tabela da Central de Produtos exibe coluna "Custo Base (C.A.)" com valor de `custoBase`
- [ ] AC-2: Tabela exibe coluna "Preco Ref." com valor de `precoReferencia`
- [ ] AC-3: Tabela exibe coluna "Margem Real (%)" calculada como `((precoReferencia - custoBase) / custoBase * 100)`
- [ ] AC-4: Badge de tendencia exibido: "subindo" (verde), "estavel" (cinza), "caindo" (vermelho) — baseado em comparacao com ultimo registro de `custosFornecedor[]`
- [ ] AC-5: Badge de competitividade: "competitivo" (verde), "na media" (amarelo), "caro" (vermelho) — baseado em `taxaConversao` (>=70% verde, 40-69% amarelo, <40% vermelho)
- [ ] AC-6: Colunas extras aparecem somente se o produto tem dados de pricing (sem noise visual para produtos sem custoBase)
- [ ] AC-7: Renderizacao completa em menos de 300ms para 500 produtos (NFR-G2-003)
- [ ] AC-8: UI segue design system existente (CSS variables, dark theme, classes `.badge-success`, `.badge-warn`, `.badge-danger`)

## Notas Tecnicas

### Arquivo Principal

- **`js/gdp-banco-produtos.js`** — funcao `renderBancoProdutos()` — e onde a tabela da Central e renderizada

### Colunas a Adicionar

| Coluna | Campo | Formato | Visibilidade |
|--------|-------|---------|-------------|
| Custo Base | `custoBase` | `R$ XX,XX` | Se `custoBase > 0` |
| Preco Ref. | `precoReferencia` | `R$ XX,XX` | Se `precoReferencia > 0` |
| Margem (%) | Calculada | `XX,X%` | Se ambos custoBase e precoReferencia existem |
| Tendencia | Derivada de `custosFornecedor[]` | Badge | Se `custosFornecedor.length > 1` |
| Competitividade | Derivada de `taxaConversao` | Badge | Se `taxaConversao != null` |

### Referencia de Implementacao Existente

A funcao `renderBanco()` em `app-banco.js` ja renderiza colunas similares. Portar a logica de calculo de margem e badges para `renderBancoProdutos()`.

### Performance

- Usar `DocumentFragment` ou `innerHTML` em batch para renderizar 500+ linhas
- Evitar `DOM.appendChild()` em loop (causa reflows)
- Calculos de margem e badges devem ser feitos em memoria antes do render

## Riscos

| Risco | Probabilidade | Mitigacao |
|-------|:-------------:|-----------|
| Tabela fica muito larga com 5 colunas extras | Media | Colunas responsivas; tooltip para valores longos; considerar layout colapsavel |
| Performance de render degradada com muitos produtos | Baixa | DocumentFragment + calculo em batch; medir com `performance.now()` |
| Divisao por zero no calculo de margem | Baixa | Guard `custoBase > 0` antes do calculo |
