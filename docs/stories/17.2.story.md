## Story
**Como** equipe técnica,
**Quero** reconciliar os registros gravados sob `empresa_id` divergentes para o id canônico,
**Para** que os dados existentes (incl. os 161 lançamentos) sejam vistos por todos os usuários.

## Contexto / Problema
Dados já foram gravados sob ids diferentes (ex.: alguns sob `LARIUCCI`, outros sob nome/cnpj). Sem reconciliação, mesmo com o código corrigido (17.1), os dados antigos continuam invisíveis para parte dos usuários.

## Critérios de Aceitação
- **AC1:** *Given* a migração executada, *When* Angela e Edson abrem o caixa, *Then* ambos veem o MESMO conjunto de lançamentos.
- **AC2:** Nenhum dado da Angela (hoje correto) é perdido.
- **AC3:** Os 161 lançamentos passam a contabilizar para todos.
- **AC4:** Contagem total pós-migração = soma esperada (sem perda nem duplicação).

## Requisitos Funcionais
- **FR-17.2.1:** Migrar `conciliacoes`, `contas_receber`, `contas_pagar`, `pedidos` dos ids divergentes → id canônico (plano da 17.0).
- **FR-17.2.2:** Tratar conflitos de ID/duplicatas sem perder nem duplicar.
- **FR-17.2.3:** Migração reversível ou com backup imediatamente antes.

## Escopo
- **IN:** migração SQL/script, validação de contagem.
- **OUT:** código (17.1, 17.3).

## Dependências
**Bloqueada por 17.0 e 17.1.** Exige **snapshot Supabase imediatamente antes** (@devops).

## Riscos
ALTO — escrita em produção. Snapshot + rollback obrigatórios. Janela fora de pico.

## Definition of Done
