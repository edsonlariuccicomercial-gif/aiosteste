# Story 4.82 — Portal escolar: contratos invisíveis/não-clicáveis em aba anônima ou primeiro acesso

- **Epic:** EPIC-4 — GDP Caixa Escolar
- **Status:** Ready for Review
- **Complexity:** S (Small)
- **Agent:** @dev
- **Depends on:** Story 4.71 (contratos no portal)
- **Priority:** High (usuárias reportando que não conseguem fazer pedidos)

## Descrição

Quando o portal escolar (`gdp-portal.html`) é aberto em aba anônima ou num dispositivo/browser novo (sem cache localStorage), a escola faz login com sucesso mas os contratos não aparecem — ou aparecem só o card ARP sem contratos vinculados. A usuária vê a escola, vê o contato, mas não consegue entrar para fazer pedidos.

### Causa raiz (diagnosticada por @analyst)

**Race condition + localStorage vazio:**

1. Login funciona pelo fallback JSON estático (`escolas-credentials.json`), mas esse JSON **não tem** `contratos_vinculados`
2. `showContractSelection()` é chamado **sincronamente** (linha 693) ANTES do Supabase retornar dados
3. `getSchoolContracts()` lê `gdp.contratos.v1` do localStorage — que está **vazio** em aba limpa
4. Resultado: zero contratos renderizados. O card ARP pode aparecer (vem do JSON estático), mas os contratos de licitação/processo não
5. O Supabase é chamado fire-and-forget (linhas 682-690) e quando retorna grava no localStorage e chama `showContractSelection()` de novo — mas pode ser tarde demais ou falhar silenciosamente

### Arquivos afetados

- `gdp-portal.html:693` — chamada síncrona antes dos dados chegarem
- `gdp-portal.html:749-773` — `getSchoolContracts()` depende 100% de localStorage
- `gdp-api.js:54-57` — `writeLS()` sem try-catch (falha em Safari privado)
- `escolas-credentials.json` — sem `contratos_vinculados`

## Acceptance Criteria

### AC1: Contratos carregam corretamente em aba anônima
- **Given** usuária abre o portal em aba anônima/browser novo (localStorage vazio)
- **When** faz login com credenciais válidas
- **Then** os contratos da escola são exibidos (vindos do Supabase)
- **And** um loading indicator aparece enquanto os dados são carregados

### AC2: showContractSelection aguarda dados do Supabase
- **Given** o login é executado
- **When** `gdpApi` está disponível
- **Then** `showContractSelection()` só renderiza os contratos APÓS o Supabase retornar (ou timeout)
- **And** se Supabase falhar, mostra mensagem orientando reconexão

### AC3: writeLS resiliente em aba privada
- **Given** o browser está em modo privado (Safari, Firefox, Chrome antigo)
- **When** `writeLS()` tenta gravar no localStorage
- **Then** não lança exceção não-capturada
- **And** o portal continua funcionando com dados em memória

### AC4: Fallback funcional sem localStorage
- **Given** localStorage está indisponível ou vazio
- **When** o Supabase retorna contratos
- **Then** `getSchoolContracts()` usa os dados retornados diretamente (não depende de localStorage como intermediário)

## Scope

### IN
- Fazer `showContractSelection()` aguardar o retorno do Supabase (com loading spinner)
- Adicionar try-catch no `writeLS()` do `gdp-api.js`
- Fazer `getSchoolContracts()` aceitar dados em memória (não só localStorage)
- Mostrar loading state enquanto contratos carregam

### OUT
- Mudanças no JSON estático `escolas-credentials.json` (é fallback, não fonte principal)
- Mudanças na tela de login
- Mudanças no dashboard do gestor

## Dev Notes

### Abordagem sugerida

1. No `doLogin()`, quando `gdpApi` disponível:
   - Mostrar spinner/loading no container de contratos
   - Await o `Promise.all([contratos.list(), pedidos.list(), clientes.list()])`
   - Só então chamar `showContractSelection()`
   - Se Supabase falhar (catch), chamar `showContractSelection()` com fallback do localStorage

2. No `gdp-api.js`:
   - Envolver `writeLS()` com try-catch para resiliência em modo privado
   - Manter dados do último `list()` em variável in-memory para `getSchoolContracts()` usar como fallback

3. No `getSchoolContracts()`:
   - Aceitar parâmetro opcional com contratos já em memória
   - Fallback: `localStorage → memória → []`

### Reprodução
1. Abrir portal em aba anônima
2. Login com qualquer escola (ex: `lauriston` / `escola2025`)
3. Observar: card ARP pode aparecer, mas contratos de processo/licitação não aparecem
4. Em aba normal (com cache): contratos aparecem normalmente
