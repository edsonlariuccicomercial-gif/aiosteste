# Story 1.6 — GDP Pós-Licitação: Otimização de Performance

**Epic:** GDP — Gestão de Pedidos
**Status:** Ready for Review
**Branch:** `feat/caixa-escolar-fase3-sgd`
**Arquivo principal:** `painel-caixa-escolar/dashboard/app.js`

---

## Contexto

O módulo GDP Pós-Licitação (pré-cotação, cotações e banco de preços) apresenta lentidão severa. O diagnóstico identificou 10 gargalos: 9 fetches sem cache recarregados a cada 60s (~400KB), re-render completo do DOM a cada keystroke, cálculos caros repetidos 100x+ sem memoização, e ausência de debounce/paginação. O resultado é UI travando ao filtrar, digitar preços, e scroll em tabelas grandes.

## Acceptance Criteria

### AC-1: Cache inteligente nos fetches de dados (CRÍTICO)
- [x] Substituir `cache: "no-store"` por estratégia com TTL nas 9 chamadas fetch em `loadQuotes()` e funções adjacentes
- [x] Implementar cache em memória com TTL de 5 minutos (configurável) — dados só re-baixam se expirados
- [x] Auto-refresh de 60s verifica TTL antes de re-fetch; se cache válido, skip
- [x] `price-history-summary.json` (pesado) usa TTL separado maior (15min) por ser dado estável
- [x] Indicador visual "Atualizando..." já existente em `refreshDataAndRender()` preservado
- [x] **Bônus:** boot() e refresh paralelizados com `Promise.all` (antes: 9 awaits sequenciais)

### AC-2: Debounce nos inputs de busca e filtro (CRÍTICO)
- [x] Debounce de 300ms nos event listeners de `keyup` no campo de busca
- [x] Eventos `change` nos selects (SRE, cidade, status) permanecem imediatos
- [x] `renderAll()` protegido contra chamadas múltiplas via debounce

### AC-3: Memoização de funções caras em loops de renderização (ALTO)
- [x] Memoizar `priority(row)` — cache por `row.id`, invalidar no próximo ciclo
- [x] Memoizar `quoteMinPricing(row)` — lookup de SKU estável intra-render
- [x] Memoizar `historyRangeForSku(sku, sre)` — dados de histórico estáveis intra-sessão
- [ ] `suggestedBid()` não memoizado (depende de args calculados, custo de memo > benefício)
- [x] Memoizar `opportunityScore(row)` e `daysTo(date)`
- [x] Cache invalidado via `invalidateAllCaches()` após fetch de novos dados

### AC-4: Renderização parcial — não re-gerar DOM inteiro (ALTO)
- [x] `renderObjetoGroups()`: skip de `tableRows` para grupos colapsados (lazy render)
- [x] Grupos colapsados renderizam apenas header (0 rows processados vs N rows antes)
- [x] `renderKPIs()`, `renderSummary()`: usam dados pré-computados em `_precomputed` — sem re-filter

### AC-5: Pré-cotação — update targeted no form (ALTO)
- [x] Input handler atualiza SOMENTE subtotal da linha e total do pedido via `data-total-order`
- [x] NÃO chama `renderPrequoteOrders()` — preserva foco, scroll e estado
- [x] Totais recalculados incrementalmente via `prequoteTotals()`

### AC-6: Pipeline de filtragem single-pass (ALTO)
- [x] `filteredRows()` refatorado de 5 `.filter()` encadeados para um único `for` loop
- [x] `renderAll()` pré-computa `open`, `urgent`, `totalRevenue`, `marginSum` em uma passada
- [x] `renderKPIs`, `renderAlerts`, `renderSummary` usam `_precomputed` — zero `.filter()` redundante

### AC-7: Paginação nas tabelas de cotações e produtos (MÉDIO)
- [x] Tabela de cotações: máximo 50 linhas/página com controles anterior/próximo
- [x] Tabela de produtos GDP: paginação de 50 itens com controles
- [x] Filtrar reseta para página 1 automaticamente
- [x] Contagem "Mostrando X–Y de Z resultados" visível

### AC-8: Otimização CSS — reduzir blur e repaint (BAIXO)
- [x] `blur(60px)` reduzido para `blur(30px)` em `.bg-shape`
- [x] `backdrop-filter: blur(3px)` substituído por `background: rgba(21,34,28,0.88)`
- [x] `will-change: transform` adicionado nos `.bg-shape`

## Risks

- **Regressão funcional:** refatoração de renderização pode quebrar interações existentes (mitigação: testar todos os fluxos pré-cotação, filtros, paginação)
- **Perda de estado:** renderização parcial (AC-4/AC-5) pode não preservar estado de inputs/scroll (mitigação: testar com formulários preenchidos)
- **Cache stale:** TTL de 5-15min pode mostrar dados desatualizados (mitigação: botão manual "forçar atualização")
