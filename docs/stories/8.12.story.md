# Story 8.12 — Migracao de Dados Historicos — Backfill preco_historico

**Epic:** Epic 8 — Consolidacao GDP e Fechamento de Gaps Operacionais
**Status:** Ready
**Priority:** P2
**Story Points:** 2

---

## Descricao

A Story 8.9 instala os 6 pontos de insercao que popula `preco_historico` daqui para frente. Porem, dados historicos ja existem em localStorage (Banco de Precos) e no Supabase (`resultados_orcamento`). Esta story executa uma migracao one-time que popula `preco_historico` retroativamente a partir de 4 fontes de dados historicos.

Isso garante que o dashboard de serie historica (Story 8.11) tenha dados desde o primeiro dia, nao apenas dados novos.

**Referencia SOP:** Etapa 14 — Serie Historica (retroalimentacao)

---

## Criterios de Aceitacao

- [ ] AC-1: Dados de `bancoPrecos.itens[].custosFornecedor[]` migrados como tipo `nf_entrada`
- [ ] AC-2: Dados de `bancoPrecos.itens[].propostas[]` migrados como tipo `proposta`
- [ ] AC-3: Dados de `bancoPrecos.itens[].historicoResultados[]` migrados como tipo `ganho` ou `perdido` conforme resultado
- [ ] AC-4: Dados de `resultados_orcamento` no Supabase migrados como tipo `ganho`/`perdido`
- [ ] AC-5: Migracao usa `ON CONFLICT DO NOTHING` para evitar duplicatas (CON-G4-002)
- [ ] AC-6: Migracao e idempotente — executar multiplas vezes nao duplica registros
- [ ] AC-7: Flag `localStorage.setItem('preco-historico-backfilled.v1', 'true')` setada apos conclusao
- [ ] AC-8: Log de migracao com contadores: `{total, inserted, skipped, errors}`

## Notas Tecnicas

### 4 Fontes de Dados

| # | Fonte | Localizacao | tipo em preco_historico |
|---|-------|-------------|----------------------|
| 1 | Custos de fornecedor | `caixaescolar.banco.v1` -> `itens[].custosFornecedor[]` | `nf_entrada` |
| 2 | Propostas enviadas | `caixaescolar.banco.v1` -> `itens[].propostas[]` | `proposta` |
| 3 | Resultados historicos | `caixaescolar.banco.v1` -> `itens[].historicoResultados[]` | `ganho` / `perdido` |
| 4 | Resultados no Supabase | Tabela `resultados_orcamento` | `ganho` / `perdido` |

### Algoritmo de Migracao

```javascript
async function backfillPrecoHistorico() {
  if (localStorage.getItem('preco-historico-backfilled.v1') === 'true') return;

  const counters = { total: 0, inserted: 0, skipped: 0, errors: 0 };

  // --- Fonte 1, 2, 3: localStorage (Banco de Precos) ---
  const banco = JSON.parse(localStorage.getItem('caixaescolar.banco.v1') || '{}');
  const itens = banco.itens || [];

  for (const item of itens) {
    // Buscar SKU na Central unificada
    const produto = buscarNaCentral(item.item);
    const sku = produto?.sku || normalizarSku(item.item);

    // Fonte 1: custosFornecedor -> nf_entrada
    for (const custo of (item.custosFornecedor || [])) {
      counters.total++;
      await inserirComConflict({
        empresa_id: empresaAtual.id,
        sku,
        tipo: 'nf_entrada',
        valor: custo.preco,
        fonte: 'backfill-banco-precos',
        metadata: { fornecedor: custo.fornecedor, data_original: custo.data },
        created_at: custo.data || new Date().toISOString(),
      }, counters);
    }

    // Fonte 2: propostas -> proposta
    for (const proposta of (item.propostas || [])) {
      counters.total++;
      await inserirComConflict({
        empresa_id: empresaAtual.id,
        sku,
        tipo: 'proposta',
        valor: proposta.valor || proposta.preco,
        fonte: 'backfill-banco-precos',
        metadata: { escola: proposta.escola, data_original: proposta.data },
        created_at: proposta.data || new Date().toISOString(),
      }, counters);
    }

    // Fonte 3: historicoResultados -> ganho/perdido
    for (const res of (item.historicoResultados || [])) {
      counters.total++;
      await inserirComConflict({
        empresa_id: empresaAtual.id,
        sku,
        tipo: res.resultado === 'ganho' ? 'ganho' : 'perdido',
        valor: res.valor,
        fonte: 'backfill-banco-precos',
        metadata: { escola: res.escola, data_original: res.data },
        created_at: res.data || new Date().toISOString(),
      }, counters);
    }
  }

  // --- Fonte 4: Supabase resultados_orcamento ---
  const { data: resultados } = await supabase
    .from('resultados_orcamento')
    .select('*')
    .eq('empresa_id', empresaAtual.id);

  for (const res of (resultados || [])) {
    counters.total++;
    const tipo = res.resultado === 'ganho' ? 'ganho' : 'perdido';
    await inserirComConflict({
      empresa_id: res.empresa_id,
      sku: res.sku || 'unknown',
      tipo,
      valor: res.valor,
      escola: res.escola,
      sre: res.sre,
      fonte: 'backfill-resultados-sb',
      metadata: { orcamento_id: res.orcamento_id },
      created_at: res.created_at,
    }, counters);
  }

  localStorage.setItem('preco-historico-backfilled.v1', 'true');
  console.log('Backfill preco_historico concluido:', counters);
}
```

### Funcao inserirComConflict()

```javascript
async function inserirComConflict(registro, counters) {
  try {
    const { error } = await supabase
      .from('preco_historico')
      .insert(registro);
    // Se duplicata (unique violation), conta como skipped
    if (error && error.code === '23505') {
      counters.skipped++;
    } else if (error) {
      counters.errors++;
      console.warn('Backfill insert error:', error);
    } else {
      counters.inserted++;
    }
  } catch (err) {
    counters.errors++;
  }
}
```

## Riscos

| Risco | Probabilidade | Mitigacao |
|-------|:-------------:|-----------|
| Volume alto de inserts pode atingir rate limit do Supabase | Media | Batch de 50 registros por vez; delay entre batches |
| Dados historicos sem SKU consistente | Alta | Usar `buscarNaCentral()` para mapear; fallback `normalizarSku(item.item)` |
| Dados historicos sem escola/SRE/municipio | Alta | Campos opcionais; metadata como `data_original` preserva rastreabilidade |
| Supabase free tier storage | Media | Monitorar tamanho; dados historicos compactados (sem arrays grandes) |
