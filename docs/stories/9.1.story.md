## Problema

Ao registrar um orçamento como "ganho" no Intel Preços, o sistema exibe toast "Contrato gerado" mas o contrato **não aparece** no módulo GDP (aba Contratos).

### Evidência
- `gdp.contratos.v1` no localStorage contém 30 contratos
- Contrato "Escritora Carolina" **não está entre eles**
- Função `criarContratoGdpComCentral()` em `app-results.js:557` é chamada mas falha silenciosamente

## Contexto Técnico

### Fluxo atual (app-results.js)
1. Usuário clica "Ganho" no Intel Preços → `registrarResultado()`
2. Linha 194: `criarContratoGdpComCentral(orcId, pre, numContrato)` é chamada
3. Linha 557-580: `criarContratoGdpComCentral()` enriquece itens com Central de Produtos
4. Linha 580: delega para `criarContratoGdp()` (linha 419)
5. Linha 531: salva em `localStorage.setItem(GDP_CONTRATOS_KEY, ...)`

### Hipóteses de falha
1. **Erro silencioso** no `criarContratoGdpComCentral()` — try/catch engolindo exceção
2. **Conflito de formato** — `unwrapData()` vs array direto no parse do localStorage
3. **Race condition** — outro save sobrescrevendo o localStorage antes do contrato ser persistido
4. **Dados do pré-orçamento** inválidos causando crash na criação dos itens
