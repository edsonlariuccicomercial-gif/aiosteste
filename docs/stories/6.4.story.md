# Story 6.4 — Engine de Preço Sugerido v2 (Signals Inteligentes)

**Epic:** Epic 6 — Inteligência de Preços: Fechamento de Loops
**Status:** Done
**Priority:** P1
**Story Points:** 5

---

## Problema

O `suggestedUnitPriceForSku()` atual calcula preço sugerido com fórmula simples:
```
minPrice = custoBase × regionFactor × (1 + frete) / (1 - opex - impostos - margem)
reference = max(minPrice, historyMedian)
```

Faltam ajustes inteligentes baseados em dados que o sistema JÁ coleta mas não usa:
- **taxaConversão** (ganho/perdido) — se converte pouco, preço está alto
- **menorConcorrente** — se concorrente ganha com preço X, preciso estar perto
- **preçoÚltimoGanho** — referência escola-específica do último preço aceito
- **recorrência** — item frequente justifica margem menor (volume compensa)
- **urgência** — prazo curto justifica preço mais agressivo

O `suggestedBid()` já tem fator de urgência parcial, mas não integra os outros signals.

## Scope

### IN
- Refatorar `suggestedUnitPriceForSku()` para consumir `price-history-summary.json` enriquecido (Story 6.3)
- Adicionar 5 signals de ajuste ao cálculo de margem
- Manter fallback: se dados insuficientes, usar cálculo atual (sem quebrar)
- Exibir breakdown do cálculo na UI (tooltip ou card expandível)

### OUT
- Dashboard de analytics completo (futuro)
- Machine learning / regressão (overengineering para o volume atual)

### DEPENDS ON
- **Story 6.3** — Price history enriquecido com taxaConversao, medianGanho, medianPerdido

## Acceptance Criteria

### AC-1: Signal de Taxa de Conversão
- [ ] Ler `taxaConversao` do `price-history-summary.json` (campo novo da Story 6.3)
- [ ] Ajuste:
  - `taxaConversao < 0.50` → reduzir margem em 5pp (preço está alto)
  - `taxaConversao > 0.80` → aumentar margem em 5pp (espaço pra subir)
  - `0.50 ≤ taxa ≤ 0.80` → manter margem padrão
- [ ] Se dados insuficientes (< 3 resultados no SKU) → não aplicar ajuste

### AC-2: Signal de Concorrência
- [ ] Ler `concorrentes[]` do banco de preços (já populado por `alimentarBancoComResultado`)
- [ ] Calcular `menorConcorrente` = min(concorrentes[].preco) dos últimos 90 dias
- [ ] Ajuste:
  - Se `precoSugerido > menorConcorrente × 1.15` → reduzir para `menorConcorrente × 1.10`
  - Se `precoSugerido < custoBase × 1.05` → FLOOR no custo + 5% (nunca abaixo do custo)
- [ ] Se sem dados de concorrência → não aplicar ajuste

### AC-3: Signal de Referência Escola-Específica
- [ ] Buscar último preço ganho na mesma escola + SKU similar
- [ ] Se encontrar e for dos últimos 6 meses → usar como `precoReferencia`
- [ ] Ajuste: `precoSugerido = max(floor, min(precoReferencia × 1.05, precoSugerido))`
- [ ] Permitir até +5% acima da última referência ganha

### AC-4: Signal de Recorrência
- [ ] Contar aparições do SKU nos últimos 12 meses de histórico
- [ ] Ajuste:
  - `recorrencia ≥ 10` → reduzir margem em 3pp (volume compensa)
  - `recorrencia < 3` → aumentar margem em 2pp (oportunidade rara)
- [ ] Se sem histórico suficiente → não aplicar

### AC-5: Breakdown visual na UI
- [ ] Ao exibir preço sugerido, mostrar ícone de info (ℹ️) com tooltip:
  ```
  Preço Sugerido: R$ 8,06
  ├─ Custo base: R$ 6,20
  ├─ Margem base: 18%
  ├─ Ajuste conversão: -5pp (taxa 42%)
  ├─ Ajuste concorrência: —
  ├─ Ref. escola: R$ 7,80 (EE Brasil, mar/26)
  └─ Recorrência: 15× (margem -3pp)
  ```
- [ ] Se nenhum signal aplicado, mostrar: "Cálculo padrão (sem dados suficientes)"

### AC-6: Fallback e segurança
- [ ] NUNCA sugerir preço abaixo de `custoBase × 1.05` (margem mínima 5%)
- [ ] NUNCA sugerir preço acima de `custoBase × 2.5` (250% — outlier)
- [ ] Se todos os signals falharem, retornar cálculo v1 atual (sem quebrar)
- [ ] Log no console com cada signal aplicado e valor ajustado
