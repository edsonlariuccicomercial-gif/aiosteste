## Story

**Como** operador do Licit-AIX,
**Quero** que ao importar Excel/PDF/NF os preços sejam gravados na tabela `custos_fornecedores` separada (não sobrescrevam o produto),
**Para que** eu tenha múltiplos registros de custo por fornecedor × data × região, com campos TCO (frete, prazo pagamento, confiabilidade).

---

## Acceptance Criteria

- [ ] **AC1:** Import de Excel grava em `custos_fornecedores` com `origem: "excel"` e `confiabilidade: 0.95`
- [ ] **AC2:** Import de PDF (texto) grava com `origem: "pdf"` e `confiabilidade: 0.85`
- [ ] **AC3:** Import de PDF (OCR) grava com `origem: "pdf-ocr"` e `confiabilidade: 0.50`
- [ ] **AC4:** Produto existente na central NÃO é sobrescrito — apenas novo registro de custo é adicionado
- [ ] **AC5:** Produto novo (não existe na central) é criado automaticamente + primeiro custo registrado
- [ ] **AC6:** Campo `fornecedor` capturado do nome do arquivo ou campo mapeado na importação
- [ ] **AC7:** Campo `frete_estimado` disponível no mapeamento de colunas (opcional)
- [ ] **AC8:** Campo `prazo_pagamento_dias` disponível no mapeamento de colunas (opcional)
- [ ] **AC9:** Preview de importação mostra quantos custos novos serão adicionados vs produtos existentes
- [ ] **AC10:** Após import, modal de custos do produto mostra os novos registros com badge de origem

## Dev Notes

### Refatoração principal:
- `app-import.js` → `mergeImportIntoBanco()` deve gravar em `custos_fornecedores` via `addCustoFornecedor()` em vez de sobrescrever `custoBase`
- Manter backward compat: se `centralProdutos` não tiver o produto, criar via `centralProdutos.push()`
- O produto na central fica "limpo" (sem preço embutido) — preços vivem na tabela de custos

### Confiabilidade automática por origem:

| Fonte | Tag | Score |
|-------|-----|-------|
| Nota Fiscal XML | nf | 1.00 |
| Excel | excel | 0.95 |
| Manual | manual | 0.90 |
| B2B | b2b | 0.85 |
| PDF texto | pdf | 0.85 |
| PDF OCR | pdf-ocr | 0.50 |
| API | api | 0.90 |

### Arquivos a modificar:
- `app-import.js` — `mergeImportIntoBanco()` e `mergeMapaIntoBanco()`
