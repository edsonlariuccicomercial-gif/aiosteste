## Story

**Como** operador do painel caixa escolar,
**Quero** que "Central de Preços" seja renomeada para "Central de Produtos" em todo o sistema, e que todas as funcionalidades disponíveis no GDP sejam acessíveis também no módulo IntelPreços como atalho,
**Para que** eu não tenha confusão entre dois nomes para a mesma coisa e possa gerenciar produtos sem sair do fluxo de inteligência de preços.

---

## Acceptance Criteria

### Renomeação
- [x] AC1: Todas as referências visuais "Central de Preços" renomeadas para "Central de Produtos" (labels, títulos, abas, menus)
- [x] AC2: Todas as referências no código (variáveis, comentários, logs) atualizadas para "Central de Produtos" ou "centralProdutos"
- [x] AC3: Storage keys mantidas inalteradas para não quebrar dados existentes (`gdp.produtos.v1` permanece)
- [x] AC4: Tooltips e textos de ajuda atualizados

### Espelhamento no IntelPreços
- [x] AC5: Módulo IntelPreços possui aba/seção "Central de Produtos" com as mesmas funcionalidades do GDP
- [x] AC6: As funcionalidades espelhadas incluem: listagem, busca, cadastro, edição, exclusão, filtros, exportação
- [x] AC7: A aba no IntelPreços funciona como atalho — mesma fonte de dados (`gdp.produtos.v1`), não duplicação
- [x] AC8: Alterações feitas na Central via IntelPreços refletem imediatamente no GDP e vice-versa

---

## Scope

### IN
- Renomear labels/títulos "Central de Preços" → "Central de Produtos" em todo o frontend
- Renomear variáveis/comentários no código
- Adicionar aba "Central de Produtos" no IntelPreços apontando para mesma fonte de dados
- Garantir paridade funcional entre GDP e IntelPreços

### OUT
- Alterar storage keys (preservar compatibilidade)
- Criar nova estrutura de dados separada
- Modificar lógica de negócio da Central
