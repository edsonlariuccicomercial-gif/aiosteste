# Story 1.3 — Cadastro de Produtos: Alinhamento, Categoria, Origem NF-e e Importação Excel

## Status: InProgress

## Complexity: L (Large)
- Bloco 1 Alinhamento: S | Bloco 2 Categoria: M | Bloco 3 Origem: S | Bloco 4 Import Excel: L
- 21 ACs, 1 dependência CDN (SheetJS), múltiplas funções novas

## Descricao

Como operador do GDP na seção Inteligência > Produtos, quero que o card de Cadastro de Produtos tenha colunas alinhadas corretamente, um campo de Categoria (com CRUD de categorias), um campo de Origem do Produto (obrigatório para emissão de NF-e/SEFAZ) e a funcionalidade de importar produtos via planilha Excel com modelo para download, para manter o cadastro organizado e pronto para emissão de nota fiscal sem retrabalho.

## Acceptance Criteria

### Bloco 1 — Fix Alinhamento de Colunas
- [x] **AC1:** Colunas do `<thead>` da tabela de Produtos (Escola, Produto, Preço Unit., Estoque) devem ter `width` fixo proporcional para manter alinhamento com o `<tbody>` — usar `table-layout: fixed` ou larguras percentuais
- [x] **AC2:** Colunas de preço e estoque alinhadas à direita (`text-align: right`) para legibilidade numérica
- [x] **AC3:** Alinhamento consistente em resoluções desktop (1024px+) e mobile (<768px)

### Bloco 2 — Campo Categoria
- [x] **AC4:** Nova coluna "Categoria" visível na tabela de produtos (entre Produto e Preço Unit.)
- [x] **AC5:** Filtro por categoria no dropdown de controles (ao lado do filtro de escola existente)
- [x] **AC6:** Modal/painel de "Gerenciar Categorias" acessível via botão ao lado do filtro — permite criar, editar e excluir categorias
- [x] **AC7:** Categorias persistidas em `localStorage` (chave: `gdp-categorias`) com estrutura `[{id, nome, cor}]`
- [x] **AC8:** Cada categoria pode ter uma cor associada (badge colorido na tabela)
- [x] **AC9:** Ao cadastrar/editar produto, campo select de categoria disponível

### Bloco 3 — Campo Origem do Produto (NF-e)
- [x] **AC10:** Nova coluna "Origem" na tabela de produtos (após Categoria)
- [x] **AC11:** Campo select com as 8 opções padronizadas SEFAZ:
  - `0` — Nacional
  - `1` — Estrangeira - Importação direta
  - `2` — Estrangeira - Adquirida no mercado interno
  - `3` — Nacional - Mercadoria com conteúdo de importação superior a 40%
  - `4` — Nacional - Produção conforme processos produtivos básicos
  - `5` — Nacional - Mercadoria com conteúdo de importação inferior ou igual a 40%
  - `6` — Estrangeira - Importação direta, sem similar nacional (CAMEX)
  - `7` — Estrangeira - Adquirida no mercado interno, sem similar nacional (CAMEX)
- [x] **AC12:** Valor default "0 — Nacional" pré-selecionado (maioria dos produtos é nacional)
- [x] **AC13:** Origem exibida na tabela como badge com código e label resumido (ex: "0-Nacional", "1-Import.Direta")
- [x] **AC14:** Origem incluída no export CSV existente

### Bloco 4 — Importação via Excel
- [x] **AC15:** Botão "Importar Excel" ao lado do botão "Exportar CSV" existente na barra de controles
- [x] **AC16:** Botão "Baixar Modelo" que gera e faz download de uma planilha `.xlsx` modelo contendo: colunas (Escola, Produto, Categoria, Origem, Preço Unitário, Estoque) + 2 linhas de exemplo preenchidas + aba "Referência" com tabela de códigos de origem
- [x] **AC17:** Ao clicar "Importar Excel", abre file picker aceitando `.xlsx` e `.csv`
- [x] **AC18:** Preview dos dados importados em modal antes de confirmar — exibe tabela com os dados lidos e indicadores de erro (linhas com campos obrigatórios vazios destacadas em vermelho)
- [x] **AC19:** Validações na importação: produto (obrigatório), preço (numérico >= 0), origem (código 0-7 válido), escola (deve existir no sistema)
- [x] **AC20:** Após confirmação, produtos importados são mesclados com `allProducts[]` e a tabela re-renderiza
- [x] **AC21:** Dados importados persistidos em `localStorage` (chave: `gdp-produtos-custom`) separados dos dados ARP originais

### Referência — Códigos de Origem SEFAZ (campo `<orig>` no XML NF-e):

| Código | Descrição |
|--------|-----------|
| 0 | Nacional, exceto as indicadas nos códigos 3, 4 e 5 |
| 1 | Estrangeira — Importação direta, exceto a indicada no código 6 |
| 2 | Estrangeira — Adquirida no mercado interno, exceto a indicada no código 7 |
| 3 | Nacional — Mercadoria ou bem com Conteúdo de Importação superior a 40% |
| 4 | Nacional — Cuja produção tenha sido feita em conformidade com os processos produtivos básicos |
| 5 | Nacional — Mercadoria ou bem com Conteúdo de Importação inferior ou igual a 40% |
| 6 | Estrangeira — Importação direta, sem similar nacional, constante em lista CAMEX |
| 7 | Estrangeira — Adquirida no mercado interno, sem similar nacional, constante em lista CAMEX |

> **IMPORTANTE:** O campo origem é OBRIGATÓRIO na emissão de NF-e. Ele compõe a tag `<orig>` dentro do grupo `<ICMS>` de cada item. Sem ele, a SEFAZ rejeita a nota. Atualmente o sistema (caixa-escolar) usa hardcoded `origem: "0"` — este campo no cadastro elimina essa limitação.

## Scope

**IN:**
- Fix alinhamento visual da tabela de produtos
- Campo categoria com CRUD e persistência localStorage
- Campo origem do produto com códigos SEFAZ
- Importação de produtos via Excel com modelo para download
- Preview e validação antes de importar

**OUT:**
- Integração com Supabase (futuro)
- Edição inline de produtos na tabela
- Sincronização com API externa
- Emissão de NF-e (outra story)

## Risks

- SheetJS CDN pode ficar indisponível — mitigação: fallback para import CSV puro
- localStorage tem limite de ~5MB — suficiente para milhares de produtos, mas não infinito
