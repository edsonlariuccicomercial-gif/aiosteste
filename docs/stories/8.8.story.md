# Story 8.8 — Persistir Contrato no Supabase — Dual-Layer Async

**Epic:** Epic 8 — Consolidacao GDP e Fechamento de Gaps Operacionais
**Status:** Ready
**Priority:** P1
**Story Points:** 2

## Descricao

A Story 8.7 cria contratos automaticamente em localStorage. Esta story adiciona a camada de persistencia no Supabase (tabela `contratos`), seguindo o padrao dual-layer ja estabelecido no sistema: gravacao imediata em localStorage + upsert async no Supabase com retry.

O padrao de referencia e `_SB_RESULTADOS.upsert()` ja implementado em `app-results.js`.

**Referencia SOP:** Etapa 11 — Contrato (persistencia permanente)

## Notas Tecnicas

### Padrao Dual-Layer Existente

O sistema ja usa este padrao para resultados. Referencia em `app-results.js`:

```javascript
// Padrao existente — _SB_RESULTADOS
const _SB_RESULTADOS = {
  async upsert(resultado) {
    try {
      const { error } = await supabase
        .from('resultados_orcamento')
        .upsert(resultado, { onConflict: 'pre_orc_id' });
      if (error) throw error;
    } catch (err) {
      console.error('Supabase upsert falhou:', err);
      // localStorage ja tem os dados — retry posterior
    }
  }
};
```

### Novo: _SB_CONTRATOS

```javascript
const _SB_CONTRATOS = {
  async upsert(contrato) {
    const payload = {
      id: contrato.id,                    // CTR-XXXXXXXX-XXXX
      empresa_id: contrato.empresa_id,
      pre_orc_id: contrato.preOrcId,
      escola: contrato.escola,
      origem: contrato.origem,
      resultado_id: contrato.resultado_id,
      itens: contrato.itens,              // JSONB com produto_central_id, sku, ncm
      status: contrato.status,
      created_at: contrato.dataCriacao,
    };

    for (let attempt = 1; attempt <= 3; attempt++) {
      try {
        const { error } = await supabase
          .from('contratos')
          .upsert(payload, { onConflict: 'pre_orc_id' });
        if (error) throw error;
        return; // Sucesso
      } catch (err) {
        console.warn(`Supabase contratos upsert tentativa ${attempt}/3:`, err);
        if (attempt < 3) await sleep(attempt * 1000); // Backoff: 1s, 2s
      }
    }
    console.error('Supabase contratos upsert falhou apos 3 tentativas');
  }
};
```

### Tabela `contratos` no Supabase

A tabela ja existe (migration 001). Os campos adicionais necessarios sao inseridos dentro do JSONB `itens` e como campos opcionais:

- `origem` TEXT — `'manual'` | `'auto-resultado'` | `'caixa-escolar'`
- `resultado_id` TEXT — referencia ao resultado que originou
- `pre_orc_id` TEXT — chave para idempotencia

NAO requer nova migration — campos sao adicionais no INSERT e a tabela JSONB `itens` ja aceita campos arbitrarios.

### Integracao com criarContratoGdp()

Ao final de `criarContratoGdp()` (apos salvar em localStorage), chamar:

```javascript
// Async — fire and forget
_SB_CONTRATOS.upsert(contrato).catch(err =>
  console.error('Falha ao persistir contrato no Supabase:', err)
);
```

## Riscos

| Risco | Probabilidade | Mitigacao |
|-------|:-------------:|-----------|
| Tabela `contratos` nao tem coluna `pre_orc_id` para upsert | Media | Verificar schema atual; se ausente, adicionar via ALTER TABLE simples |
| RLS bloqueia upsert se `empresa_id` nao bater | Baixa | Contrato herda `empresa_id` do resultado (CON-G3-003) |
| Retry com backoff gera multiplas chamadas concorrentes | Baixa | Retry sequencial com await; idempotencia do upsert protege |
