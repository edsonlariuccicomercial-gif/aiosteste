## Story
**Como** dono do sistema (e futuro cliente por assinatura),
**Quero** configurar minhas preferências de recebimento (prazo padrão, condição de pagamento, conta de cobrança) numa aba "Finanças",
**Para** não ter que ajustar o prazo manualmente em cada pedido e preparar o produto para multi-empresa (SaaS).

## Contexto / Problema
A aba "Contas Bancárias" na engrenagem já concentra o financeiro (contas, PIX, saldo, API bancária, conciliação), mas não tem preferências de recebimento. O stakeholder quer **renomear** essa aba para "Finanças" e **adicionar** um card de preferências — não criar aba nova.

**Achado arquitetural (ADR):** o dashboard é multi-page; `index.html` (config) e `gdp-contratos.html` (operação) compartilham estado via **localStorage `nexedu.config.*` + cloud sync**, NÃO via import. Getters são **espelhados por página** (ex.: `getBankApiConfig` definido em `app-config.js:381` e em `gdp-notas-fiscais.js:307`, ambos lendo a mesma chave).

## Decisões do ADR (obrigatórias)
- **Chave canônica:** `nexedu.config.financas` → `{ prazoRecebimentoDias: 5, condicaoPagamentoPadrao, contaCobrancaPadraoId }`.
- **Rename:** trocar **apenas o LABEL** "Contas Bancárias" → "Finanças". **MANTER** `data-config-tab="contas-bancarias"` e o id `#config-contas-bancarias` e a chave `nexedu.config.contas-bancarias` (label desacoplado do slug; renomear o slug exigiria migração de dados em produção).
- **`getFinancasConfig()`** com fallback defensivo (default `prazoRecebimentoDias = 5`), definido em `app-config.js` (esta story) e espelhado em `gdp-notas-fiscais.js` (na 20.7).

## Critérios de Aceitação
- **AC1:** *Given* a engrenagem, *When* abro as configurações, *Then* a aba aparece como "Finanças" e todos os cards existentes (contas, lista, API bancária) continuam funcionando normalmente.
- **AC2:** *Given* o card "Preferências de Recebimento", *When* defino prazo = 5 e salvo, *Then* o valor persiste após reload e sincroniza (cloud sync / Supabase `sync_data`).
- **AC3:** *Given* `getFinancasConfig()`, *When* chamado, *Then* retorna `{ prazoRecebimentoDias, condicaoPagamentoPadrao, contaCobrancaPadraoId }` com defaults sãos quando a config está vazia (prazo → 5).
- **AC4:** *Given* o select de conta de cobrança, *When* aberto, *Then* lista as contas bancárias já cadastradas.
- **AC5 (não-regressão):** `data-config-tab`, id e chave de storage de contas bancárias permanecem inalterados.

## Escopo
- **IN:** rename do label da aba; novo card "Preferências de Recebimento"; `FINANCAS_CONFIG_STORAGE_KEY = "nexedu.config.financas"`; `loadFinancasConfig()`, `saveFinancasConfig()`, `getFinancasConfig()`; cloud sync; binding do botão Salvar.
- **OUT:** consumo do prazo no vencimento (20.7); remoção dos hardcodes de 28 (20.8); mexer em `getConfiguredDefaultBankAccount` (conta fiscal — risco residual do ADR).
