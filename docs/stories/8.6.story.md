# Story 8.6 — Tabela de Conversao de Unidades e Sugestao de Preco Corrigido

**Epic:** Epic 8 — Consolidacao GDP e Fechamento de Gaps Operacionais
**Status:** Ready
**Priority:** P0
**Story Points:** 2

## Descricao

Com o modal de revisao implementado (Story 8.5), esta story adiciona a logica de conversao de unidades e sugestao de preco corrigido. Quando o sistema detecta que a unidade do SGD difere da unidade cotada (ex: "CX C/ 12" vs "UN"), ele calcula automaticamente o preco corrigido e oferece ao operador aceitar ou manter o original.

A tabela de conversao e extensivel e armazenada em localStorage para funcionar offline.

**Referencia SOP:** Etapa 5 — Revisao da Cotacao (prevencao de erro de unidade)

## Notas Tecnicas

### Tabela de Conversao (FR-G1-003)

```javascript
const CONVERSOES_DEFAULT = [
  { de: 'CX C/ 12',  para: 'UN', fator: 12,   direcao: 'dividir' },
  { de: 'CX C/ 6',   para: 'UN', fator: 6,    direcao: 'dividir' },
  { de: 'PCT C/ 10',  para: 'UN', fator: 10,   direcao: 'dividir' },
  { de: 'FD C/ 12',   para: 'UN', fator: 12,   direcao: 'dividir' },
  { de: 'KG',         para: 'UN', fator: null,  direcao: 'calcular' }, // peso_medio
  { de: 'LT',         para: 'ML', fator: 1000,  direcao: 'multiplicar' },
  { de: 'CX',         para: 'UN', fator: null,  direcao: 'lookup' },   // qtd_embalagem da Central
];
```

### Armazenamento

```javascript
// Inicializacao (se nao existir)
if (!localStorage.getItem('gdp.unit-conversion.v1')) {
  localStorage.setItem('gdp.unit-conversion.v1', JSON.stringify(CONVERSOES_DEFAULT));
}

// Leitura (sempre do localStorage — offline-first)
const conversoes = JSON.parse(localStorage.getItem('gdp.unit-conversion.v1'));
```

### Calculo de Preco Corrigido (FR-G1-004)

```javascript
function calcularPrecoCorrigido(precoOriginal, unidadeSgd, unidadeCentral, conversoes) {
  const regra = conversoes.find(c =>
    normalizarUnidade(c.de) === normalizarUnidade(unidadeSgd) &&
    normalizarUnidade(c.para) === normalizarUnidade(unidadeCentral)
  );

  if (!regra || !regra.fator) return null; // Sem regra ou fator dinamico

  switch (regra.direcao) {
    case 'dividir':     return precoOriginal * regra.fator; // UN -> CX: multiplica
    case 'multiplicar': return precoOriginal * regra.fator;
    case 'calcular':    return null; // Requer peso_medio — operador decide
    case 'lookup':      return null; // Requer qtd_embalagem da Central
    default:            return null;
  }
}
```

### Integracao com buildSgdPayload()

Quando o operador aceita uma sugestao de preco, o valor corrigido e salvo no item do pre-orcamento:

```javascript
item.precoCorrigido = precoSugerido;
item.correcaoAceita = true;
item.precoOriginal = item.preco; // Preservar original
item.preco = precoSugerido;      // Atualizar preco efetivo
```

A funcao `buildSgdPayload()` em `app-sgd-integration.js:58` ja usa `item.preco`, entao o payload refletira automaticamente o preco corrigido.

## Riscos

| Risco | Probabilidade | Mitigacao |
|-------|:-------------:|-----------|
| Pares de conversao insuficientes para cenarios reais | Alta | Comecar com top 7; log divergencias sem match; operador pode manter original |
| Direcao de conversao invertida (multiplicar vs dividir) | Media | Testes com exemplos reais do SOP; validacao com operador |
| Preco corrigido sobrescreve acidentalmente preco original | Baixa | Preservar `precoOriginal` sempre antes de atualizar |
