# Story 8.20 — Bug fixes GDP + Reestruturação Relatórios Financeiros

- **Epic:** EPIC-8 — GDP Gaps
- **Status:** Ready
- **Complexity:** L (Large)
- **Agent:** @dev
- **Depends on:** None

## Descricao

Corrigir 2 bugs criticos no GDP (data prevista nao persiste, produto critico desmarca) e reestruturar completamente a secao Relatorios com pagina limpa estilo menu, relatorios financeiros profissionais (DRE, Fluxo de Caixa, Entradas/Saidas por Categoria) e relatorio de vendas (Pendencias de Entrega).

## Acceptance Criteria

### AC1: Bug — Data prevista de entrega nao salva
- **Given** um pedido aberto no detalhe
- **When** o usuario altera o campo "Data prevista de entrega"
- **Then** o valor deve persistir (salvar no localStorage e Supabase)
- **Root cause:** Input `detalhe-data-prevista-{id}` em `gdp-pedidos.js:1355` sem handler `onchange`
- **Fix:** Adicionar `onchange="salvarDataPrevistaPedido('{id}', this.value)"` no input

### AC2: Bug — Produto critico desmarca sozinho na Central de Produtos
- **Given** um produto marcado como "critico" no banco de produtos
- **When** o usuario salva e recarrega a pagina
- **Then** o campo `produto_critico` deve persistir corretamente
- **Root cause:** `PRODUTO_DEFAULTS` em `gdp-banco-produtos.js` nao inclui `produto_critico` nem `embalagem_descricao`, fazendo `getProdutoComDefaults()` descartar esses campos no save
- **Fix:** Adicionar `produto_critico: false` e `embalagem_descricao: ""` ao `PRODUTO_DEFAULTS`

### AC3: Relatorios — Pagina inicial limpa com menu
- **Given** o usuario navega para a aba Relatorios
- **When** a aba carrega
- **Then** deve exibir uma pagina limpa e organizada com duas secoes:
  - **Financeiro:** lista com DRE, Fluxo de Caixa, Entradas e Saidas por Categoria (cada item com titulo bold + descricao em cinza)
  - **Vendas:** lista com Pendencias de Entrega
- **Then** os KPIs e tabelas atuais devem ser removidos da pagina inicial
- **Then** cada item da lista deve ser clicavel e abrir a tela de configuracao do respectivo relatorio

### AC4: Relatorio DRE — Demonstracao do Resultado do Exercicio
- **Given** o usuario clica em "DRE" na lista de relatorios
- **When** a tela de configuracao abre
- **Then** deve exibir:
  - Seletor de **Ano** com setas < > (default: ano atual)
  - Dropdown **"Baseado em"** com opcao "Notas fiscais" (default)
  - Botao **"Gerar"**
- **When** o usuario clica em "Gerar"
- **Then** deve renderizar tabela mensal (Jan-Dez + Total) com linhas:
  - **RECEITA BRUTA** (soma de NFs autorizadas do mes)
  - Notas Mercadoria (NFs com itens de produto)
  - Quantidade (total de itens)
  - Ticket medio (receita / quantidade)
  - **(-)DEDUCOES** (NFs canceladas/devolvidas)
  - Devolucoes
  - **(=)LUCRO BRUTO** (Receita - Deducoes)
  - **(-)DESPESAS OPERACIONAIS** (contasPagar agrupadas por categoria)
  - Linhas por categoria: Agua/luz, Telecomunicacao, Fornecedores, Combustivel, Salarios, Funcionario, Frete, etc.
  - **(=)RESULTADO OPERACIONAL** (Lucro Bruto - Despesas)
  - **(+)OUTRAS RECEITAS** (contasReceber com categorias nao-venda)
  - Emprestimo, etc.
  - **(-)OUTRAS DESPESAS** (contasPagar com categorias nao-operacionais)
  - **(-)TRIBUTOS** (contasPagar categoria impostos/taxas)
  - **(=)RESULTADO LIQUIDO** (Resultado Operacional + Outras Receitas - Outras Despesas - Tributos)
- **Then** deve ter botoes: "Voltar", "Imprimir", "Exibir filtros"
- **Data sources:** `notasFiscais` (receita), `contasPagar` (despesas), `contasReceber` (outras receitas)

### AC5: Relatorio Entradas e Saidas por Categoria
- **Given** o usuario clica em "Entradas e Saidas por Categoria"
- **When** a tela de configuracao abre
- **Then** deve exibir:
  - Toggle **Periodo:** "Do mes" | "Intervalo"
  - Seletor de **Mes** com setas < > (default: mes atual, formato MM/AAAA)
  - Botao **"Gerar"**
- **When** o usuario clica em "Gerar"
- **Then** deve renderizar tabela com colunas: Categoria | Valor | %
- **Then** secao **Entradas** com cada categoria de contasReceber + linha **Total de Entradas** (bold, fundo escuro)
- **Then** secao **Saidas** com cada categoria de contasPagar + linha **Total de Saidas** (bold, fundo escuro)
- **Then** deve ter botoes: "Voltar", "Imprimir", "Exibir filtros"
- **Data sources:** `contasReceber` (entradas), `contasPagar` (saidas), filtradas por mes/periodo

### AC6: Relatorio Fluxo de Caixa
- **Given** o usuario clica em "Fluxo de Caixa"
- **When** a tela de configuracao abre
- **Then** deve exibir:
  - Toggle **Periodo:** "Do mes" | "Ate o mes" | "Intervalo"
  - Seletor de **Mes** com setas < > (default: mes atual)
  - Nota: "Relatorio considerara o periodo do dia atual ate o ultimo dia do mes."
  - Botao **"Gerar"**
- **When** o usuario clica em "Gerar"
- **Then** deve renderizar tabela de fluxo com entradas e saidas por data, saldo acumulado
- **Then** deve ter botoes: "Voltar", "Imprimir", "Exibir filtros"
- **Data sources:** `contasReceber` (entradas com datas), `contasPagar` (saidas com datas)

### AC7: Relatorio Pendencias de Entrega (reposicionamento)
- **Given** o usuario clica em "Pendencias de Entrega" na secao Vendas
- **Then** deve abrir a mesma funcionalidade existente (`renderRelatorioPendenciasEntrega`)
- **Then** com botao "Voltar" para retornar ao menu de relatorios
- **Then** manter botao "Imprimir" existente

### AC8: Navegacao e UX
- **Given** o usuario esta dentro de qualquer relatorio
- **When** clica em "Voltar"
- **Then** retorna ao menu principal de relatorios
- **Then** botao "Exibir filtros" / "Ocultar filtros" deve mostrar/esconder a area de configuracao
- **Then** botao "Imprimir" abre dialogo de impressao via iframe
