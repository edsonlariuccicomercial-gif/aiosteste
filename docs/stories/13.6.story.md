## Story

**Como** operador do Licit-AIX,
**Quero** que o sistema registre automaticamente resultados de licitações (ganho/perdido) via varredura SGD em batch, e que eu possa importar meu histórico retroativo (141+ propostas perdidas),
**Para que** eu tenha inteligência competitiva completa sem registrar manualmente cada resultado.

---

## Acceptance Criteria

### Registro Automático
- [ ] **AC1:** Botão "Atualizar Resultados" na aba Envios SGD varre resultados pendentes
- [ ] **AC2:** Sistema cruza `sgdId` dos pré-orçamentos enviados com status final do SGD
- [ ] **AC3:** Se vencedor = empresa do operador → registro automático "ganho" com dados do contrato
- [ ] **AC4:** Se vencedor ≠ empresa → registro automático "perdido" com `valorVencedor`, `fornecedorVencedor`
- [ ] **AC5:** `deltaTotalPercent` calculado automaticamente (nosso preço vs vencedor)
- [ ] **AC6:** Resultados registrados disparam `schedulCloudSync()`

### Importação Retroativa
- [ ] **AC7:** Botão "Importar Histórico" aceita CSV/Excel com resultados passados
- [ ] **AC8:** Colunas mapeáveis: escola, produto, preço proposto, preço vencedor, resultado (ganho/perdido), data
- [ ] **AC9:** Dados importados alimentam `caixaescolar.resultados.v1` + `historico_licitacoes`
- [ ] **AC10:** Dashboard de resultados atualiza com dados retroativos + novos

## Dev Notes

### Varredura de resultados — como funciona:
- Reutilizar `BrowserSgdClient` para consultar status dos budgets enviados
- Para cada pré-orçamento com status "enviado", verificar se SGD tem resultado final
- Se resultado disponível: extrair vencedor, valor, data → registrar automaticamente

### Importação retroativa — formato CSV esperado:
```
Escola;Produto;Preço Proposto;Preço Vencedor;Fornecedor Vencedor;Resultado;Data
EE Exemplo;Arroz 5kg;27.50;25.00;Concorrente ABC;perdido;2025-03-15
EE Outro;Feijão 1kg;8.90;8.90;LARIUCCI;ganho;2025-04-20
```

### Arquivos a modificar:
- `app-sgd-integration.js` — nova função `varrerResultadosSgd()`
- `app-results.js` — função `importarHistoricoRetroativo()`
- `index.html` — botões "Atualizar Resultados" e "Importar Histórico"
