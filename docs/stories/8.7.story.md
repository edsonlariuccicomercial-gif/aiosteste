# Story 8.7 — Auto-Criar Contrato em Resultado Ganho com Vinculo Central

**Epic:** Epic 8 — Consolidacao GDP e Fechamento de Gaps Operacionais
**Status:** Ready
**Priority:** P1
**Story Points:** 3

---

## Descricao

Quando um orcamento e marcado como "ganho" (`resultado === "ganho"` em `app-results.js:138`), a criacao do contrato hoje depende do operador marcar manualmente um checkbox. Se esquecer, o contrato nao e criado. Alem disso, a deteccao automatica de status SGD (`checarStatusSgd()`) detecta vitoria mas NAO cria contrato.

Esta story torna a criacao de contrato automatica em ambos os fluxos (registro manual e deteccao SGD), vincula os itens do contrato a Central de Produtos unificada (SKU, NCM), e mantém o checkbox como opt-out.

**Referencia SOP:** Etapa 11 — Contrato (pos-vitoria)

---

**Fluxo Automatico (deteccao SGD):**

### Refatoracao de criarContratoGdp()

A funcao atual (`app-results.js:392`) requer interacao com modal. Refatorar para aceitar parametros programaticos:

```javascript
function criarContratoGdp(options = {}) {
  const {
    preOrcId,
    resultado,
    itens,
    escola,
    empresa_id,
    origem = 'manual',           // 'manual' | 'auto-resultado' | 'caixa-escolar'
    resultado_id = null,
    numero = null,               // Se null, gera automaticamente
  } = options;

  // Verificar idempotencia
  const contratos = JSON.parse(localStorage.getItem('gdp.contratos.v1') || '{}');
  if (contratos[preOrcId]) {
    console.log(`Contrato ja existe para preOrcId ${preOrcId}`);
    return contratos[preOrcId];
  }

  // Vincular itens a Central de Produtos
  const itensVinculados = itens.map(item => {
    const match = RadarMatcher.match(item.nome);
    return {
      ...item,
      produtoCentralId: match?.id || null,
      sku: match?.sku || null,
      ncm: match?.ncm || null,
      marca: match?.marca || null,
    };
  });

  // Gerar numero do contrato
  const numeroCtr = numero || gerarNumeroContrato(); // CTR-20260420-0001

  // Salvar
  const contrato = {
    id: numeroCtr,
    preOrcId,
    empresa_id,
    escola,
    origem,
    resultado_id,
    itens: itensVinculados,
    dataCriacao: new Date().toISOString(),
    status: 'ativo',
  };

  contratos[preOrcId] = contrato;
  localStorage.setItem('gdp.contratos.v1', JSON.stringify(contratos));

  // Toast
  mostrarToast(`Contrato ${numeroCtr} criado automaticamente!`, 'success', { link: '#contratos' });

  return contrato;
}
```

### Idempotencia (NFR-G3-001)

- Verificar `gdp.contratos.v1[preOrcId]` antes de criar
- Se ja existe, retornar contrato existente sem duplicar
- Isso cobre o caso de `salvarResultado()` e `checarStatusSgd()` dispararem simultaneamente para o mesmo orcamento (ISS da Critique)

### ISS-4 (Critique): gerarContratoUnificado()

Verificar se `gerarContratoUnificado()` em `app-results.js` tambem precisa de vinculacao com Central. Se sim, aplicar mesmo padrao de matching.
